# 设计原则 Principle

## 1. 文档定位

本文定义 dongDesign Chart 的通用图表设计原则，用于约束所有图表类型、业务场景和 AI 生成结果。

本文不维护具体色值、字号、坐标轴样式或图表组件规则。具体样式值应继续读取：

- `theme-token.md`
- `color.md`
- `typography.md`
- `layout.md`
- `axis.md`
- `legend.md`
- `label.md`
- `tooltip.md`

## 2. 总原则

图表设计必须优先服务业务判断，而不是优先服务装饰效果。

执行顺序：

```txt
业务问题
→ 数据结构
→ 图表选型
→ 视觉编码
→ 信息层级
→ 交互与增强
→ 自查和修正
```

AI 不得跳过业务问题和数据结构，直接根据用户偏好的图表外观生成最终方案。

## 3. 业务优先

| 原则 | 执行规则 | 反例风险 |
|---|---|---|
| 先问要判断什么 | 先明确用户是看比较、趋势、占比、分布、关系、排名还是流程 | 图表好看但无法回答问题 |
| 指标口径优先 | 金额、比例、人数、件数、时长等指标必须保留单位和口径 | 用户误读业务含义 |
| 只表达有证据的判断 | 异常、达标、风险、恶化必须有目标、阈值或基准 | AI 编造结论 |
| 不用复杂图表炫技 | 简单比较优先柱状图、条形图、表格或指标卡 | 增加理解成本 |

## 4. 数据诚实

图表不得通过视觉手段改变数据事实。

必须遵守：

- 不得截断数值轴后仍用夸张标注表达增长或下降。
- 不得删除不利数据点，除非明确说明筛选条件。
- 不得把相关性表达为因果关系。
- 不得把没有目标线的数据判断为达标或未达标。
- 不得把没有异常规则的数据判断为异常。
- 不得通过面积、气泡大小或 3D 透视夸大差异。

## 5. 视觉一致

所有图表共享同一套视觉语言。

| 视觉对象 | 规则来源 | AI 要求 |
|---|---|---|
| 色彩 | `color.md`、`theme-token.md` | 先判断数据关系，再选择分类、顺序、发散或语义色 |
| 字体 | `typography.md` | 坐标轴、图例、Tooltip、标签使用规范字号和行高 |
| 布局 | `layout.md` | 标题、图例、绘图区、轴标签区应有稳定区域关系 |
| 坐标轴 | `axis.md` | 轴类型、刻度、网格线和轴标题必须与字段类型一致 |
| 图例 | `legend.md` | 多系列或多类别时保持图例语义和顺序 |
| 标签 | `label.md` | 标签只服务读数和解释，不铺满图表 |
| Tooltip | `tooltip.md` | Tooltip 展示精确值、单位和必要上下文 |

## 6. 信息层级

图表应建立清晰的阅读顺序。

优先级从高到低：

1. 标题和业务问题。
2. 当前最重要的数据主体。
3. 关键标签、异常点、参考线或区间。
4. 坐标轴、图例、Tooltip。
5. 网格线、背景和辅助结构。

执行规则：

- 数据主体优先，增强元素不得抢占主视觉。
- 同一张图默认只突出最重要的 `1-3` 个重点。
- 图例、轴和标签是辅助理解，不应压缩绘图区到不可读。
- 网格线和背景只支撑定位，不表达业务含义。

## 7. 克制增强

视觉增强必须服务明确阅读任务。

允许：

- 用最新值、最大值、最小值帮助定位重点。
- 用目标线、阈值线、均值线帮助判断差距。
- 用活动区间或事件点解释阶段变化。
- 用 hover、筛选、下钻承接更多细节。

禁止：

- 为了装饰添加箭头、图标、光效或无意义背景。
- 所有点都标注，导致真正重点被淹没。
- 同时使用过多颜色、callout、区间和参考线。
- 只用红绿表达风险，不提供文字、符号或 Tooltip 说明。

## 8. 可维护与可实现

图表设计必须能被设计工具和代码实现稳定承接。

VibeDesign 要求：

- 使用 `chartFrame / headerFrame / bodyFrame / plotAreaFrame` 等结构化图层。
- 标题、图例、坐标轴、Tooltip 和标注保持可编辑结构。
- 数据系列归属于 `seriesLayer`，不得散落在根画布。

VibeCode 要求：

- 字段映射清楚，不能只给静态截图。
- 主题样式绑定 token，不随机写死。
- G2 / ECharts 映射语义一致。
- 交互、空状态、加载态和异常态可验证。

## 9. AI 自查清单

- 是否先明确业务问题和分析目标？
- 是否基于数据结构选择图表？
- 是否保留指标单位、时间范围和口径？
- 是否没有编造异常、风险、达标或原因判断？
- 是否引用了视觉语言规则而不是临时发挥？
- 是否控制了颜色、标签、参考线和增强数量？
- 是否保证图表在设计工具中可编辑、在代码中可实现？
- 是否能通过 `06-self-check/design-checklist.md` 和 `code-checklist.md` 检查？
