# 坐标轴 Axis

## 1. 文档定位

本文定义图表坐标轴的类型、构成、显示规则和设计约束。

本文回答：

> 图表应该使用什么类型的坐标轴？轴标题、轴标签、轴线、刻度线、网格线应该如何显示？

参考来源：

- Figma 文件：`可视化设计实践`
- Figma 节点：`坐标轴 Axis`

本文依赖：

- `01-design-language/theme-token.md`
- `01-design-language/color.md`
- `01-design-language/typography.md`

## 2. 定义

坐标轴是一维空间中统计图表中的轴，用来定义坐标系中数据在方向和值上的映射关系。

坐标轴是坐标系的构成部分，是定义域轴和值域轴的统称。坐标轴所表示的范围越大，轴所包含的概念量越多。

坐标轴用于帮助用户：

- 理解数据方向。
- 判断数据大小。
- 建立数据与图形位置之间的映射。
- 阅读图表中的趋势、分布、差异和比例关系。

## 3. 应用场景

### 3.1 直角坐标系

直角坐标系适合展示具有线性关系、趋势对比或类别对比的数据。

常见结构：

```txt
xAxis = 横向维度或时间
yAxis = 纵向数值或指标
```

适用图表：

- 折线图
- 柱状图
- 条形图
- 散点图
- 区域图
- 组合图

使用规则：

- X 轴通常承载维度、时间或有序分类。
- Y 轴通常承载连续数值。
- 条形图中，维度通常在 Y 轴，指标通常在 X 轴。

### 3.2 极坐标系

极坐标系适合展示环状、角度或比例关系的数据。

常见结构：

```txt
angleAxis = 角度方向
radiusAxis = 半径方向
```

适用图表：

- 饼图
- 玫瑰图
- 雷达图
- 玉玦图

使用规则：

- 角度轴用于表达分类或周期位置。
- 半径轴用于表达数值大小。
- 雷达图中的指示器名称相当于轴标签，必须保持可读。

## 4. 坐标轴构成

虽然不同坐标系下坐标轴的表现形式不同，但构成元素基本一致。

| 构成 | 英文 | 作用 | 变量 / 样式 |
|---|---|---|---|
| 轴标题 | title | 说明轴变量、单位或口径 | `.textSmall` `12/18`，`text/300` |
| 轴线 | axisLine / line | 标识坐标轴基线 | `坐标轴/axisLineColor` |
| 轴刻度 | tick | 标识刻度位置 | `坐标轴/tickColor` |
| 轴标签 | label | 展示刻度值或分类名称 | `.textSmall` `12/18`，`坐标轴/axisLabelColor` |
| 网格线 | grid / splitLine | 辅助跨区域读数 | `坐标轴/splitLineColor` |
| 分隔区域 | splitArea | 辅助分区阅读 | `坐标轴/splitAreaColor` |

基础样式：

| 项目 | 固定规则 |
|---|---|
| 轴标签字号 | `.textSmall` `12/18` |
| 轴标签字重 | `400` |
| 轴标题字号 | `.textSmall` `12/18` |
| 轴标题字重 | `400`，必要时可用 `600` |
| 轴线颜色 | `坐标轴/axisLineColor` |
| 刻度线颜色 | `坐标轴/tickColor` |
| 网格线颜色 | `坐标轴/splitLineColor` |
| 网格线线型 | 笛卡尔坐标系默认 `1px` 虚线，建议 dash `4 / 4` |
| 分隔区域颜色 | `坐标轴/splitAreaColor` |

坐标轴属于辅助阅读元素，视觉优先级必须低于数据图形。

执行约束：

- 轴标签必须显式绑定 `.textSmall`：`font-size: 12px`、`line-height: 18px`、`font-weight: 400`、`letter-spacing: 0`。
- 设计工具生成时，不得依赖默认文本样式；如果未显式设置字号，Figma / Relay / SVG 导入链路可能回退为 `16px`。
- 使用 SVG 作为轻量验证稿时，应在每个轴标签 `text` 或 `tspan` 上显式写入 `font-size="12"`、`font-weight="400"`，不要只依赖 CSS `font` shorthand。
- 使用可编辑节点生成时，必须在创建 TextNode 后设置 `fontSize = 12`、`lineHeight = { value: 18, unit: "PIXELS" }`、`fontWeight = 400` 或对应 Regular 字体样式。
- 验收时若轴标签渲染为 `16px`，即使内容和颜色正确，也判定为坐标轴还原失败。
- 验收时若轴标签与轴线、网格线或绘图区的距离不符合本文布局规格，即使字号正确，也判定为坐标轴还原失败。

## 5. 坐标轴类型

根据数据变量的类型，坐标轴分为以下类型。

| 类型 | 适用数据 | 常见用途 | 规则 |
|---|---|---|---|
| 连续轴 | 连续数值 | 金额、数量、百分比、评分 | 刻度应等距，单位明确 |
| 分类轴 | 无序分类 | 品类、渠道、会员等级 | 保留分类名称完整性 |
| 时间轴 | 时间序列 | 日期、小时、月份、季度 | 按时间粒度抽样显示 |
| 对数轴 | 跨数量级数值 | 金额、规模、流量 | 必须明确说明对数尺度 |
| 对称对数轴 | 含正负且跨度大的数值 | 增长率、偏离值、正负差异 | 必须说明中心值和尺度 |

AI 判断规则：

```txt
如果字段是连续数值 → 连续轴
如果字段是无序类别 → 分类轴
如果字段是日期或时间 → 时间轴
如果数值跨越多个数量级 → 对数轴
如果数值同时包含正负且跨度大 → 对称对数轴
```

## 6. 设计原则

坐标轴设计的核心原则：

```txt
弱化坐标系，突出数据元素。
```

应删除不必要的非数据元素，例如过多轮廓、背景、轴线、网格线，把重点放在数据元素上。

### 6.1 轴标题

轴标题用于说明坐标轴表达的变量、单位或统计口径。

文案规则：

- 简洁准确表达变量含义。
- 不使用过于专业或生僻的术语。
- 特殊格式数据应说明单位、范围或常规数据格式除外的口径，例如百分比、日期时间。
- 标题包含单位时，优先写成 `指标（单位）`，例如 `成交金额（万元）`。

布局规则：

- X 轴标题固定放在底部。
- Y 轴标题默认放在 Y 轴标签上方的左上角说明区，适合 G2 / ECharts / SVG 等运行时实现。
- 当设计组件明确要求竖排标题时，Y 轴标题可放在标签侧方，但必须使用独立 title rail，不得与轴标签同列挤压。
- X 轴标题水平居中。
- 竖排 Y 轴标题应在独立 title rail 内垂直居中。
- Y 轴标题放在标签上方时，必须为标题预留独立 title band；标题底部与首个 Y 轴标签至少保持 `12px` 间距。
- 轴标题应与轴标签保持清晰距离，避免贴合。
- 多个坐标轴标题应保持样式一致。

使用建议：

- 当图表标题、图例或上下文已经清楚说明轴含义时，可以省略轴标题。
- 当单位不明显、左右双轴、对数轴或复杂口径出现时，必须显示轴标题。

### 6.2 轴标签

轴标签用于展示刻度值、时间点或分类名称。

通用规则：

- 轴标签固定使用 `.textSmall` `12/18` `400`。
- 轴标签颜色绑定 `坐标轴/axisLabelColor`。
- 标签不得加粗。
- 标签不得与数据图形、图例、Tooltip 或其他标签重叠。
- 标签密集时，优先抽样、省略、旋转或使用缩略格式。

连续轴 / 时间轴：

- 可采用抽样显示。
- 当多个标签在容器内重叠时，优先保留首尾标签。
- 时间轴应按粒度显示，例如小时、日、周、月，不堆叠完整时间戳。

分类轴：

- 分类数据不具备连续性和相似性，应优先保证数据展示完整性。
- 分类名称过长时，优先使用旋转、换行或截断。
- 不应为了整齐而任意丢弃关键分类标签。

### 6.3 轴标签省略

对于连续轴和时间轴，可以采用抽样显示。

规则：

- 优先保留首尾标签。
- 中间标签按等步长显示。
- 抽样后仍需保持读数连续感。
- 不得让用户误以为被省略的区间没有数据。

适用：

- 小时级时间轴。
- 日期跨度较长的趋势图。
- 连续数值刻度过密的图表。

### 6.4 轴标签旋转

分类轴标签过长或数量较多时，可以旋转标签。

使用规则：

- 仅在横向空间不足时使用旋转。
- 旋转优先用于分类轴，不优先用于连续轴或时间轴。
- 旋转后仍需保证标签可读，不得过密堆叠。
- 如果旋转后仍不可读，应减少分类数量、换行或改用条形图。

不建议：

- 为了显示全部时间刻度而大面积旋转时间轴标签。
- 在窄容器中旋转过多长文本标签。

### 6.5 坐标轴隐藏

当图表已经提供有效数据标签时，可以弱化或隐藏坐标轴。

适用场景：

- 堆叠条形图已经显示百分比数据标签。
- 排名条形图每个条目都有直接数值。
- 简单指标对比图中，轴线和刻度不会增加理解。

规则：

- 隐藏坐标轴前，必须保证数据标签足够有效。
- 若隐藏轴标签，仍需通过数据标签或 Tooltip 提供数值。
- 不得同时隐藏坐标轴和所有数值说明。
- 隐藏后图表仍应能判断大小关系。

## 7. 轴线与网格线

轴线和网格线都属于辅助阅读元素，需要根据数据构成谨慎显示。

### 7.1 显示原则

根据数据构成：

- 维度数据对应轴线。
- 指标数据对应网格线。

常见映射：

| 图表结构 | 建议 |
|---|---|
| 维度在横轴 X | X 轴显示轴线，Y 轴显示水平网格线 |
| 维度在纵轴 Y | Y 轴显示轴线，X 轴显示垂直网格线 |
| 极坐标 / 雷达图 | 使用环形或径向分隔线辅助读数 |

### 7.2 网格线数量

默认情况下，直角坐标系的数值轴刻度 / 网格线建议只显示 `5` 个，形成 `4` 个等距区间。

规则：

- 过多分割线会让图表过于杂乱。
- 柱状图、折线图、面积图等笛卡尔坐标系图表，Y 轴默认从 `0` 起步，不使用数据最小值作为轴起点。
- 默认刻度应均匀分布，例如 `0 / 25 / 50 / 75 / 100` 或 `0 / 50 / 100 / 150 / 200`。
- 不应生成类似 `0 / 10 / 20 / ... / 90` 的密集刻度，也不应在普通线性轴上出现类似对数轴的非均匀间隔。
- 网格线应弱于数据系列。
- 网格线颜色绑定 `坐标轴/splitLineColor`。
- 笛卡尔坐标系的默认网格线 / 分隔线使用 `1px` 虚线，建议 dash `4 / 4`；轴线仍可使用 `1px` 实线。
- 不得把网格线和坐标轴轴线混为同一种线型：分隔线默认虚线，底部 / 左侧基线按组件规范使用实线或隐藏。
- 若图表空间较小，应减少网格线数量。
- 不应为了精确读数而堆叠大量网格线；精确值应由 Tooltip 或数据标签承载。

例外情况必须显式说明：

- 业务明确要求观察局部波动，可以使用非零起点，但必须在轴标题、说明或交互中表达口径。
- 数据跨越多个数量级并确实需要对数轴时，必须声明 `log` 或 `symlog` 尺度，不得伪装成普通线性轴。
- 指数、增长率、偏离值等有业务基准线的指标，可以使用业务基准作为视觉参考，但仍需保证刻度等距且可解释。

### 7.3 数值轴动态取值算法

笛卡尔坐标系默认 `splitNumber = 4`，即：

```txt
tickCount = splitNumber + 1 = 5
ticks = 0 / step / 2step / 3step / 4step
axisMax = step * splitNumber
```

默认算法：

```txt
1. min = 0
2. splitNumber = 4
3. targetMax = dataMax × 1.08
4. rawStep = targetMax / splitNumber
5. step = niceStep(rawStep)
6. axisMax = step × splitNumber
7. 如果 axisMax / dataMax > 1.25，说明顶部留白过大，应改用 precisionStep(rawStep) 计算 step
8. ticks = [0, step, 2step, 3step, axisMax]
```

`niceStep` 推荐优先使用以下倍数：

```txt
1 / 1.25 / 1.5 / 2 / 2.5 / 5 / 10 × 10^n
```

`precisionStep` 用于避免盲目拉伸最大值：

| 数据类型 | precisionStep 建议 |
|---|---|
| 百分比，展示 1 位小数 | `0.1` |
| 小数指标 | 按展示精度向上取整 |
| 金额 / 数量，rawStep >= 100 | `10` 或业务单位 |
| 金额 / 数量，rawStep >= 10 | `5` |
| 金额 / 数量，rawStep < 10 | `1` |

示例：

| dataMax | splitNumber | targetMax | step | axisMax | ticks |
|---|---:|---:|---:|---:|---|
| `5.5%` | `4` | `5.94` | `1.5` | `6.0` | `0 / 1.5 / 3.0 / 4.5 / 6.0` |
| `330万` | `4` | `356.4` | `100` | `400` | `0 / 100 / 200 / 300 / 400` |

判断标准：

- `axisMax` 必须大于等于 `dataMax`。
- `axisMax / dataMax` 默认不应超过 `1.25`；超过时应使用更细的 `precisionStep`。
- 不得为了保持“漂亮整数”把顶部留白拉得过大，导致趋势压扁。
- 不得为了贴合数据最大值而破坏 `4` 个等距分隔区间。

### 7.4 轴线与刻度线

规则：

- 轴线颜色绑定 `坐标轴/axisLineColor`。
- 刻度线颜色绑定 `坐标轴/tickColor`。
- 刻度线只用于辅助定位，不应成为视觉焦点。
- 坐标轴线可在极简趋势图中弱化或隐藏，但必须保留必要读数线索。

### 7.5 直角坐标系布局规格

当目标是还原 Figma / Relay 中的标准直角坐标系组件时，必须先确定绘图区、轴标签轨道和轴线的位置，再绘制数据图形。折线图、柱状图、散点图、气泡图、面积图等笛卡尔坐标系图表共用本节规格。

参考来源：`可视化设计实践 / 坐标轴 Axis / 直角坐标系`。

#### 7.5.1 结构分区

标准直角坐标系应拆成以下区域：

```txt
chartFrame
  title / legend / controls
  plotGroup
    yAxisLabelRail
    plotArea
      grid / splitLine
      axisLine
      series
    xAxisLabelBand
```

布局规则：

- `plotArea` 只包含网格线、轴线、数据图形、hover 指示线等绘图区内容。
- X / Y 轴标签不得放入 `plotArea` 内部，也不得挤占数据点、柱体或折线。
- 网格线必须裁切在 `plotArea` 内，不得延伸到轴标签轨道或标题区域。
- 数据图形的坐标映射必须以 `plotArea` 的左、右、上、下边界为准，不得以整个 chartFrame 为准。

#### 7.5.2 距离规格

| 元素 | 规则 |
|---|---|
| Y 轴标签轨道宽度 | 默认 `50px` |
| 左侧 Y 轴标签右内距 | `10px`，即标签右边缘距离绘图区左边界 / 网格线起点 `10px` |
| 左侧 Y 轴标签对齐 | 数字右对齐，垂直居中对齐对应网格线 / 刻度位置 |
| 右侧异轴标签左内距 | `10px`，即标签左边缘距离绘图区右边界 / 网格线终点 `10px` |
| 右侧异轴标签对齐 | 数字左对齐，垂直居中对齐对应网格线 / 刻度位置 |
| Y 轴标签行高 | `18px`，标签框高度按 `18px` 计算 |
| Y 轴标签垂直定位 | 标签框中心必须与对应轴线 / 分隔线严格对齐；端点标签也不得为了留在轨道内而上移或下移 |
| X 轴标签带高度 | 至少 `28px`，由轴线下方间距 `10px` + 标签行高 `18px` 构成 |
| X 轴标签上边距 | 标签文本框顶部距离底部轴线 `10px` |
| X 轴标签对齐 | 水平居中对齐对应刻度 / 分割位置 |
| 维度轴首尾留白 | 默认保留半个 band 或等效 `categoryPadding`，避免首尾数据、刻度、标签和图形贴住坐标轴两侧边缘 |
| 刻度线长度 | 启用时默认 `4px`，线宽 `1px` |
| 刻度线到标签间距 | X 轴中，若刻度线向下 `4px`，标签仍从轴线下方 `10px` 起排，形成约 `6px` 净距 |
| 轴线线宽 | `1px` |
| 网格线线宽 | `1px`，默认虚线 dash `4 / 4` |

执行公式：

```txt
plotLeft = chartLeft + yAxisLabelRailWidth
plotBottom = chartBottom - xAxisLabelBandHeight
yAxisLabelRight = plotLeft - 10
rightYAxisLabelLeft = plotRight + 10
xAxisLabelTop = plotBottom + 10
dimensionAxisFirstPosition = plotLeft + dimensionAxisInnerPadding
dimensionAxisLastPosition = plotRight - dimensionAxisInnerPadding
```

可编辑设计稿中建议直接建立 `yAxisLabelRail` 和 `xAxisLabelBand` 两个容器，避免每个图表类型各自手算偏移。维度轴上承载图表数据时，应建立独立的 `dimensionScaleRange` 或等效映射区，将首尾数据点从 `plotArea` 左右边缘内缩；默认内缩量可取一个类目带宽的 `0.5`，或在连续时间轴上取 `plotWidth / dataCount / 2` 的近似值。

#### 7.5.3 生成禁忌

- 不得把 X 轴标签的 `10px` 下偏移误用为所有方向上的统一间距。
- 不得让 Y 轴标签中心或左边缘距离轴线 `10px`；规范约束的是标签右边缘到绘图区左边界 `10px`。
- 不得把右侧异轴标签也做成右对齐；右侧异轴标签约束的是标签左边缘到绘图区右边界 `10px`。
- 不得为了避免端点标签超出轴标签轨道而改变其垂直中心；`0`、最大值等两端标签仍必须与轴线 / 分隔线中心对齐。
- 不得让 X 轴标签、Y 轴标签和网格线使用同一组 `x/y` 起点；标签属于轴外辅助区，网格线属于绘图区。
- 不得为散点图、气泡图单独发明坐标轴间距；它们必须继承直角坐标系布局规格。
- 不得让维度轴上的首尾数据点、柱体、刻度或标签直接贴住 `plotArea` 两侧边缘；除非业务明确要求边界对齐，否则必须保留首尾留白，让图表有呼吸感。

### 7.6 折线图坐标轴还原规格

当目标是还原 Figma / Relay 中的标准折线图组件时，坐标轴应采用更具体的结构，而不是只画通用直角坐标系。

参考规格：

| 元素 | 规则 |
|---|---|
| X 轴轴线 | 仅底部显示 `1px` 实线，绑定 `坐标轴/axisLineColor` |
| Y 轴轴线 | 默认不显示左侧实线，仅显示标签和水平网格线 |
| 水平网格线 | 使用 `坐标轴/splitLineColor`，默认 `1px` 虚线，dash `4 / 4` |
| 网格线数量 | `5` 条，对应 `0 / 25 / 50 / 75 / 100` 等均分刻度，形成 `4` 个等距区间 |
| Y 轴起点 | 默认 `0`；不得默认用数据最小值作为起点 |
| Y 轴标签 | 数字右对齐，标签区宽度建议 `50px`，与网格线间距 `10px` |
| X 轴标签 | 时间序列可保留首尾和固定间隔标签，其余标签透明或隐藏 |
| X 轴标签偏移 | 标签位于底部轴线下方 `10px` 左右 |
| hover 指示线 | 垂直 `1px` 实线，绑定 `提示/tooltipAxisColor`；不得使用虚线或 dash |
| hover 辅助线 | 默认只显示垂直指示线，不显示横向 crosshair，除非业务明确需要同时定位 X/Y 值 |
| 轴标签文本 | 固定 `.textSmall 12/18 400`；不得回退为设计工具默认 `16px` |
| 轴标题布局 | 优先使用 Y 轴标签上方的左上角说明区，并预留 title band；需要竖排时必须使用独立 title rail，不得挤占轴标签区 |

折线图的坐标轴应优先弱化容器感，避免出现完整外框。上一次 Relay 验证中出现的左侧实线、过重网格和过多图表卡片装饰，均属于还原度不足的表现。

线型区分约束：

- 网格线 / 分隔线可以使用 `1px` 虚线，建议 dash `4 / 4`。
- hover 指示线不是网格线，必须是单根垂直 `1px` 实线。
- Figma / Relay 设计稿中应优先使用原生 Line / Rectangle 或等价单节点绘制 hover 指示线；不得用大量短矩形拼接成虚线。
- 若使用 G2 / ECharts，应显式关闭默认 crosshair 中不需要的横线，并设置 axisPointer / hover layer 为垂直实线。

## 8. 异轴与双轴

异轴，也就是双轴图，指在同一个图表中使用两个不同的 Y 轴比较两个或多个数据系列。

适用场景：

- 两个数据系列单位不同。
- 两个数据系列值域差异很大。
- 需要展示两个数据系列之间的关系。

使用规则：

- 双轴必须明确左右轴分别对应哪个系列。
- 左右轴标题必须显示单位。
- 左右轴样式应保持一致。
- 左侧 Y 轴标签默认右对齐，标签右边缘距离绘图区左边界 `10px`。
- 右侧 Y 轴标签默认左对齐，标签左边缘距离绘图区右边界 `10px`。
- 左右 Y 轴默认都从 `0` 起步，并尽量保持相同数量的等距刻度。
- 如果双轴因为单位差异必须使用不同最大值，仍应控制每侧刻度 / 网格线约 `5` 个，避免某一侧出现密集分割线。
- 不得通过双轴制造虚假的趋势相关性。
- 如果两个指标可以统一单位或归一化，应优先统一尺度，而不是使用双轴。

推荐：

- 左右 Y 轴都清楚显示单位和刻度。
- 系列颜色、图例和轴标题之间应建立明确对应关系。

不建议：

- 双轴标题位置不一致。
- 一个轴有标题，另一个轴无标题。
- 用双轴把无关指标强行放在同一图表中。

## 9. 与主题变量的关系

坐标轴组件必须绑定主题变量，不得写死颜色。

| 轴元素 | 绑定变量 | Light | Dark |
|---|---|---|---|
| 轴线 | `坐标轴/axisLineColor` | `#D9D9D9` | `#666666` |
| 刻度线 | `坐标轴/tickColor` | `#D9D9D9` | `#666666` |
| 网格线 | `坐标轴/splitLineColor` | `#EBEBEB` | `#4A4A4A` |
| 分隔区域 | `坐标轴/splitAreaColor` | `#F7F7F7` | `#333333` |
| 轴标签 | `坐标轴/axisLabelColor` | `#8C8C8C` | `#FFFFFF` |

说明：

- Light / Dark 只改变变量解析值，不改变坐标轴结构。
- 坐标轴颜色必须弱于数据系列。
- 如果暗色模式下轴标签过亮或过弱，应调整 mode token，而不是在单个图表中写死色值。

## 12. 禁止项

- 禁止让坐标轴视觉强于数据系列。
- 禁止在没有单位说明时使用复杂数值轴。
- 禁止分类轴随意省略关键分类标签。
- 禁止连续轴或时间轴标签密集重叠。
- 禁止为了展示全部时间标签而大面积旋转标签。
- 禁止隐藏坐标轴后不提供任何数值说明。
- 禁止网格线数量过多导致图表杂乱。
- 禁止柱状图、折线图在无特殊说明时使用数据最小值作为 Y 轴起点。
- 禁止普通线性轴出现非均匀刻度间隔，制造类似对数轴的视觉效果。
- 禁止把轴标签放入绘图区内部，或让轴标签轨道、绘图区、网格线共用错误起点。
- 禁止将 `10px` 间距泛化为所有轴元素的统一边距；必须区分 Y 轴标签右内距和 X 轴标签上边距。
- 禁止不说明原因就使用双轴。
- 禁止写死坐标轴颜色，必须绑定主题变量。
- 禁止在图表内部使用非 `12/18 400` 的轴标签样式。

## 13. AI 自查清单

生成或审核图表时检查：

- 是否判断了坐标系类型？
- X 轴和 Y 轴的数据类型是否正确？
- 轴标题是否必要，单位是否清楚？
- 轴标签是否使用 `.textSmall 12/18 400`？
- 轴标签是否绑定 `坐标轴/axisLabelColor`？
- 轴线、刻度线、网格线是否绑定对应变量？
- 时间轴或连续轴标签是否做了合理抽样？
- 分类轴标签是否完整可读？
- 标签旋转后是否仍然可读？
- 隐藏坐标轴时是否有有效数据标签或 Tooltip 补充？
- 数值轴是否默认从 `0` 起步？
- 网格线 / 刻度数量是否控制在默认 `5` 个左右，并保持等距？
- 是否避免了无说明的对数轴、非均匀刻度或数据最小值起点？
- 双轴是否有明确必要性和单位说明？
- 是否建立了独立的 `yAxisLabelRail`、`plotArea`、`xAxisLabelBand`，且距离满足 `50 / 10 / 28` 规格？
