# 图例 Legend

## 1. 文档定位

本文定义图表图例的类型、构成、布局、设计原则和交互规则。

## 2. 定义

图例是图表中的辅助标记，用于解释图形元素含义。

图例通过颜色、形状、大小、线型或渐变范围区分不同的数据分组，帮助用户理解图表中的数据类别、数据系列或连续数值范围。

图例应让图表更易解读，确保数据信息准确传达。图例本身不是数据主体，不应抢占图表阅读焦点。

## 3. 应用场景

当图表中使用多种颜色、图案、形状、线型或尺寸来代表不同数据类别、数据系列或数值范围时，应使用图例。

需要图例的场景：

- 多系列柱状图、折线图、面积图。
- 饼图、环图、玫瑰图中的多个分类。
- 散点图中颜色、形状或大小编码了不同分组。
- 热力图、地图等使用连续色阶表达数值范围。
- 气泡图中尺寸表达数值大小。
- 图表允许通过图例进行筛选、突出或分页时。

可以省略图例的场景：

- 单系列图表且标题或轴标题已经说明含义。
- 数据标签已经完整表达分类与数值。
- 图表中只有一个明确对象，图例不会增加理解。

## 4. 图例构成

图例根据关联数据类型分为离散图例和连续图例。

### 4.1 离散图例

离散图例用于解释不同类别、不同系列或不同标记形态所代表的数据含义。

构成：

| 构成 | 英文 | 作用 |
|---|---|---|
| 图例项 | item | 表示一个数据类别或数据系列 |
| 图形 shape | shape | 与图表中的颜色、形状或线型对应 |
| 标签 label | label | 图例项文本，表示类别或系列名称 |
| 值 value | value | 可选，用于展示该类别对应的数值或占比 |
| 分页器 | page | 图例项过多时用于翻页浏览 |

样式规则：

- 图例文字固定使用 `.textSmall` `12/18` `400`。
- 图例文字颜色绑定 `图例/legendTextColor`。
- 图形标记必须与图表中的对应数据系列严格一致。
- 分类色图例应绑定 `分类色板/item-*`。
- 语义状态图例应绑定 `语义色板/*`。
- 线图图例的 shape 应体现线型，不应全部使用色块。

推荐尺寸：

| 元素 | 建议尺寸 |
|---|---:|
| 色块 / 点状 shape | `10 × 10` |
| 线型 shape | `10 × 2` |
| 图例文字行高 | `18` |
| 图例项高度 | `18` |
| shape 与 label 间距 | `5` |

#### 默认 shape 字典

默认情况下，离散图例只使用 `rectangle`、`line`、`dot` 三种 shape，用于映射数据图形的面、线、点三类形态。除非组件规范或业务图表有明确扩展，不新增三角形、菱形、星形、描边圆、图标等装饰性图例。

| shape | 尺寸 | 形态 | 样式约束 | 对应图形语义 |
|---|---:|---|---|---|
| `rectangle` | `10 × 10` | 圆角矩形 | 圆角 `2px`，填充对应系列色，不加描边和阴影 | 面、柱体、面积、饼图扇区、热力色块 |
| `line` | `10 × 2` | 水平线段 | 高度 `2px`，圆角 `2px`，填充对应系列色；虚线系列保持虚线语义 | 线、趋势、路径、目标线、预测线 |
| `dot` | `10 × 10` | 圆形 | 圆形填充对应系列色，不加描边和阴影 | 点、散点、气泡点、事件点 |

图表映射：

| 图表 / 系列形态 | 默认 legend shape |
|---|---|
| 折线图、曲线图、趋势线、预测线 | `line` |
| 柱状图、条形图、堆叠柱、瀑布图 | `rectangle` |
| 面积图、堆叠面积图 | `rectangle`；如同时强调边界线，可在图表说明中补充线语义 |
| 饼图、环图、玫瑰图 | `rectangle` |
| 散点图 | `dot` |
| 气泡图 | `dot`；尺寸编码另用连续尺寸图例说明 |
| 雷达图 | 仅线框时用 `line`，填充面时用 `rectangle` |
| 热力图、地图离散分组 | `rectangle` |
| 组合图 | 每个系列按自身图形语义分别选择，例如柱为 `rectangle`，线为 `line` |

状态规则：

- 禁用或未选中图例项可使用 `text/disable` 与对应弱化系列色，但 shape 尺寸和形态不变。
- 图例 shape 与 Tooltip icon 必须保持同一语义；折线图图例是 `line`，Tooltip item 也应是 `line`。
- 图例形状不用于表达交互状态；交互状态通过颜色、透明度或 aiming / focus 标记表达。

### 4.2 连续图例

连续图例用于解释连续数值范围，通常和顺序色、发散色、尺寸编码相关。

构成：

| 构成 | 英文 | 作用 |
|---|---|---|
| 滑块标记 | mark | 表示当前查看的数据值 |
| 滑块轨道 | rail | 表示完整连续范围 |
| 选中轨道 | track | 表示当前选择的数据范围 |
| 手柄标签 | handlerLabel | 表示连续区间的最小值和最大值 |
| 手柄 | handles | 用于选择数据区间范围 |

适用：

- 热力图色阶。
- 地图数值范围。
- 气泡图尺寸范围。
- 连续筛选器。
- 发散色或顺序色图例。

使用规则：

- 连续图例必须显示最小值和最大值。
- 当存在当前选中值时，应显示 mark。
- 当支持范围筛选时，应显示 handles。
- 顺序色图例应从低到高排列。
- 发散色图例必须说明中心值和两端含义。
- 连续图例不应用来表达无序分类。

## 5. 图例类型

### 5.1 离散图例类型

离散图例根据视觉编码方式分为颜色图例和形状图例。

| 类型 | 用途 | 规则 |
|---|---|---|
| 颜色图例 | 用颜色区分不同类别或系列 | shape 颜色必须与数据图形一致 |
| 形状图例 | 用点形、线型、虚线等区分系列 | shape 形态必须与数据图形一致 |

使用规则：

- 分类数据优先使用颜色图例。
- 当颜色不足以区分，或存在打印 / 无障碍要求时，可叠加形状图例。
- 多系列折线图应使用线型或点线组合，避免只用色块。
- 图例 label 必须使用业务名称，不使用字段名或技术名。

#### 折线图线型图例规格

折线图图例必须表达“线”这一图形语义，而不是只用圆点或色块表示系列。

| 元素             | 规则                             |
| -------------- | ------------------------------ |
| shape 容器       | `10 × 2px` 或等价尺寸               |
| shape 形态       | 水平线段，圆角 `2px`                  |
| 实测线            | 实线，绑定对应 `分类色板/item-*`          |
| 预测 / 目标 / 非实测线 | 虚线，dash 样式与图中线型一致              |
| shape 与文字间距    | `5px`                          |
| 图例项高度          | `18px`                         |
| 图例位置           | 单图标准折线图优先右上角，水平排列              |
| 单系列图例          | 如果标题已说明系列，可省略；若用于组件验收，应保留以验证样式 |

Figma 标准折线图中，图例项为右上角水平布局：`10px` 线型 shape + `.textSmall` 文案。AI 生成 Relay / Figma 验证稿时，不应使用带大圆点的图例作为默认样式。

#### 图例 shape 选择优先级

当图表组件文档、图形语义和默认 shape 字典出现冲突时，按以下顺序判断：

```txt
目标组件实例属性
→ 图表系列真实形态
→ 默认 shape 字典
→ 图表类型常规映射
```

示例：组合图中“成交金额”为柱体时使用 `rectangle`，“成交转化率”为折线时使用 `line`；不得因为二者同在一个图例列表中统一改成圆点或色块。

### 5.2 连续图例类型

连续图例根据视觉编码方式分为颜色图例和尺寸图例。

| 类型 | 用途 | 规则 |
|---|---|---|
| 连续颜色图例 | 表示数值由低到高、弱到强 | 绑定顺序色板或发散色板 |
| 连续尺寸图例 | 表示数值大小、权重或规模 | 尺寸变化必须与数据值单调对应 |

使用规则：

- 连续颜色图例必须配合 `有序色板/*` 或 `发散色板/*`。
- 尺寸图例必须显示最小值和最大值。
- 当数值范围存在业务阈值，应在图例中标注阈值或中心值。

## 6. 布局

图例分为水平布局和垂直布局，两种布局都支持多个位置。

布局原则：

- 图例应靠近图表，但不遮挡数据图形。
- 图例应放在用户自然阅读路径中。
- 图例占用空间应与图表复杂度匹配。
- 图例项较多时，应优先分页、折叠或筛选。
- 笛卡尔坐标系图表的离散图例默认位于图表画布右上角，水平排列，并与图表内容区右侧对齐。
- 水平离散图例的系列项默认间距为 `20px`；每个系列项内部仍按 shape 与 label 间距 `5px` 执行。
- 图例只承载数据系列或合法图例项，不添加轴说明、读图说明、图表类型说明等非系列文本。

### 6.1 水平布局

适用：

- 图例项较少。
- 图表上方或下方有稳定空间。
- 多系列趋势图、柱状图、组合图。

推荐：

- 放在图表顶部或底部。
- 图例项从左到右排列。
- 同一行图例项数量不宜过多。
- 当图例项过多时，启用分页或换行。

### 6.2 垂直布局

适用：

- 图例项较多。
- 标签文本较长。
- 饼图、环图、地图或需要右侧解释的图表。

推荐：

- 放在图表右侧或左侧。
- 图例项从上到下排列。
- 若图例高度超过图表高度，应启用滚动、分页或折叠。

### 6.3 位置规则

Figma 规范中图例支持水平和垂直两类布局，并可放置在图表周围多个位置。

AI 生成图表时应优先选择：

| 场景 | 推荐位置 |
|---|---|
| 折线图 / 柱状图，图例项少 | 顶部或底部 |
| 饼图 / 环图 | 右侧 |
| 图例项多且文本较长 | 右侧垂直布局 |
| 连续色阶图例 | 图表底部或右侧 |
| 移动端窄容器 | 底部，必要时分页 |

补充规则：

- 标准分析卡片中，折线图图例优先放在 plot area 上方右侧，并与图表右边界对齐。
- 图例与顶部边界建议保持 `0-8px` 的紧凑距离，避免压缩主绘图区。
- 多图并列时，图例位置应在同类图表间保持一致。
- 图例不应放在 Tooltip 常驻出现的位置上。

## 7. 设计原则

图例设计应确保清晰、简洁、准确，并与整体图表风格协调统一。

### 7.1 清晰

图例中的符号、颜色、形状必须与图表中的数据元素清晰对应。

规则：

- 图例 shape 与数据图形必须一一对应。
- 图例 label 必须说明数据含义。
- 图例不应产生歧义或误导。
- 用户应能通过图例正确理解图表表达的信息。

### 7.2 简洁

图例应保持简洁，避免添加不必要的视觉装饰和过多图例项。

规则：

- 图例项较多时，应合并同类项、分页、折叠或筛选。
- 不要让图例占用过多图表空间。
- 图例不承载解释性长句。
- 图例不应重复图表标题已经表达清楚的信息。

### 7.3 准确

图例所呈现的颜色、形状等视觉标记必须与图表中的数据系列严格一致。

规则：

- 标签文字必须准确描述所代表的数据含义。
- 图例顺序应与图表系列顺序、堆叠顺序或视觉阅读顺序一致。
- 任意偏差都可能导致用户误读数据。
- 图例值 value 如果显示，应与图表数据保持同一单位和格式。

### 7.4 协调统一

图例视觉风格应与整体图表及所在页面保持协调统一。

规则：

- 图例字体、颜色、间距应使用系统 token。
- 图例 shape 大小、圆角、间距应保持一致。
- 图例不应使用额外阴影、描边或装饰造成视觉割裂。
- 图例应服从整体布局，不应压缩主图形阅读区域。

## 8. 交互原则

图例允许交互，在不同交互状态下应有明确反馈。

### 8.1 可悬停

当鼠标悬停在图例项上时，对应图表元素应同步高亮。

反馈效果：

- 改变对应图例项和图表元素的颜色、透明度或强调状态。
- 帮助用户快速识别图例与图表中数据元素的对应关系。
- 不悬停的系列可适当降低视觉优先级。

使用规则：

- hover 不改变数据含义。
- hover 状态应可逆。
- hover 强调不能造成其他系列完全不可见。

### 8.2 可高亮

点击或通过其他方式选中图例项后，对应图表元素高亮显示，其他元素降低优先级。

适用：

- 多系列趋势图。
- 分组柱状图。
- 饼图 / 环图分类。
- 需要聚焦某个系列的分析场景。

反馈效果：

- 与 hover 类似，但状态可保持。
- 选中系列颜色保持正常或增强。
- 未选中系列降低透明度。

### 8.3 可翻页

当图例项过多，超出显示区域时，应提供翻页功能。

使用规则：

- 所有图例项都必须可访问。
- 不得直接截断图例项。
- 翻页状态应显示当前页和总页数，例如 `1 / 2`。
- 翻页样式应保持轻量，不抢占图表主体。

### 8.4 可筛选

图例可作为筛选控件，用于控制图表中对应数据系列的显示与隐藏。

反馈效果：

- 点击图例项后，图表仅显示对应数据或隐藏对应数据。
- 再次点击可恢复原始视图。
- 被隐藏系列的图例项应呈现禁用或弱化状态。

使用规则：

- 筛选状态必须可见。
- 不得让用户误以为数据缺失。
- 当只剩一个系列时，应仍保持图表可读。

## 9. 与主题变量的关系

图例必须绑定主题变量，不得写死颜色。

| 元素         | 绑定变量                 | Light     | Dark      |
| ---------- | -------------------- | --------- | --------- |
| 图例文字       | `图例/legendTextColor` | `#595959` | `#FFFFFF` |
| 分类图例 shape | `分类色板/item-*`        | 读取当前主题解析值 | 读取当前主题解析值 |
| 语义图例 shape | `语义色板/*`             | 读取当前主题解析值 | 读取当前主题解析值 |
| 连续颜色图例     | `有序色板/*` 或 `发散色板/*`  | 读取当前主题解析值 | 读取当前主题解析值 |

样式规则：

- 图例文字固定使用 `.textSmall` `12/18` `400`。
- 图例文字绑定 `图例/legendTextColor`。
- 图例 shape 绑定对应数据系列的颜色 token。
- Light / Dark 只改变变量解析值，不改变图例结构。

## 11. AI 输出格式

AI 生成或审核图表时，应输出图例决策。

推荐字段：

| 字段 | 示例 |
|---|---|
| 是否需要图例 | `true` |
| 图例类型 | `离散图例` |
| 编码方式 | `颜色图例` |
| 图例位置 | `top` |
| 图例方向 | `horizontal` |
| 图例文字样式 | `.textSmall 12/18 400` |
| 图例文字颜色 | `图例/legendTextColor` |
| 图例 shape token | `分类色板/item-1` |
| 图例 shape type | `rectangle / line / dot` |
| 图例 shape size | `10×10 / 10×2 / 10×10` |
| 是否显示 value | `false` |
| 是否分页 | `分类超过 8 个时启用` |
| 是否可筛选 | `true` |

## 12. 禁止项

- 禁止图例颜色、形状与图表数据系列不一致。
- 禁止把所有图表默认画成圆点图例。
- 禁止折线图使用 `rectangle` 或 `dot` 代替 `line`，除非目标组件实例明确如此。
- 禁止柱状图、饼图、面积图使用 `line` 代替 `rectangle`。
- 禁止散点图使用 `rectangle` 代替 `dot`。
- 禁止图例 label 使用字段名、技术名或无法理解的缩写。
- 禁止图例项过多时直接截断。
- 禁止图例抢占图表主体空间。
- 禁止用图例承载解释性长句。
- 禁止离散图例表达连续数值范围。
- 禁止连续图例表达无序分类。
- 禁止筛选状态不可见。
- 禁止写死图例文字颜色或系列颜色。
- 禁止在图例文字中使用非 `.textSmall 12/18 400` 的样式。

## 13. AI 自查清单

生成或审核图表时检查：

- 图表是否真的需要图例？
- 图例类型是离散还是连续？
- 图例 shape 是否与图表数据系列严格一致？
- 图例 label 是否是业务可读名称？
- 图例文字是否使用 `.textSmall 12/18 400`？
- 图例文字是否绑定 `图例/legendTextColor`？
- 图例颜色是否绑定对应主题变量？
- 图例顺序是否与图表系列顺序一致？
- 图例项过多时是否分页、折叠或筛选？
