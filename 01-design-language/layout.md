# 间距布局 Layout

## 1. 文档定位

本文定义图表容器、标题区、图例区、绘图区、坐标轴、标签、Tooltip 和标注之间的布局关系。

本文不定义具体图表的数据图形绘制方式。柱、线、点、面积、节点、边等图形主体规则应读取 `02-chart-type/` 下对应图表文档。

## 2. 布局目标

图表布局需要同时满足三类目标：

| 目标 | 要求 |
|---|---|
| 可读 | 标题、图例、轴标签、数据主体和 Tooltip 不重叠 |
| 稳定 | 数据、筛选、hover、图例变化时布局不明显跳动 |
| 可维护 | VibeDesign 中能通过 Frame / Auto Layout / 命名层级继续调整 |

## 3. 标准结构

推荐图表结构：

```txt
chartFrame
  headerFrame
    title
    subtitle / optional
    legendFrame / optional
  bodyFrame
    yAxisTitleBand / optional
    plotRowFrame
      yAxisLabelRail / optional
      plotAreaFrame
      rightAxisLabelRail / optional
    xAxisLabelBand / optional
    xAxisTitleBand / optional
  footerFrame / optional
```

规则：

- `chartFrame` 是唯一根容器。
- `headerFrame` 只放标题、说明和图例，不放数据系列。
- `plotAreaFrame` 是唯一数据坐标空间。
- 轴标签轨道、绘图区和轴标题带应作为明确区域，而不是散落文本。
- Tooltip 浮层可覆盖图表，但不得遮挡当前 hover 点和主要读数。

## 4. 区域职责

| 区域 | 职责 | 约束 |
|---|---|---|
| 标题区 | 说明图表主题、口径、时间范围 | 不得与图例、Tooltip 重叠 |
| 图例区 | 说明颜色、形状或线型对应的系列 | 多行图例不得压缩绘图区到不可读 |
| 绘图区 | 承载数据主体、网格线、标注和交互反馈 | 必须有稳定宽高和边界 |
| Y 轴标签轨道 | 承载数值刻度、单位或轴标题 | 宽度由最长标签决定 |
| X 轴标签带 | 承载类目或时间标签 | 标签过密时抽样、旋转或换图 |
| 标注层 | 承载关键点、参考线、区间和 callout | 不得遮挡主数据和 Tooltip |
| 页脚区 | 放数据来源、口径补充或说明 | 不参与主要读数 |

## 5. 尺寸与间距原则

本文不强制所有画布使用固定尺寸，但要求布局关系稳定。

通用规则：

- 标题区、图例区、绘图区、坐标轴区应有明确分区。
- 图表主体应优先获得足够空间，不得为了说明文字牺牲读数。
- 坐标轴标签与绘图区之间必须保留可读间距。
- 标题、图例和 Tooltip 的文字不得压到容器边界。
- 标注距离数据点通常应保持 `8-12px`，具体规则读取 `label.md`。
- 网格线、参考线和区间必须限制在 `plotAreaFrame` 内。

## 6. 图例布局

图例布局读取 `legend.md`，本文件只定义区域关系。

规则：

- 单行图例优先放在标题区右侧或标题下方。
- 图例项过多时，应换行、折叠、筛选或改用交互，不得压缩绘图区。
- 图例顺序必须与系列顺序一致。
- 图例宽度变化时，标题区和绘图区不得发生不可预期偏移。

## 7. 坐标轴布局

坐标轴布局读取 `axis.md`，本文件只定义空间规则。

规则：

- Y 轴标签轨道宽度应按最长标签预留，不得覆盖绘图区。
- X 轴标签过密时，优先抽样显示；仍不可读时考虑横向条形图、缩放或筛选。
- 双轴图必须给左轴、右轴分别预留轨道，且系列与轴的对应关系清楚。
- 轴标题应有独立区域或与轴标签保持稳定间距。

## 8. 响应式与可维护

当图表宽高变化时：

- 标题和说明优先保持完整，可换行但不得遮挡图表主体。
- 绘图区宽高应按剩余空间重新计算。
- 轴标签可抽样或隐藏部分普通标签，但必须保留读数路径。
- 图例可换行、折叠或移动到下方。
- Tooltip 应自动避让容器边界。

VibeDesign 中最低要求：

- 标题、图例、坐标轴、绘图区和页脚为独立 Frame 或可识别分组。
- 数据图形归属 `seriesLayer`。
- 标注归属 `annotationLayer`。
- 交互反馈归属 `interactionLayer`。

## 9. 常见布局反模式

| 反模式 | 问题 | 替代方案 |
|---|---|---|
| 所有元素平铺在根画布 | 设计师无法维护 | 使用结构化 Frame |
| 图例挤压绘图区 | 数据主体不可读 | 折叠图例或减少系列 |
| X 轴标签密集堆叠 | 用户无法读取类目 | 抽样、缩放、筛选或换条形图 |
| Tooltip 遮挡 hover 点 | 当前读数被挡住 | Tooltip 自动避让 |
| 标注覆盖主趋势线 | 重点干扰数据 | 外移标注或进入 Tooltip |
| 轴标题覆盖刻度 | 口径和数值冲突 | 使用独立 title band |

## 10. AI 自查清单

- 是否建立了 `chartFrame / headerFrame / bodyFrame / plotAreaFrame`？
- 标题、图例、绘图区、轴标签区是否有明确空间关系？
- 绘图区是否没有被说明文字、图例或标签压缩到不可读？
- X / Y 轴标签是否没有重叠和溢出？
- Tooltip 和标注是否不遮挡主要读数？
- 响应式变化时是否有抽样、折叠或换图策略？
- VibeDesign 输出是否保留可编辑结构？
