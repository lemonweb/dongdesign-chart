# File Path Finder Skill

## 1. 文档定位

本 Skill 用于在当前 Wiki 中根据中文名、英文名、业务关键词、图表类型或工具名定位文件路径。

它不负责改写文档内容，只负责帮助 AI Agent 快速找到应该读取或返回给用户的文件路径。

## 2. 触发场景

当用户出现以下意图时触发：

- “帮我找一下色彩系统在哪里”
- “折线图规范文件路径是什么”
- “查一下 VibeDesign 规则”
- “我想找营销活动分析相关文档”
- “用中文关键词帮我定位文件”
- “返回对应文件路径”
- “哪个文件讲 ECharts / G2 / tooltip / 图例 / 坐标轴”

## 3. 必读文件

1. `ai-file-index.md`
2. `00-overview/ai-reading-flow.md`

如涉及具体任务，再按索引命中的目录读取对应 README。

## 4. 执行流程

1. 提取用户输入中的中文关键词、英文关键词、工具名、图表名和业务名。
2. 在 `ai-file-index.md` 中匹配“中文名 / 英文名 / 关键词”列。
3. 如果只命中一个文件，直接返回路径。
4. 如果命中多个文件，按以下优先级排序：
   - 具体主题文件优先于 README。
   - 图表类型文件优先于选型总则。
   - Skill 文件优先于注册表。
   - 文档治理问题优先返回 `07-document-governance/` 下的文件。
5. 如果没有命中，使用 `rg --files` 搜索文件名，再使用 `rg` 搜索正文关键词。
6. 返回路径时使用反引号包裹，并简短说明为什么匹配。

## 5. 输出格式

命中单个文件：

```text
你要找的是：`相对路径`
匹配原因：关键词“xxx”对应索引中的“中文名 / 关键词”。
```

命中多个文件：

```text
可能相关的文件有：

1. `相对路径` - 用于 xxx
2. `相对路径` - 用于 xxx
```

## 6. 命名处理原则

- 不因为路径包含中文或空格就自行改名。
- 需要新增文件时，优先使用英文小写 kebab-case。
- 中文名、中文别名和业务描述写入 `ai-file-index.md`，不要写入新增机器路径。
- 如果用户明确要求批量改名，必须先生成迁移表，再更新全部 Markdown 链接和 Skill 引用。

## 7. 不负责事项

- 不直接修改图表规范。
- 不直接生成代码。
- 不批量重命名文件。
- 不替代 `wiki-maintainer-skill`、`doc-scaffold-skill` 或 `dependency-checker-skill`。

## 8. 自查清单

- 是否先查了 `ai-file-index.md`？
- 是否返回了精确路径？
- 是否避免了自行猜测不存在的路径？
- 是否在多命中时说明了每个路径的用途？
- 是否保留了当前仓库的既有命名，不做隐式改名？
