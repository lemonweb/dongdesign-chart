# Visual Style Rules

## 1. 视觉定位

面向经营决策报告，风格应清晰、克制、数据可信。避免营销海报化、过度装饰和低信息密度。

默认主题：

```txt
business_diagnosis_light
```

## 2. 色彩规则

| 语义 | 建议 |
|---|---|
| 正向变化 | 稳定绿色，低饱和 |
| 负向变化 | 风险红 / 橙红，避免大面积刺眼红 |
| 中性信息 | 灰蓝 / 中性灰 |
| 重点高亮 | 品牌主色或深蓝，不超过页面 10% 面积 |
| 背景 | 浅灰或白色，确保卡片与图表可读 |

不得只用红绿表达差异；需要配合箭头、文字、标签或图形编码。

## 3. 字体层级

| 层级 | 用途 |
|---|---|
| Display | 报告主标题，不用于卡片内部 |
| H1/H2 | 一级区块标题 |
| H3 | 卡片标题 |
| Body | 解释文案 |
| Caption | 数据来源、单位、风险提示 |
| Metric | KPI 数值 |

指标数值应突出，变化率和单位弱一级。

## 4. KPI 卡规则

- 每张卡只表达一个核心指标。
- 必须显示指标名、数值、变化、状态。
- 有口径或周期时放在 caption。
- 负向指标必须可扫视识别，但不得制造恐慌。

## 5. 图表样式规则

- 图表标题回答业务问题，不只写指标名。
- 坐标轴单位必须明确。
- 趋势图必须标注异常点或关键日期。
- 排名条形图按业务重要性排序。
- 图例数量控制在 5 个以内。
- Tooltip/标注中的数值格式与 KPI 卡一致。

## 6. 卡片与密度

- 使用 8px 或更小圆角，保持工作台气质。
- 区块之间保持稳定间距，不堆叠浮层卡片。
- 首屏信息密度要高，但不得让图表和结论文案互相抢焦点。
- 长报告可分段，不强行一屏塞满。

## 7. 风险提示样式

风险提示必须可见，但不应压过主叙事：

| 风险等级 | 样式 |
|---|---|
| blocker | 顶部明显提示，阻止继续渲染正式结论 |
| risk | 区块内风险卡 |
| warning | caption 或侧栏提示 |
| info | QA 附录 |

## 8. 输出一致性

- 相同指标在全文使用同一单位和格式。
- 同一 polarity 使用同一状态颜色。
- 同一时间周期在所有图表中保持一致。
- 不在视觉层改写数据事实。
