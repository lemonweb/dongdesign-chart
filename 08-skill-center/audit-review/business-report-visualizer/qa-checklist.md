# QA Checklist

## 1. 数据一致性

- [ ] Report IR 中的指标值与原 Markdown 一致。
- [ ] 单位、周期、同比/环比口径没有混用。
- [ ] OCR 或低置信字段已标注。
- [ ] 冲突日期、冲突数值已写入 `source_trace.conflicts`。
- [ ] 没有新增原文不存在的数据事实。

## 2. 证据链

- [ ] 每条核心结论都有 `evidence_refs`。
- [ ] 每条行动建议至少绑定一条证据。
- [ ] 归因结论没有超出原文数据支持。
- [ ] 无证据推断已标注 `inferred` 或进入风险项。

## 3. 叙事质量

- [ ] 报告回答“发生了什么、问题多严重、原因在哪里、下一步做什么”。
- [ ] 首屏出现开场结论、风险等级和核心指标。
- [ ] 原始章节顺序已按诊断叙事重组。
- [ ] 行动建议按优先级排序。

## 4. 图表选型

- [ ] 每个图表都声明 `question`。
- [ ] 图表类型匹配数据关系。
- [ ] 表格没有被默认照搬，除非满足保留条件。
- [ ] 高风险图表已标注误读风险。
- [ ] 复杂图表已准备传给 `chart-selector-skill` 审查。

## 5. 布局与视觉

- [ ] Bento 布局按优先级排序。
- [ ] 明细表没有占据首屏核心区域。
- [ ] 移动端可按单列阅读。
- [ ] 颜色、状态、单位和数值格式一致。
- [ ] 风险提示可见。

## 6. 输出可用性

- [ ] Visual JSON 组件都有稳定 `id`。
- [ ] 组件 props 可被渲染器消费。
- [ ] 图表数据字段与 encoding 对齐。
- [ ] HTML / React / Figma handoff 没有遗漏关键资源。
- [ ] 输出包含 QA 结果与风险缺口。

## 7. 阻断条件

出现以下任一情况时，不输出正式可视化报告，只输出修复清单：

- 缺少原始报告正文。
- 核心指标无法识别且用户要求经营结论。
- 关键数据冲突无法判断且会影响主结论。
- 用户要求生成图表，但没有可用于图表的数据关系。
- OCR 置信度过低且没有人工确认。
