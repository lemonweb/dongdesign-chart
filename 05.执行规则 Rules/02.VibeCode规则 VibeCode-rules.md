# VibeCode 规则 VibeCode Rules

## 1. 文档定位

本文约束 AI 使用 G2、ECharts、SVG、Canvas 或 DOM 生成可交互图表代码时的执行行为。

VibeCode 的目标不是“代码能运行”即可，而是：

```txt
业务表达正确
→ 图表选型正确
→ 视觉语言还原稳定
→ 交互行为符合组件规范
→ 多引擎实现语义一致
```

本文不重新定义颜色、字体、坐标轴、图例和 Tooltip 规范。生成代码前必须读取 `01.视觉语言 Design Language/` 与目标图表文档。

## 2. 触发场景

以下任务必须读取本文：

- 生成 G2 / ECharts 图表代码。
- 生成可交互 HTML / SVG / Canvas 图表原型。
- 对比 VibeDesign 与 VibeCode 输出质量。
- 在不同图表引擎之间切换实现。
- 修复图表代码中的视觉还原问题。

## 3. 执行顺序

VibeCode 必须按以下顺序执行：

```txt
1. 读取 chart-selector-skill 输出或重新完成图表选型
2. 读取 chart-builder-skill 输出的字段映射和组件构成
3. 读取目标图表组件文档
4. 读取相关视觉语言文档
5. 读取目标引擎适配文档
6. 生成 Visual Fidelity Packet
7. 生成 Engine Mapping Packet
8. 编码实现
9. 执行代码自查清单
10. 通过浏览器或截图做视觉验证
```

不得跳过视觉语言文档直接写引擎配置。

## 4. Visual Fidelity Packet

每个 VibeCode 输出必须在实现前形成以下约束包。若缺失任一项，不得进入最终交付。

| 字段 | 必填 | 说明 |
|---|---|---|
| `typography_constraints` | 是 | 轴标签、图例、Tooltip、标签的字号、行高、字重 |
| `axis_constraints` | 是 | 轴线、数值轴起点、网格线数量、标签抽样、轴标题位置和重叠规避 |
| `legend_shape_dict` | 是 | 图例 shape 类型、尺寸、圆角、颜色映射 |
| `tooltip_constraints` | 是 | hover 指示线、浮层、文本、图标、阴影 |
| `series_constraints` | 是 | 线宽、点位默认显示策略、hover 强调策略 |
| `token_bindings` | 是 | 每个图表元素绑定的 Wiki token 与当前显示值 |
| `engine_specific_risks` | 是 | G2 / ECharts / SVG 的差异和规避策略 |

## 5. 样式还原硬性规则

### 5.1 字体

- 图表内部只允许使用 Wiki 中定义的 `12 / 14 / 16` 字号。
- 坐标轴标签、图例文字、Tooltip 正文必须显式设置为 `12px / 18px / 400`。
- Tooltip 标题必须显式设置为 `12px / 18px / 600`。
- Tooltip value 必须显式设置为 `12px / 18px / 600`；label 和 note 保持 `12px / 18px / 400`。
- 不得依赖 G2 / ECharts / 浏览器默认字体大小。

### 5.2 坐标轴

- 轴标签不得与数据线、数据点、标题、图例或 Tooltip 重叠。
- 时间轴标签必须抽样显示，优先保留首尾和等步长标签。
- 柱状图、折线图等笛卡尔坐标系图表，数值轴默认 `min = 0`，不得默认使用数据最小值作为起点。
- Y 轴默认 `splitNumber = 4`，即 `5` 个刻度 / 网格线，并保持线性等距；不得输出密集刻度或类似对数轴的非均匀间隔。
- 数值轴上界必须由统一算法计算：`targetMax = dataMax × 1.08`，`axisMax = step × 4`；当 `axisMax / dataMax > 1.25` 时，必须使用按展示精度向上取整的 `precisionStep`，避免顶部留白过大压扁趋势。
- 笛卡尔坐标系的网格线 / 分隔线默认使用 `1px` 虚线，建议 dash `4 / 4`；坐标轴轴线和 hover 指示线按规范使用实线。
- Y 轴标题默认放在 Y 轴标签上方的左上角说明区；若使用左侧竖排标题，必须有独立 title rail，不得与轴标签同列挤压，也不得放入绘图区、分隔线或数据线所在区域。
- Y 轴标题位于标签上方时，必须为标题预留独立 title band；标题底部与首个 Y 轴标签至少保持 `12px` 间距。
- 折线图中，Y 轴默认不显示左侧实线；仅保留标签和水平网格线。
- 折线图 hover 指示线必须为垂直 `1px` 实线，绑定 `提示/tooltipAxisColor`；默认不显示横向 crosshair。
- 多分面共享时间轴时，优先只在最下方 plot 显示 X 轴标签。

### 5.3 图例

- 折线图图例 shape 必须使用 `line`：`10 × 2px`、圆角 `2px`、实测线为实线。
- 不得用大圆点、色块或图标替代折线图默认 line shape。
- 图例 shape 与 Tooltip icon 必须保持同一图形语义。
- 图例交互只能改变高亮、弱化或筛选状态，不得改变数据含义。

### 5.4 数据点与 MarkPoint

- 折线图默认不显示每个数据点的 marker。
- 数据点只在 hover、选中、最新值、最大值、最小值、异常点等必要场景显示。
- 不得为了“看起来完整”给所有 30 个时间点都显示 markPoint。
- ECharts 的 `showSymbol`、G2 的 `point` mark、SVG 的 circle 节点都必须按上述规则控制。

### 5.5 Tooltip

- Tooltip 必须包含时间 / 类目、系列名、数值和单位。
- 多系列 Tooltip 的顺序必须与图例顺序一致。
- Tooltip icon 必须与图例 shape 语义一致；折线图使用 line icon。
- 多系列 Tooltip 必须使用 `label_column + value_column` 两列结构，系列行高 `18px`、行间距 `0px`，note 距系列列表 `4px`。
- Tooltip title 与 value 必须加粗；label 与 note 不加粗。
- Tooltip 不得遮挡当前 hover 点和主要读数区域。

## 6. 多引擎一致性

当同一页面支持多个引擎时，必须保证：

- 图表类型一致。
- 字段映射一致。
- 图例顺序、shape 和颜色一致。
- Tooltip 内容、数值格式和单位一致。
- 坐标轴标签抽样策略一致。
- hover 指示线样式一致。
- 默认数据点显示策略一致。

如果某个引擎无法稳定还原某项样式，必须：

1. 在代码中增加补偿实现。
2. 在输出中标记该引擎的风险。
3. 不得静默降级为不符合规范的样式。

## 7. 交付前验证

VibeCode 交付前必须完成：

- 静态检查：确认没有使用未定义颜色、字号、线宽和装饰。
- 运行检查：页面可打开，图表非空白。
- 交互检查：hover、图例筛选、范围切换或缩放行为可用。
- 视觉检查：截图中无文字重叠、无图例错误、无轴标题遮挡。
- 多引擎检查：ECharts / G2 切换后关键视觉语义一致。

## 8. 禁止项

- 禁止只验证代码语法，不验证视觉截图。
- 禁止在 Wiki 已有明确规范时，把 G2 / ECharts 默认样式当作设计规范覆盖使用。
- 禁止在 Wiki 缺少明确约束时自行发挥；应使用 G2 / ECharts 常规布局样式作为兜底，并在输出中标记 `fallback_reference`。
- 禁止在未读取图例规范时生成 legend。
- 禁止在未读取坐标轴规范时生成 axis。
- 禁止默认显示所有折线数据点。
- 禁止无业务说明时让数值轴从数据最小值起步。
- 禁止普通线性轴出现非均匀或过密刻度。
- 禁止把轴标题移动到分隔线、数据线或 plot 内部来规避重叠。
- 禁止把笛卡尔坐标系网格 / 分隔线渲染为默认实线后声明还原完成。
- 禁止让轴标题覆盖首个刻度标签。
- 禁止默认显示横向 crosshair 或字段名 tooltip 后声明交互符合规范。
- 禁止 hover layer 覆盖标题区或画布外部，导致指示线溢出 plot。
- 禁止 hover 点使用与图表系列不同的坐标来源，导致 MarkPoint 偏移；hover 点必须来自引擎坐标转换、实际渲染 geometry，或与 series 同层同 scale 绘制。
- 禁止 hover 指示线样式与组件规范不一致。
- 禁止让轴标题、轴标签和数据图形重叠。
- 禁止多引擎输出语义不一致却声明验证通过。
