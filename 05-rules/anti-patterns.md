# 禁止项与反模式 Anti Patterns

## 1. 文档定位

本文沉淀 AI 生成图表设计、代码、评审和 Wiki 文档时的常见反模式。

本文用于快速阻断错误方向，不替代具体规则。发现问题后应回到对应规范文件修正。

## 2. 选型反模式

| 反模式 | 问题 | 正确处理 |
|---|---|---|
| 只按图表名称选图 | 忽略业务问题和数据结构 | 先读 `02-chart-type/capability-matrix.md` 和 `selection-rules.md` |
| 用复杂图表表达简单问题 | 增加理解成本 | 优先使用 L1 / L2 图表 |
| 用户指定图表就直接执行 | 可能与数据结构不匹配 | 先做适配性判断，必要时说明风险 |
| 用饼图表达趋势 | 饼图不表达时间变化 | 改用折线图、面积图或柱状图 |
| 用双轴掩盖单位差异 | 容易制造虚假相关 | 拆图或明确轴对应关系 |
| 把相关性写成因果 | 缺少归因证据 | 只写相关或需要进一步验证 |

## 3. 数据反模式

| 反模式 | 问题 | 正确处理 |
|---|---|---|
| 缺字段仍定稿 | 图表成立条件不足 | 输出缺失字段和候选方案 |
| 编造目标、阈值或异常规则 | 误导业务判断 | 要求用户提供口径或标记 `needs_spec` |
| 截断坐标轴后夸大变化 | 改变视觉感知 | 明确标注截断或使用从 0 起步轴 |
| 单位混用 | 用户无法比较 | 拆分指标或补充轴标题 / Tooltip |
| 隐藏筛选条件 | 数据范围不透明 | 标题、副标题或 Tooltip 中说明 |
| 用截图读数当精确数据 | 精度不可控 | 标记近似或要求原始数据 |

## 4. 视觉反模式

| 反模式 | 问题 | 正确处理 |
|---|---|---|
| 随机配色 | 破坏主题和语义 | 读取 `color.md` 和 `theme-token.md` |
| 所有类别都用高饱和色 | 重点不清 | 只突出重点，其余使用分类色或弱化 |
| 所有点都显示标签 | 图表拥挤 | 只标关键点，完整值进 Tooltip |
| 用小于 `12px` 的文字塞内容 | 可读性下降 | 抽样、折叠、筛选或扩大布局 |
| 图例 shape 与图表类型不匹配 | 语义错位 | 读取 `legend.md` |
| Tooltip 使用字段名直接交付 | 用户看不懂 | 使用业务名称、单位和格式化值 |
| hover 指示线溢出标题区 | 破坏空间语义 | 限制在 `plotAreaFrame` 内 |

## 5. VibeDesign 反模式

| 反模式 | 问题 | 正确处理 |
|---|---|---|
| 把整个图表做成单张图片 | 不可编辑 | 使用 Frame / Text / Shape / Component 结构 |
| 所有元素平铺在根画布 | 无法维护 | 建立 `chartFrame / bodyFrame / plotAreaFrame` |
| 数据系列、坐标轴、Tooltip 混在一起 | 结构不清 | 使用 `seriesLayer / annotationLayer / tooltipLayer` |
| 用绝对坐标堆标题和图例 | 响应式差 | 使用 Auto Layout 或明确 Frame 层级 |
| Tooltip 做成散落文本 | 无法修改内容 | 使用两列结构和独立 Frame |

## 6. VibeCode 反模式

| 反模式 | 问题 | 正确处理 |
|---|---|---|
| 只生成静态样式不写字段映射 | 无法接入真实数据 | 输出 `x / y / color / size / tooltip` |
| 使用图表库默认主题 | 与 dongDesign 不一致 | 映射到 `04-adaptation/theme-schema.md` |
| 多引擎实现语义不一致 | G2 / ECharts 结果不统一 | 读取 `mapping-table.md` |
| 只做语法检查 | 可能视觉失败 | 做浏览器或截图验证 |
| 没有空状态 / 加载态 / 异常态 | 交付不完整 | 输出状态方案 |

## 7. 文档治理反模式

| 反模式 | 问题 | 正确处理 |
|---|---|---|
| 新增文档不登记索引 | AI 找不到路径 | 更新 `ai-file-index.md` 和 README |
| 复制大段外部原文 | 不可维护且风险高 | 转化为规则、表格和自查项 |
| 把推断写成正式规范 | 误导后续使用 | 标记 `inferred` 或 `needs_spec` |
| 单文件重复定义全局样式 | 规范冲突 | 引用 `01-design-language/` |
| 改路径不更新引用 | 链接失效 | 同步检查 README、Skill 和依赖地图 |

## 8. AI 阻断规则

出现以下情况时，不得输出“已完成 / 可交付”：

- 图表无法回答用户的主要业务问题。
- 必要字段缺失但未说明。
- 异常、风险、达标、原因等判断没有证据。
- 颜色、字号、轴、图例、Tooltip 未读取对应规范。
- VibeDesign 输出不可编辑或无结构。
- VibeCode 输出不可运行或无字段映射。
- 复杂图表没有解释必要性和误读风险。
