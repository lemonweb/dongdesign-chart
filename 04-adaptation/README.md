## 1. 目录定位

本目录用于说明如何将统一的数据可视化设计体系落地到具体图表引擎中。

当前建议优先支持 AntV G2，同时兼容 ECharts。

## 2. 使用时机

以下情况应读取本目录：

- 生成 G2 图表代码
- 生成 ECharts 图表代码
- 将设计 token 映射为图表主题
- 统一不同图表引擎的视觉表现
- 检查代码是否符合设计体系

## 3. 读取顺序

建议按以下顺序读取：

1. `theme-schema.md`
2. `g2-adapter.md`
3. `echarts-adapter.md`
4. `mapping-table.md`

## 4. 使用原则

- 设计标准应先来自 `01-design-language/`。
- 图表类型应先来自 `02-chart-type/`。
- 代码生成时不得绕过主题模型。
- 不同引擎实现可以不同，但视觉结果应保持一致。

## 5. 不在本目录处理的内容

- 图表为什么这样选：查阅 `02-chart-type/`
- 图表业务组合方式：查阅 `03-pattern/`
- AI 执行流程和禁止项：查阅 `05-rules/`
