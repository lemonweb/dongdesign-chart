# Agent 增强设计流程 Agent Workflow

## 1. 文档定位

本文定义 Agent 为图表、指标卡、表格或洞察卡设计增强表达时的执行流程。

## 2. 默认流程

```txt
1. 识别业务主题和用户问题
2. 抽取指标、维度、时间范围、对象和单位
3. 判断数据关系：变化 / 结构 / 联系
4. 判断数据状态：偏离 / 波动 / 异常 / 失衡 / 恶化
5. 确定叙事重点：个体 / 变化 / 总量 / 对比 / 时间 / 占比 / 异常 / 原因
6. 选择增强方式：标注 / 高亮 / 参考线 / 区间 / callout / 交互
7. 绑定数据证据和规范来源
8. 输出增强方案和风险标记
9. 按 QA 清单自查
```

## 3. 必须先回答的问题

| 问题 | 输出字段 |
|---|---|
| 用户想看什么关系 | `relationship_type` |
| 当前是否需要状态判断 | `state_judgement` |
| 叙事重点是什么 | `narrative_focus` |
| 增强服务哪个读数或判断 | `enhancement_purpose` |
| 判断依据来自哪里 | `evidence_source` |
| 使用了哪些 Wiki 规范 | `wiki_refs` |

## 4. 分支规则

- 如果任务是“看谁高谁低”，优先考虑排序、高亮、差值标识或基准线。
- 如果任务是“看怎么变化”，优先考虑关键点标注、阶段区间、事件标注、同比环比说明。
- 如果任务是“看是否达标”，优先考虑目标线、达成状态、差距 callout。
- 如果任务是“看结构是否健康”，优先考虑主次突出、Others 合并、长尾展开、结构健康提示。
- 如果任务是“看异常在哪里”，优先考虑异常点 / 行高亮、阈值线、正常区间、证据 Tooltip。
- 如果任务是“验证结论”，优先考虑历史基准、同类对比、预测对比、条件排除和证据展开。

## 5. 输出要求

Agent 输出增强方案时不得只给视觉描述，必须包含：

- 为什么需要增强。
- 增强绑定的数据关系或状态判断。
- 具体增强元素和图层位置。
- 数据证据、置信度和缺失项。
- 是否依赖尚未补齐的基础规范。
