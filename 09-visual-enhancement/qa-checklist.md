# 增强设计自查清单 QA Checklist

## 1. 文档定位

本文定义可视化增强交付前的自查项。

## 2. 关系与叙事

- [ ] 是否先识别了数据关系，而不是直接添加视觉效果？
- [ ] 是否明确用户问题和叙事重点？
- [ ] 增强是否服务最重要的 `1-3` 个信息点？
- [ ] 是否避免把相关性写成因果？

## 3. 数据证据

- [ ] 最大 / 最小 / 最新 / Top 是否来自明确数据字段？
- [ ] 异常、偏离、风险是否有阈值、目标、历史基准或同类基准？
- [ ] 缺少证据时是否标记 `inferred` 或 `needs_spec`？
- [ ] 数值格式是否与坐标轴、Tooltip 和指标卡一致？

## 4. 视觉规范

- [ ] 标签是否继承 `label.md` 的字号、距离和重叠策略？
- [ ] 参考线和区间是否绑定 Mark token？
- [ ] hover 指示器是否继承 `tooltip.md` 和 `axis.md`？
- [ ] 高亮是否没有改变数据语义？
- [ ] 增强是否没有遮挡主数据系列、图例和 Tooltip？

## 5. 交互反馈

- [ ] hover、选中、筛选、联动是否有可见反馈？
- [ ] 全局筛选和局部筛选作用范围是否清楚？
- [ ] 下钻是否保留返回路径？
- [ ] AI 建议或洞察是否可查看证据和验证路径？

## 6. 复杂度

- [ ] 是否避免为所有点显示标签？
- [ ] 系列、类目或点位过多时是否采用降噪策略？
- [ ] 是否避免用过多颜色、callout 或背景区间堆叠？
- [ ] Tooltip 或明细入口是否能补充被隐藏的信息？
