# 可视化增强反模式 Anti Patterns

## 1. 文档定位

本文定义可视化增强中的常见反模式。

## 2. 表达反模式

| 反模式 | 问题 |
|---|---|
| 图表装饰化 | 高亮、箭头、背景只为好看，没有业务含义 |
| 标注泛滥 | 所有点都标注，用户看不到重点 |
| 无证据洞察 | 没有阈值、目标或基准却判断异常 / 达标 / 风险 |
| 相关当因果 | 散点、双轴或趋势同向就写原因 |
| 只靠颜色 | 异常、风险、状态只用红绿表达 |
| 参考线无含义 | 画线但不说明目标、阈值或基准是什么 |
| 区间当背景 | 大面积色块没有起止和业务含义 |
| 交互无反馈 | 点击、筛选或联动后用户不知道影响范围 |
| AI 不可验证 | AI 给出结论但没有证据展开或验证路径 |

## 3. 样式反模式

- 使用 G2 / ECharts 默认样式覆盖 dongDesign 规范。
- hover 指示线溢出标题区或使用虚线。
- 柱状图 hover 同时出现类目高亮和垂直 line。
- Tooltip 遮挡当前 hover 点或主要读数。
- 高亮颜色与系列色、语义色或主题变量冲突。
- 为了容纳内容使用小于 `12px` 的图表文字。

## 4. 处理方式

发现反模式时，Agent 应：

```txt
指出问题
→ 说明影响
→ 回到对应基础规范
→ 给出替代增强方案
→ 标记缺失规范或风险
```
