# 可视化增强原则 Principle

## 1. 文档定位

本文定义可视化增强的通用原则、信息层级和适用边界。

可视化增强不是给图表增加装饰，而是围绕数据关系、用户问题和叙事重点，帮助用户更快识别重点、理解状态并形成判断。

## 2. 核心链路

Agent 设计增强表达时必须先建立以下链路：

```txt
数据关系
→ 用户问题
→ 叙事重点
→ 表达载体
→ 视觉语法
→ 理解增强
```

如果无法说明增强服务哪个用户问题，不应添加增强。

## 3. 增强目标

可视化增强主要服务三类目标：

| 目标 | 回答的问题 | 典型方式 |
|---|---|---|
| 看懂关系 | 数据之间是什么关系 | 排序、对比、高亮、参考线 |
| 判断状态 | 当前是否正常、偏离、异常或恶化 | 阈值线、状态标签、异常标注、区间背景 |
| 继续分析 | 用户下一步如何探索、验证或行动 | 联动、下钻、筛选、证据展开、行动入口 |

## 4. 信息层级

增强元素应遵守以下层级：

```txt
Tooltip / 操作反馈
→ 关键标注 / callout
→ 当前选中或 hover 对象
→ 主数据系列
→ 参考线 / 标注区间
→ 网格线 / 背景结构
```

规则：

- 数据主体优先，增强元素不得抢占主视觉。
- 参考线和区间标注应低于主数据系列，高于网格线。
- Tooltip、hover 点和选中态属于交互反馈层，不应被静态标注遮挡。
- 同一张图默认只突出最重要的 `1-3` 个增强点。

## 5. 证据原则

- 增强必须绑定数据字段、业务口径或用户明确给出的判断依据。
- 最大值、最小值、最新值可以由数据直接计算。
- 异常、偏离、风险、恶化等状态判断必须有目标、阈值、历史基准、同类基准、预测值或明确规则。
- 缺少证据时只能写增强机会，不得写确定结论。

## 6. 与现有规范的关系

增强设计必须继承现有基础规范：

- 文本、字号、数字格式读取 `../01-design-language/typography.md`。
- 标签、引导线和关键点读取 `../01-design-language/label.md`。
- 参考线和标注区间读取 `../01-design-language/theme-token.md` 的 Mark 变量。
- Tooltip 和 hover 指示器读取 `../01-design-language/tooltip.md`。
- 坐标空间和 hover 线读取 `../01-design-language/axis.md`。
- 具体图表能力读取 `../02-chart-type/` 对应文档。

当本目录与具体图表文档冲突时，优先遵守具体图表文档。

## 7. 禁止原则

- 禁止为了好看添加无业务含义的高亮、箭头、图标或背景。
- 禁止只用颜色表达异常，必须配合文字、符号、标签或 Tooltip。
- 禁止把相关性标注为因果，除非有明确归因规则或实验依据。
- 禁止通过视觉强调夸大差异，例如截断坐标轴后仍用夸张 callout 表达增长。
- 禁止在数据密集图表上铺满标签，优先使用 Tooltip、筛选、抽样或展开。
