# 数据证据与置信度 Data Evidence

## 1. 文档定位

本文定义可视化增强所需的数据证据、置信度和禁止推断边界。

## 2. 证据类型

| 证据类型 | 可支持的增强 |
|---|---|
| 原始数值 | 最大值、最小值、排序、差值、Top / Bottom |
| 时间序列 | 趋势、拐点、峰谷、连续下降、阶段变化 |
| 目标 / 计划值 | 目标线、完成率、差距 callout、达成状态 |
| 阈值 / SLA / 风控规则 | 阈值线、异常、预警、风险区间 |
| 历史基准 | 同比、环比、正常范围、波动判断 |
| 同类基准 | 同类对比、排名、偏离、异常对象 |
| 预测值 | 预测偏差、置信区间、达成风险 |
| 业务事件 | 事件标注、阶段区间、解释线索 |
| 用户口径 | 状态解释、筛选条件、业务动作入口 |

## 3. 置信度

| 置信度 | 条件 | Agent 表达 |
|---|---|---|
| `confirmed` | 数据字段和判断规则都明确 | 可输出确定增强和判断 |
| `partial` | 有数据但缺少完整业务规则 | 可输出增强，并标记判断限制 |
| `inferred` | 只能从可见图形或上下文推断 | 只能输出“建议 / 机会” |
| `needs_spec` | 缺少基础规范或判断口径 | 不输出确定状态，记录缺口 |

## 4. 禁止推断

- 没有目标或阈值时，不得判断“达标 / 未达标”。
- 没有正常范围或基准时，不得判断“异常”。
- 没有连续时间点时，不得判断“趋势恶化”。
- 没有贡献拆解或归因模型时，不得判断“主要原因”。
- 只有截图且读数不完整时，不得虚构精确数值。

## 5. 输出字段

Agent 必须在增强方案中标明：

```txt
evidence_source:
confidence:
missing_evidence:
risk_note:
```
