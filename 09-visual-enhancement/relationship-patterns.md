# 数据关系与状态范式 Relationship Patterns

## 1. 文档定位

本文定义可视化增强设计前必须识别的数据关系、状态判断和叙事重点。

增强表达的出发点不是“加什么效果”，而是“当前数据正在表达什么关系”。

## 2. 三类数据关系

| 类型 | 子范式 | 核心问题 | 常见增强 |
|---|---|---|---|
| 看变化 | 对比、趋势、排序、目标 | 谁更高、怎么变、谁优先、离目标多远 | 差值高亮、阶段标注、Top 高亮、目标线 |
| 看结构 | 构成、层级、分布 | 谁组成谁、如何展开、集中在哪里 | 主次突出、面包屑、区间分层、异常区间 |
| 看联系 | 关联、流转、因果 | 是否相关、从哪到哪、为什么发生 | 趋势线、路径高亮、转化标注、贡献度排序 |

## 3. 五类状态判断

状态判断是基于数据关系形成的判断，不是独立装饰。

| 状态 | 依赖关系 | 推荐增强 |
|---|---|---|
| 偏离 | 对比、目标、基准、阈值、分布区间 | 目标线、差距标签、偏离区间、状态标签 |
| 波动 | 趋势、历史区间、稳定性基准 | 波动区间、峰谷标注、历史范围、稳定性说明 |
| 异常 | 对比、趋势、分布、阈值、预测 | 异常点高亮、阈值线、异常说明、影响范围 |
| 失衡 | 构成、分布、层级、集中度 | Top 集中度、长尾展开、结构健康度 |
| 恶化 | 趋势、目标、偏离、异常、结构变化 | 连续下降标识、风险升级、阶段对比 |

## 4. 叙事重点

同一组数据可以有不同增强策略。Agent 必须先判断叙事重点。

| 叙事重点 | 增强方向 |
|---|---|
| 强调个体 | 高亮对象、关键标签、明细入口 |
| 强调变化 | 拐点、峰谷、阶段区间、事件标注 |
| 强调总量 | 指标卡、趋势摘要、目标差距 |
| 强调对比 | 差值、排序、基准线、对比 callout |
| 强调时间 | 时间段标记、粒度切换、事件点 |
| 强调占比 | 主次突出、结构标签、Others 合并 |
| 强调异常 | 异常点、阈值、正常区间、证据说明 |
| 强调原因 | 贡献度排序、归因路径、证据展开 |

## 5. Agent 使用规则

- 先判断关系，再选择图表或增强元素。
- 先判断状态是否有依据，再决定是否使用风险、异常、恶化等语义。
- 叙事重点只能有一个主线，其他增强作为辅助。
- 关系不明确时，优先输出“待确认用户问题”，不要堆叠增强。
