# 增强决策树 Enhancement Decision Tree

## 1. 文档定位

本文定义 Agent 在不同数据关系、状态判断和用户任务下如何选择增强方式。

## 2. 总决策树

```txt
是否有明确用户问题？
├─ 否：不添加增强，只给出可选增强机会
└─ 是：
   是否需要状态判断？
   ├─ 是：检查目标 / 阈值 / 基准 / 正常区间 / 同类对比
   │  ├─ 有依据：使用参考线、区间、状态标签、异常标注
   │  └─ 无依据：标记 needs_spec，不输出确定状态
   └─ 否：
      判断数据关系
      ├─ 对比 / 排序：高亮、差值、Top N、基准线
      ├─ 趋势：关键点、阶段区间、事件标注、同比环比
      ├─ 目标：目标线、差距 callout、达成状态
      ├─ 构成：主次突出、Others 合并、占比标签
      ├─ 分布：区间分层、异常区间、密度提示
      ├─ 关联：趋势线、象限、相关说明、异常点
      ├─ 流转：路径高亮、节点转化、流失标注
      └─ 因果：贡献度排序、影响路径、证据展开
```

## 3. 增强方式选择表

| 用户问题 | 首选增强 | 谨慎使用 | 禁止 |
|---|---|---|---|
| 谁最高 / 最低 | Top / Bottom 高亮、排序标签 | 多色分类 | 随机给每个对象上色 |
| 差多少 | 差值标签、基准线 | 大面积 callout | 无基准时写“显著” |
| 什么时候变化 | 拐点、峰谷、事件标注 | 所有点标签 | 用箭头替代趋势证据 |
| 是否达标 | 目标线、达成状态、差距说明 | 多条目标线 | 目标未知时判断达标 |
| 是否异常 | 阈值线、异常点、正常区间 | 红色大面积告警 | 只用颜色表达异常 |
| 结构是否健康 | 占比标签、Top 集中度、长尾展开 | 饼图过多标签 | 无规则时写“结构不健康” |
| 为什么发生 | 贡献度排序、归因路径、证据展开 | 单一原因 callout | 把相关性写成因果 |
| 如何继续分析 | 下钻、筛选、联动、证据展开 | 过多入口 | 无反馈的隐藏交互 |

## 4. 冲突处理

- 如果增强会遮挡主数据系列，优先降低增强强度或移动到 Tooltip。
- 如果增强数量超过 `3` 个，优先保留与叙事重点最相关的增强。
- 如果图表类型文档明确禁止某种增强，以图表类型文档为准。
- 如果基础规范缺失，输出 `needs_spec`，并引用 `foundation-gaps.md`。
