# 交互增强 Interaction Enhancement

## 1. 文档定位

本文定义图表增强中的交互能力，用于帮助用户从静态查看进入探索、验证和行动。

交互增强的核心不是增加操作，而是支持连续分析路径。

## 2. 八类交互能力

| 能力 | 作用 | 常见方式 |
|---|---|---|
| 范围控制 | 缩小观察范围 | 时间范围、条件筛选、搜索、多维切片 |
| 视角切换 | 改变观察角度 | 指标切换、维度切换、粒度切换、口径切换 |
| 层级探索 | 从总览进入细节 | 下钻、上卷、展开、面包屑 |
| 关联联动 | 建立视图间关系 | 多图联动、表图联动、洞察联动 |
| 细节显现 | 补充局部信息 | Tooltip、抽屉、行展开、点位详情 |
| 比较验证 | 验证判断是否成立 | 同比环比、基准线、对照组、条件排除 |
| 解释归因 | 解释为什么发生 | 贡献拆解、原因链、影响路径 |
| 沉淀协同 | 保存过程并行动 | 保存视图、生成报告、创建任务、订阅 |

## 3. 与数据关系的对应

| 数据关系 / 状态 | 交互增强重点 |
|---|---|
| 对比 / 趋势 / 排序 / 目标 | 时间范围、同比环比、Top N、目标差距下钻 |
| 构成 / 层级 / 分布 | 结构下钻、Others 展开、区间筛选、长尾展开 |
| 关联 / 流转 / 因果 | 节点高亮、路径追踪、变量切换、原因证据查看 |
| 偏离 / 波动 / 异常 / 失衡 / 恶化 | 阈值查看、正常区间、影响范围联动、诊断入口 |

## 4. 反馈状态

每次交互都必须让用户理解：

```txt
当前操作影响了哪些数据
当前看到的是什么范围
哪些视图发生了联动
如何回到上一步
```

必须设计的状态：

- hover 态
- 选中态
- 联动态
- 加载态
- 空状态
- 禁用态
- 错误态
- 筛选条件回显
- 下钻面包屑或路径

## 5. 复杂度控制

- 默认简单，按需展开。
- 高频操作前置，低频能力进入更多操作、抽屉或高级设置。
- 全局筛选和局部筛选必须区分作用范围。
- 下钻必须保留返回路径。
- AI 建议必须可验证，可展开证据和条件。
