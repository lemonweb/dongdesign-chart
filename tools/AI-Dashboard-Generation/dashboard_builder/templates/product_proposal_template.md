# Product Proposal Template: AI搭建看板

## 1. Product positioning

AI搭建看板 is an independent AI Native BI entry. It lets users generate real-data-bound, high-expression BI dashboards through conversation and vibe coding.

## 2. Why now

Traditional BI dashboards are stable but rely heavily on manual configuration, chart literacy, and visual judgment. Many users can build usable dashboards, but not necessarily readable, beautiful, or decision-oriented dashboards.

AI搭建看板 solves this by combining:

- natural-language business intent understanding
- real BI dataset binding
- structured analysis framework generation
- vibe-coding visual rendering
- Harness governance
- organizational memory accumulation

## 3. Target users

- Business operators
- Category managers
- Store managers
- Business analysts
- Executives or managers preparing review meetings
- Product/data teams building scenario templates

## 4. Core scenarios

- Campaign review
- Monthly/weekly operating review
- Store comparison
- Product/category analysis
- Executive report
- Risk monitoring
- Temporary topic analysis

## 5. Core workflow

1. Enter AI搭建看板
2. Describe business goal
3. Select or recommend dataset
4. AI maps metrics and dimensions
5. AI previews analysis framework
6. AI generates page through vibe coding
7. User edits through conversation
8. Harness validates data and logic
9. Save/publish/export
10. Capture reusable learning

## 6. Key capabilities

- Intent recognition
- Dataset binding
- Metric/dimension mapping
- Analysis framework generation
- Vibe-coding visual page generation
- Natural-language revision
- Data and permission validation
- Insight traceability
- Template and organizational memory accumulation

## 7. MVP scope

Recommended first version:

- Independent entry
- 3–5 preset scenarios
- Dataset selection
- Field mapping confirmation
- Analysis outline preview
- Generated dashboard preview
- Natural-language style/structure edits
- Save as generated dashboard/report page
- Basic Harness validation

## 8. Risks

- Beautiful but fake pages
- Metric definition ambiguity
- Permission leakage
- Over-free code generation
- Weak maintainability
- User over-trust in AI-generated conclusions

## 9. Risk controls

- Data contract first
- Metric dictionary validation
- Permission inheritance
- Human confirmation for ambiguous metrics
- Unsupported insight warnings
- Traceability from insight to field
- Generated page modularization

## 10. Long-term direction

AI搭建看板 should evolve from a dashboard generator into a BI strategy asset system. It should continuously accumulate scenario templates, metric mappings, design patterns, analysis frameworks, and Agent workflows.
