# Organizational Memory

AI搭建看板 should accumulate reusable capability instead of only producing one-off dashboards.

## What to capture

Capture after each generation or edit:

- user role
- business scenario
- original user prompt
- selected dataset
- accepted metric mappings
- corrected metric mappings
- generated analysis framework
- accepted sections
- removed sections
- preferred visual style
- follow-up editing commands
- final dashboard structure
- final conclusions if approved by human
- save/share/export behavior

## What to convert into assets

### Scenario strategy asset

Reusable analysis path for a scenario.

Example:

- 618 campaign review
- monthly operating review
- store comparison
- product/category analysis
- executive report

### Metric dictionary improvement

Reusable mapping from business wording to metric definitions.

Example:

- “销售额” → GMV or paid amount, depending on business context
- “利润” → gross profit, net profit, contribution profit, or margin; requires confirmation

### Visual pattern asset

Reusable visual structures.

Example:

- executive KPI hero area
- campaign rhythm trend
- contribution ranking board
- anomaly alert strip
- action recommendation panel

### Agent workflow asset

Reusable workflow for repeated scenarios.

Example:

- campaign review workflow
- boss report workflow
- two-store comparison workflow

### Harness rule asset

Reusable validation rules.

Example:

- campaign GMV must define payment time range
- comparison baseline must be explicit
- profit metrics require permission check

## Memory quality criteria

Only promote a pattern to organizational memory if:

- it was used more than once, or
- it was explicitly saved/approved by a user, or
- it corrected a common ambiguity, or
- it materially improved dashboard quality.

Do not memorize sensitive business data as a generic template.
