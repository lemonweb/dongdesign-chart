# 全局摘要总结卡片（洞察摘要）规范

> 报告头部的 AI 全局摘要卡片。提取自 Figma「洞察摘要」三种形态。颜色/字体一律引用《颜色令牌.md》《文本规范.md》。

---

## 一、组件定位

**使用场景**：位于报告头部（诊断建议区），由 AI 对整份报告做**全局摘要与结论**，是用户进入报告后最先读到的"结论先行"区块。

**三种形态**（按信息复杂度递进）：

| 形态 | 内容 | 典型用途 |
|---|---|---|
| ① 基础版 | 分段标题 + 正文 + 数字（含加粗/变色/下划线高亮） | 纯结论陈述 |
| ② 联动交互版 | AI 标签分段 + 正文 + **可点击紫色标签/词**（点击定位跳转到报告对应内容） | 结论可下钻、与正文联动 |
| ③ 优化建议版 | 摘要 + 优化建议卡（建议标题 + 预计提升 + 一键操作按钮） | 结论 + 可执行动作 |

---

## 二、通用结构

```
洞察摘要卡片
├─ 渐变标题「洞察摘要」   ← AI 星标图标(18×18) + 渐变文字
└─ 内容区（分段）
   ├─ 段1：分段标题/AI标签 + 正文（含高亮）
   ├─ 段2 …
   └─ 段N …
```

| 元素 | 规格 |
|---|---|
| 卡片背景 | `--jd-color-surface-200`（#FFFFFF） |
| 卡片圆角 | 8px（`--jd-radius-xxl`） |
| 卡片内边距 | 上下 16px、左右 20px |
| 标题「洞察摘要」 | 18px / 28px，Semibold，**AI 渐变文字**：`linear-gradient(259deg,#8F55FD,#6B36FA 30%,#6B36FA 85%,#3544EB)` + `background-clip:text` |
| 标题左侧 AI 星标图标 | 18×18，图标与文字间距 4px |
| 标题 → 正文间距 | 12px（`--jd-spacing-vertical-s`） |
| 段落行高 | 22px；行间距 8px |

## 三、重点样式细则（重点关注）

### 3.1 重点文案 加粗 / 高亮

> ⚠️ **紫色 = 交互专用（重要）**：紫色 AI 标签、紫色术语下划线**只用于"带联动跳转交互"的结论**（点击可定位到正文来源 / 查看趋势详情，见 3.3）。**若结论没有联动交互，一律不得使用紫色标签或紫色下划线**——分段标签用中性文字标签（`text-400` 加粗、无底色），重点词用普通加粗即可。

正文默认 **14px / 22px / Regular(400) / `text-300`(#515357)**；在此基础上分三档强调：

| 强调类型 | 样式 | 用途 |
|---|---|---|
| 普通正文 | Regular 400 · `text-300` | 连接性叙述文字 |
| **加粗强调** | **Semibold 600 · `text-400`(#1C1D1F)** | 核心判断词、关键结论（如"规模大但转化偏低""核心因素"） |
| **数字强调** | Semibold 600 · 数字字体 Roboto · `text-400` | 行内指标数值（如 17亿、2635万、3.76%） |
| **术语下划线高亮** | `text-400` 加粗 + **AI 紫下划线** `rgba(107,54,250,.2)`、4px、置于文字底部 | 关键业务名词（如"规模大但转化偏低"），提示"这是一个可关注/可下钻的概念" |

> 加粗与下划线可叠加；下划线颜色恒为 AI 紫 20% 透明度，不随涨跌变色。

### 3.2 数字的颜色变化

行内数字分两类：

| 类型 | 颜色 |
|---|---|
| 中性数值（规模、占比等客观量） | `text-400`（#1C1D1F）黑色加粗 |
| **变化/对比值**（同比、环比、pp、涨跌） | **按涨跌 + 产品规则上色**（见下） |

**涨跌配色 —— 按产品区分（引用 `--jd-color-trend-*`）**

| 产品 | 上涨/正向 | 下跌/负向 | 持平 |
|---|---|---|---|
| 对外产品（红升绿降） | 红 `trend-up` #F33B50 | 绿 `trend-down` #009A5F | 灰 #B0B2B8 |
| 对内产品（绿升红降） | 绿 #009A5F | 红 #F33B50 | 灰 #B0B2B8 |

> 变化值一律按"产品涨跌规则"上色、字重 Semibold；上文 `4.08pp` 标红仅为演示，不是固定值——实际颜色由产品（对外/对内）决定。

### 3.3 可交互元素（形态②）

正文中有**两类可点击元素**，语义不同：

**A. 紫色 AI 标签 —— 点击定位「信息来源」**

- 摘要结论必然来源于报告正文的数据；点击紫色 AI 标签后，页面**滚动定位到报告正文中该结论的信息来源模块并高亮**，帮助用户快速溯源、下钻查看依据。
- 元素为 AI 色 `--jd-color-ai-normal`（#6B36FA）的标签样式（填充式 AI tag）。
- **位置不固定**：不一定在段首，可**内联出现在正文任意位置**，跟随它所修饰的内容。
- 视觉态：默认 AI 紫；Hover 加深至 `ai-hover` #9975FC 或显下划线；`cursor:pointer`；键盘可聚焦，有 focus 态。

**B. inline mini 趋势图 —— 点击查看「趋势详情」**

- 正文中内联的**迷你趋势图**（sparkline，约 40×18，含 `symbol` 标记），表达该指标近期走势。
- 点击后**查看趋势详情**（放大 / 弹层展示完整趋势图）。
- 与 A 区分：**A = 定位到正文来源，B = 展开趋势细节**；两者可同时出现在同一句话中。

## 四、形态③：优化建议卡

摘要正文之后，追加"优化建议"分段，内含若干**建议卡**（一行 2 张，可多行）：

| 元素 | 规格 |
|---|---|
| 建议卡尺寸 | 单卡约 452×68，**双列**，内边距 16×12，两卡间距 8px |
| **建议卡背景** | **浅紫 `#F7F5FF`**（AI 浅底，非白色）；圆角 8，无描边 |
| 建议标题 | 14px / 22px，Semibold，`text-400`（如"优化商品主图视频突出卖点"） |
| 建议描述 | 12px / 18px，`text-200`（如"商品详情页浏览转化率预估可提升"） |
| **预计提升值** | 紧随描述，数字 Roboto Medium 12；**按产品涨跌规则上色**（正向色：对外红 / 对内绿）+ 红色 `↗` 图标；源文本无数值时不显示、不臆造 |
| **一键操作按钮** | 右侧 Button，约 **76×28**，圆角 4；**白底 `#FFFFFF` + 1px 描边 `line-200`(#DCDDE0) + 文字 `text-300`(#515357) + `›` 箭头**（次级按钮样式，**非品牌蓝描边**）；文案可配置（去优化 / 去设置 / 去发布…） |

> ⚠️ **这是洞察摘要专属的优化建议卡，勿与《action-cards.md》的"策略卡"混用**：
> - 本卡：**双列（每张约 452×68）**，结构 = 标题 + [描述 +（可选）预估提升 `↗`] + 右侧 **76×28** CTA 按钮；**无优先级 Tag、无时间分组**。
> - action-cards 策略卡：可带**优先级 Tag（P0/P1/P2）、时间分组**，按钮约 **52×24**，有单卡/卡组/公式等多形态。
> 两者按钮尺寸、是否分组/带优先级都不同，生成洞察摘要时用本节规格。

> ⚠️ **忠实设计、不臆造元素**：问题定位等多条要点**按原文条数直接呈现，不自动添加 1/2/3/4 序号、项目符号或设计稿中没有的装饰元素**。只渲染源文本与规范中确有的内容。

## 六、交互与配置说明（已确认）

1. **标题字号**：18px / 28px Semibold，AI 渐变文字。
2. **可点击范围（形态②）**：仅**紫色 AI 标签**可点击；位置不固定、可内联在正文任意处。点击 → 定位到报告正文中该结论的**信息来源**（帮助用户快速溯源）。
3. **inline mini 趋势图**：正文中的迷你趋势图可点击 → **查看趋势详情**（与"定位来源"是两种不同交互）。
4. **预计提升值**：颜色随**产品涨跌规则**（对外红升 / 对内绿升）；**按钮文案可按实际报告内容配置**。
5. **变化值配色**：统一按既定的**产品规则**（对外红升绿降 / 对内绿升红降），示例中的 4.08pp 标红仅为演示。

# 附：局部诊断条（Module Diagnosis Bar）

# 局部诊断条（Module Diagnosis Bar）

> 提取自 Figma（node 39:12636）。放在**每个模块标题下方**，用一句话总结该模块的关键信息。是《summary-card.md》的"模块级、轻量单行版"。颜色/字体引用《color-tokens.md》《typography.md》。

---

## 一、组件定位

- **位置**：报告中每个（一级/二级）模块标题的**正下方**。
- **作用**：对该模块内容做**一句话结论式总结**，帮助用户在看图表/表格前先拿到判断。
- **特点**：样式简单（单行渐变条），但同样需要**重点文案加粗、关键术语高亮、数值变色**。

---

## 二、结构与尺寸

```
[AI 星标图标 18×18] + 4px + [结论文案（加粗数字 / 下划线术语 / 变色变化值）]
```

| 项 | 值 |
|---|---|
| 宽度 | 对齐内容列（示例 912） |
| 高度 | 46px（单行）；文案过长时换行、高度自适应 |
| 内边距 | 水平 16px、垂直 12px（文本行高 22） |
| 背景 | AI 浅色渐变 `ai-light-background`：`linear-gradient(225deg,#FAF6FF 0%,#F2F5FF 70%,#F1F4FF 100%)` |
| 圆角 | 6px |
| 描边 | 可选 `ai-light-border` |
| 左侧图标 | AI 星标 18×18（AI 渐变），与文字间距 4px |

> 与"诊断：条"的区别：本条以 **AI 星标图标起头、无"诊断："文字标签**，更轻量。

## 三、文案样式（重点）

| 类型 | 样式 | 示例 |
|---|---|---|
| 普通正文 | 14 / 22，Regular，`text-300`（#515357）或 `text-400` | "…主力品类成交金额…" |
| **重点加粗** | Semibold 600，`text-400`（#1C1D1F） | 核心判断词"规模大但转化偏低" |
| **关键数字** | Semibold 600，数字用 Roboto | `2亿`、`3.01%`、`31.01%` |
| **术语下划线高亮** | 加粗 + **AI 紫下划线** `rgba(107,54,250,.2)`、置于文字底部 | "规模大但转化偏低" |
| **变化/对比值** | Semibold；**按产品涨跌规则上色**（`--jd-color-trend-*`） | `-12.21%`（示例下降；对内标红 / 对外标绿） |

## 六、落地约定

- 每个模块标题下方**可选**放一条；一句话概括，突出 1–2 个关键数字或结论，不堆砌。
- 重点信息加粗、关键术语加 AI 紫下划线、变化值按产品涨跌变色（不写死）。
- 文案较长时自动换行，高度随之自适应，内边距保持 16·12。
- 颜色、字体、圆角一律引用令牌。
