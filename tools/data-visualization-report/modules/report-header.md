# AI 可视化报告生成规范

> 本文档定义 AI 自动生成可视化分析报告时的设计规范，确保生成结果具备一致的视觉语言与信息结构。

---

## 一、报告头部组件规范

### 1.1 组件定位

**使用场景**

报告头部用于 AI 自动生成分析报告的顶部信息区域，帮助用户快速理解：

- 当前报告主题
- 分析对象
- 数据范围
- 分析条件
- 报告生成信息

**设计目标**

让用户在 10 秒内完成以下认知：

- 这是什么报告？
- 分析什么对象？
- 看什么时间？
- 覆盖什么范围？

### 1.3 生成规则

#### （1）报告标题（必有）


| 属性 | 规则 |
|------|------|
| 生成方式 | 由 AI 自动生成 |
| 内容格式 | 时间范围（可选）+ 分析对象 + 报告类型，例：`2026年6月手机品类经营分析报告` |
| 展示规则 | 单行展示，不超过 30 个中文字符，超出省略 |

#### （2）数据统计周期（必有）

- **格式**：`数据统计周期:[label]`

**label 生成规则：**

单个时间点按入参粒度格式化；区间的起止时间同样根据入参粒度选择对应格式，起止以 ` ~ ` 连接。

| 粒度 | 单点格式示例 | 区间格式示例 |
|------|-------------|-------------|
| 秒 | `YYYY-MM-DD HH:mm:ss` | `YYYY-MM-DD HH:mm:ss ~ YYYY-MM-DD HH:mm:ss` |
| 时分 | `YYYY-MM-DD HH:mm` | `YYYY-MM-DD HH:mm ~ YYYY-MM-DD HH:mm` |
| 日 | `YYYY-MM-DD` | `YYYY-MM-DD ~ YYYY-MM-DD` |
| 周 | `YYYY第W周` | `YYYY第W周 ~ YYYY第W周` |
| 月 | `YYYY-MM` | `YYYY-MM ~ YYYY-MM` |
| 季度 | `YYYY年 Qx` | `YYYY年 Qx ~ YYYY年 Qx` |
| 年 | `YYYY年` | `YYYY年 ~ YYYY年` |

#### （3）筛选条件（条件展示）

**维选展示规则**

每个筛选条件至多展示一个维选值：
- 若该筛选条件只有一个维选，直接展示；
- 若该筛选条件存在多个维选，只展示第一个，其余 n 个以 tag 样式显示为 `+n`；
- 鼠标悬停 `+n` tag 后，以 popover 为容器展示其余 n 个维选值。

**省略规则**

筛选条件区域至多展示一行，数量自适应容器宽度；超出一行的条件收起，以「更多」入口代替，点击后弹出面板展示所有被省略的条件。

**「更多」折叠面板视觉规范**

| 属性 | 值 |
|------|----|
| 背景色 | `--jd-color-surface-200` · #FFFFFF |
| 边框 | `1px solid --jd-color-line-100（#E6E7EB）` |
| 圆角 | 8px |
| 内边距 | 上下 16px，左右 20px |
| 阴影 | `0px 12px 24px -4px rgba(0,0,0,0.06), 0px 4px 28px 0px rgba(0,0,0,0.04)` |
| 条目布局 | flex-wrap；行间距 4px，列间距 12px |
| 触发方式 | 点击「更多」按钮展开；点击面板外部区域关闭；点击面板内部不关闭 |
| 定位方式 | 挂载在 `<body>`，`position: fixed`，通过 JS `getBoundingClientRect()` 定位到按钮正下方 4px；页面滚动时实时更新位置 |
| 面板内 +n tag | 面板内的筛选条件同样支持 +n tag，悬停交互规则与主信息行一致 |

#### （4）实体卡（按需展示）

**需要展示**：分析对象本身含丰富业务实体、需要帮助信息梳理，分析对象数量小于 3

| 场景 | 实体卡示例 |
|------|------------|
| 单品分析 | SKU/SPU 信息 |
| 营销活动分析 | 活动信息 |
| 单渠道分析 | 渠道信息 |
| 单竞品分析 | 品牌/产品信息 |
| 店铺分析 | 店铺信息 |

**不需要展示**：对象只是筛选范围而对比维度，用户不需要了解其详细属性，分析对象数量大于等于 3，展示为筛选条件即可

| 场景 | 示例 |
|------|------|
| 多品牌分析 | 苹果/华为/小米 |
| 多渠道分析 | 直播流量/外部引导/SEO/DMO/… |
| 多竞品分析 | SKU1/SKU2/SKU3 |

### 1.4 响应式规范

报告以 **1000px 宽面板 / 936px 内容区** 为基准设计，头部在不同宽度下的自适应行为如下。

#### 断点

| 断点 | 宽度区间 | 说明 |
|------|---------|------|
| 桌面 Desktop | ≥ 1000px | 基准形态，面板固定 1000px 居中 |
| 平板 Tablet | 720 ~ 999px | 面板宽度自适应容器（`width:100%; max-width:1000px`），内容区左右留白收窄至 20px |
| 移动 Mobile | < 720px | 单列布局，多处元素降级/折叠 |

#### 各元素自适应规则

| 元素 | 桌面 | 平板 | 移动 |
|------|------|------|------|
| 报告标题 | 30px / 46px，单行超 30 字省略 | 同左 | **字号降级为 22px / 32px**，最多 2 行后省略 |
| 数据统计周期 | 常规展示 | 同左 | 若与筛选条件同行放不下，则换至独立行 |
| 筛选条件行 | 自适应容器宽度，超一行收起为「更多」 | 同左（可折叠数量增多） | 默认仅展示 1~2 个高优条件，其余全部进「更多」 |
| `+n` popover / 「更多」面板 | `getBoundingClientRect` 定位，随滚动更新 | 同左 | 面板宽度不超过视口宽度减 32px，超出内部滚动 |
| 实体卡 | 支持 2 张并排、等高 | 2 张并排（间距缩至 12px） | **强制纵向堆叠**，逐张全宽 |
| 实体卡图片 | 48×48 | 48×48 | 40×40（可选降级） |
| 装饰图 | 固定 358×126，右上角 `z-index:0` | 等比缩小或右对齐裁出视口部分 | **隐藏**（`display:none`），避免遮挡标题 |

#### 通用约束

- 头部所有交互层（`+n` popover、「更多」面板）均挂载 `<body>`、`position:fixed`，窗口 resize 与页面滚动时**实时重算位置**。
- 内容区与装饰图的层叠始终为：装饰图 `z-index:0` < 内容/实体卡 `z-index:1`，任何断点下内容都在装饰图之上。
- 文本一律允许自然回流；标题、商品名等单行元素使用 `text-overflow:ellipsis` 截断，不允许撑破容器。
- 颜色一律引用《颜色令牌.md》中的 token；本文色值已全部替换为 `--jd-color-*`。仅两处暂无精确 token、保留原值：TitleBar 渐变 `#F9F8FF → #F5F4FF`、`+n` tag 的中性半透明灰 `rgba(170,170,170,.18/.38)`（如需入库可另议归入 fill / ai-light）。
