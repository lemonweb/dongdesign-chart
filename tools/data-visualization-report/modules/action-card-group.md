# 行动建议卡片规范（Action Cards）

> 提取自 Figma「行动建议卡片」规范板（node 5:40199）。颜色/字体/间距引用《color-tokens.md》《typography.md》。

---

## 一、组件定位

**使用目的**：基于 AI 分析结果，为用户提供**可执行的优化建议**，帮助用户从"发现问题"过渡到"采取行动"。

**使用原则**：
- 建议必须具备**可执行性**，避免泛化描述；
- 优先给出**高价值、高优先级**建议；
- 支持**跳转业务系统**（一键操作）。

---

## 二、卡片类型

### 1. 单策略卡（基础单元）

一张卡片聚焦**一个策略主题**。四种可叠加的变体：

| 变体 | 组成 |
|---|---|
| 基础策略卡 | 建议标题 + 副描述（依据） |
| 含 CTA 按钮 | 基础 + 右侧「去优化」按钮 |
| 含优先级 | 标题前加**优先级 Tag**（P0…）+ 可选预估提升 |
| 预期收益视觉强化 | 描述后加**预估提升值**（涨色高亮 + ↗）+ 按钮 |

**卡片规格**

| 项 | 值 |
|---|---|
| 背景 | `--jd-color-surface-200`（#FFFFFF） |
| 圆角 | 8px（`radius-surface-m`） |
| 内边距 | 12px |
| 卡高 | 两行 ~68px；三行 ~90px |
| 卡间距 | 12–13px |

### 2. 策略卡组（多条列表）

多个策略卡成组，按优先级组织，两种排布：

- **优先级分组**：每个优先级一个**浅色分区容器**（按优先级取浅底）+ 组头（优先级 Tag + 时间范围文字）+ 组内卡片（纵向堆叠或 2 列）。组头右侧可加「曝光预估提升 +X」。
- **多条列表（不分组）**：卡片平铺（常 2 列），**只在卡片组顶部标注一个优先级/时间标签**，不逐卡标注。

---

## 三、优先级体系（重点）

优先级用 **Tag + 分区浅底** 双重表达，按紧急度/时间分级：

| 级别 | 时间口径 | Tag 文字色 | Tag 底色 | 分区浅底 |
|---|---|---|---|---|
| **P0**（短期/紧急） | 1～2 周内 / 短期建议 | `red-9` #611820 | `red-2` #FEDADA | `orange-1` #FCEFE0（暖底） |
| **P1**（中期） | 1～2 月内 / 中期建议 | `orange-9` #603514 | `orange-2` #FDDEBF | `yellow-1` #FEFAEC |
| **P2**（长期） | 季度级 / 长期建议 | `blue-9` #102D68 | `blue-2` #D6E3FD | `blue-1` #EAF1FE |

> 优先级 Tag 约 24×18、圆角 4；时间口径与 P 级可二选一或组合展示（如「P0 · 1～2周内建议」）。分组容器用对应"分区浅底"，卡片仍为白底。

---

## 四、卡片内元素样式

| 元素 | 样式 |
|---|---|
| 建议标题 | 14 / 22，**Semibold**，`text-400`（#1C1D1F）。如"优化高流量低转化商品" |
| 副描述 / 依据 | 12 / 18，Regular，`text-200`（#8C8D91）。如"TOP50高流量商品专项优化" |
| 预估提升前缀 | 12 / 18，`text-200`。如"商详页浏览转化率预估可提升" |
| **预估提升值** | 数字字体 Roboto Medium 12；**按产品涨跌规则上色**（正向色）；后跟 ↗ 图标（`arrow-corner-up-right`，12×12）。如 `2.4% ~ 3.2% ↗` |
| 优先级 Tag | 24×18，圆角 4，分级色（见第三节） |
| CTA 按钮 | 52×24，圆角 4，`brand` #3768FA（描边 → hover 填充）；**文案可配置**（去优化 / 去报名 / 去投放 …） |
| 组头时间/优先级标签 | Tag + 文字（`medium-14`），组头右侧可挂"曝光预估提升 +X" |

## 五、令牌速查

| 用途 | 令牌 / 值 |
|---|---|
| 卡片底 / 圆角 / 内边距 | surface-200 #FFFFFF / 8 / 12 |
| 标题 | text-400，Semibold 14/22 |
| 描述 | text-200，Regular 12/18 |
| 预估提升值 | 数字 Roboto Medium 12 + 涨色（`trend-*`，按产品） + ↗ |
| 优先级色阶 | red / orange / yellow / blue 的 1·2·9 档（底浅、字深） |
| 按钮 | brand #3768FA，52×24，圆角 4 |
| 间距 | 卡内 12 / 卡间 12 / 元素 4·8 |

## 六、生成规则 / 落地约定

- 每条建议结构 = **标题（做什么）+ 依据/预期（为什么 / 预计提升）+（可选）一键操作**。
- 有量化预期提升 → 用**正向涨色 + ↗** 高亮（颜色随产品规则）。
- 多条建议**按优先级分组**，优先展示高优先级（P0 → P2）；建议不超过每组 2–4 条。
- 建议尽量**可跳转业务系统**（按钮或整卡可点）。
- 颜色、字体、圆角、间距一律引用令牌；优先级色用 red/orange/yellow/blue 分级色阶。
