# rules/qa-checklist · 交付前自检

生成 HTML 后、交付前逐条核对：

## 配色 / 语义
- [ ] 已判断**产品（对内/对外）**，涨跌红绿方向正确（对外红涨绿跌 / 对内绿涨红跌），未用反。
- [ ] 变化值用 `+/-`，**KPI 卡未用升降箭头**。
- [ ] 紫色标签/术语高亮**仅用于带联动交互**的结论；无交互处未用紫色。
- [ ] 颜色一律引用令牌，无手写散色值；图表走 `echarts-theme.json` 分类色板。

## 字体 / 文本
- [ ] KPI 主数值用**京东正黑体**（服务端 @font-face），单位 14px 基线对齐。
- [ ] 洞察摘要正文为 `text-300`；标题层级、字号符合 typography。
- [ ] 行内数字用 Roboto。

## 布局
- [ ] 面板底 #FAFAFF，头部渐变与之拼接；洞察摘要、模块**白卡无描边**；页脚在白卡外。
- [ ] 装饰图右侧距页面右缘 70px、底边贴首张白卡上缘、按实际尺寸不拉伸。
- [ ] 内容列 912、模块间 1px 分割线；间距符合 bento-layout。
- [ ] 表格**等比自适应**（table-layout:fixed 填满、无独占超宽列）；不横滚时首列不显竖分隔线。

## 内容忠实
- [ ] **未编造数据**；缺值/示意已注明；源文档矛盾已忠实呈现并指出。
- [ ] **未添加设计稿/源文本没有的元素**（序号、图标、装饰、预估提升等）。

## 交付
- [ ] 输出单个 .html，保存到用户文件夹，用 `computer://` 链接交付。
- [ ] 对照 `examples/single-product-report/report.html` 核验整体观感。
