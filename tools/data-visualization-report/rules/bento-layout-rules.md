# 报告整体布局规范

> 提取自 Figma「报告布局」骨架（node 37:8797，宽 1000）。定义报告的栅格、纵向结构与间距节奏。颜色/字体/数值引用《颜色令牌.md》《文本规范.md》，头部与页脚见各自组件文档。

---

## 一、栅格与宽度

| 层级 | 宽度 | 说明 |
|---|---|---|
| 报告面板 | **1000px** | 居中；**页面/面板底色渐变 `#FAFAFF`**（淡紫白），头部渐变自顶部融入此底色、无缝拼接 |
| 头部 TitleBar 内容 | 952px | 左右内边距 **24px**；背景为紫色渐变→`#FAFAFF`（与面板底色拼接） |
| 正文区 | 952px | 左右外边距 **24px**（24 + 952 + 24 = 1000） |
| **正文内容列** | **912px** | 模块/分割线/图表对齐此列；距面板边 **44px**（= 24 外边距 + 20 卡片内边距） |
| 洞察摘要卡片 | 952px（内容 912） | **纯白卡片，无描边**；标题「洞察摘要」用 AI 渐变文字 |
| 模块大白卡 | 952px（内容 912） | 洞察摘要下方的**所有模块（核心数据/趋势/流量/转化/总结…）统一包裹在一张纯白卡片内**，卡内用 1px 分割线分隔各一级模块 |

> **卡片描边规则**：洞察摘要卡与模块大白卡均为**纯白底、无边框**（仅靠白底与 `#FAFAFF` 页面底色的色差区分）；卡内的 KPI 卡、表格等子元素可保留自身描边。免责页脚在白卡**之外**，落在 `#FAFAFF` 底色上。

**圆角**：卡片 / 面板 / 图表块 = **8px**（`--jd-radius-xxl`）；标签 / 按钮 = **4px**（`--jd-radius-xs`）。

**间距标度**（`--jd-spacing-*`）：**8 / 12 / 16 / 20 / 24**。

## 二、纵向结构（从上到下）

```
报告面板 (1000)
├─ ① 头部 TitleBar            → 见 Header.md（含装饰图，区高 ~126）
├─ ② 洞察摘要卡片             → 见 全局摘要卡片.md（白卡，紧接头部）
├─ ③ 一级模块 × N             → 模块标题 + 内容
│    ├─ 可含二级模块 2.1 / 2.2
│    └─ 一级模块之间用 1px 分割线分隔
├─ ④ 数据来源模块
└─ ⑤ AI 免责声明页脚          → 见 页脚.md
```

| 衔接 | 间距 |
|---|---|
| 头部 → 正文区 | 头部区高约 126px，正文紧随 |
| 洞察摘要 → 首个一级模块 | 16px |
| 一级模块 → 分割线 | 24px |
| 分割线 → 下一个一级模块 | 24px |
| 模块 → 数据来源 → 页脚 | 各 16px 左右 |

## 三、模块内部结构

**一级模块**
- 模块标题：`一、模块标题`，**20 / 30 Semibold `text-400`**，带中文序号「一、二、三、…」。
- 标题 → 内容间距 16px。
- 内容可为：正文段落 / 诊断条 / 图表块 / 表格 / 建议卡。

**二级模块**
- 二级标题：`2.1 / 2.2 二级模块标题`，**18 / 28 Semibold `text-400`**，序号「2.1」。
- 二级标题上方间距 24px（同一级模块内），标题 → 内容 16px。

**诊断条（可选）**
- 位于模块标题下方，紫蓝渐变「诊断 / 洞察」条：底 `linear-gradient(225deg,#FAF6FF,#F2F5FF 70%,#F1F4FF)`，左侧 AI 星标图标 + 结论文字。

**图表块**
- 结构：`图表名称`（14–16px 标题）+ 12px + 图表区。
- 图表区：圆角 8，内边距 16；右上可挂 全屏 / 下载 / 收藏 图标、或 Tabs 切换（默认隐藏，按需开启）。

**正文段落**
- 正文 14 / 22 Regular `text-300`；段内行间距 12px，段间 24px（见《文本规范.md》）。

## 四、间距节奏（纵向，来自布局实测 + 文本规范）

| 场景 | 间距 |
|---|---|
| 一级标题上方 | 28px |
| 二级标题上方 | 24px |
| 三级标题上方 | 16px |
| 标题 → 正文 | 16px |
| 段落内 正文 → 正文 | 12px |
| 一级段落之间 | 24px（+ 1px 分割线，见下） |
| 正文 → 图表卡片 | 16px |
| 图表卡片 → 正文 | 16px |
| 卡片 / 图表内边距 | 16px |

**分割线**：一级模块之间加 1px 分割线，颜色 `--jd-color-line-100`（#E6E7EB），宽度对齐 912 内容列，上下各留白 24px。

## 五、令牌速查

| 用途 | 令牌 / 值 |
|---|---|
| 面板底 / 页面底 | `surface-200` #FFFFFF / `surface-100` #F2F3F7 |
| 卡片 / 图表底 | `surface-200` #FFFFFF；浅灰块 `fill-100` #F7F8FA |
| 分割线 | `line-100` #E6E7EB |
| 圆角 | 卡片 8（`radius-xxl`）/ 标签按钮 4（`radius-xs`） |
| 间距 | 8 / 12 / 16 / 20 / 24（`spacing-*`） |
| 内容列宽 | 912px（左右各 44px） |

## 六、落地约定

- 所有一级/二级模块、图表、表格、分割线**统一对齐 912 内容列**。
- 一级模块之间**必加 1px 分割线**分隔；同一级模块内的二级模块之间不用分割线，靠间距区分。
- 图表、表格、建议区**统一卡片化**（圆角 8、内边距 16、白底）。
- 颜色、字体、字号、间距**全部引用令牌与《文本规范.md》**，不手写散值。
- 头部、洞察摘要、页脚复用对应组件文档，本布局只负责它们的**排布顺序与间距**。

> ⚠️ 与存量报告的差异：本 Figma 布局的正文内容列为 **912px（左右 44px）**；我们早期几份报告用的是 **936px（左右 32px）**。以本规范 912 为准，后续统一。
