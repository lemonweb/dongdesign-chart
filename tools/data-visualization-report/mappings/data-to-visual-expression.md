# mappings/data-to-visual-expression · 数据 → 视觉表达

把数据的**语义**稳定映射为视觉编码（颜色 / 符号 / 图形），保证同类数据表达一致。

## 一、涨跌 / 变化值

| 语义 | 编码 |
|---|---|
| 上涨（正向变化） | 对外=红 `trend-up`；对内=绿（产品开关，见 visual-style-rules） |
| 下跌（负向变化） | 对外=绿 `trend-down`；对内=红 |
| 持平 | 灰 `trend-even` #B0B2B8 |
| 表达符号 | 一律 `+ / -`；**指标卡变化值禁用升降箭头**（行动建议的预估提升可用 `↗`） |

## 二、状态 / 质量

| 语义 | 圆点色 |
|---|---|
| 进行中 / 信息 | info 蓝 #3768FA |
| 待审核 / 需关注 | warning 橙 #F08433 |
| 已通过 / 良好 | success 绿 #00B26F |
| 已驳回 / 危险 | error 红 #F33B50 |
| 已取消 / 普通 | 灰 text-100 #B0B2B8 |

KPI 卡状态提示底色：危险 #FCEDED / 良好 #E7F3F0 / 警示 #FCEFE0。

## 三、多系列（分类）

- 折线/柱/饼/环等**多系列**按 `echarts-theme.json` 的 **12 色分类色板**顺序取色（item-1 起）；不写死颜色。
- 单系列默认 item-1（品牌蓝 #3365F7）。
- 漏斗等单指标递减：同色系（品牌蓝）由深到浅。

## 四、贡献 / 拆解

- **维度贡献瀑布**：负向贡献红 / 正向贡献绿 / 两端整体灰（随产品开关）。
- **公式拆解**（成交=UV×转化率×客单价）：用**公式指标卡**，不用瀑布。

## 五、数字格式

- 大数缩写（万/亿）；单位用 `.u` 小字号（14px）与数字基线对齐；KPI 主数值用京东正黑体、单位用正文字体。
- 行内数字用 Roboto；同组保持单位、精度、千分位一致；空值明确占位不伪造 0。
