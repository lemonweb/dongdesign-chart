# 图表复杂度分级 Complexity Level

## 1. 文档定位

本文定义图表复杂度分级，用于帮助 AI 在图表选型、看板规划、代码生成和审查时判断某个图表方案是否过度复杂。

复杂度不是图表“高级程度”的评价，而是用户理解成本、数据要求、交互要求和实现维护成本的综合判断。

## 2. 分级目标

AI 在输出图表方案时，应说明图表复杂度：

```txt
complexity_level: L1 / L2 / L3 / L4
complexity_reason: 为什么需要这个复杂度
fallback_chart: 更简单的替代方案
```

当简单图表足以回答业务问题时，不得使用复杂图表。

## 3. 复杂度等级

| 等级 | 定义 | 用户理解成本 | 常见图表 |
|---|---|---|---|
| L1 基础读数 | 单指标、单维度或少量类别，可直接读取 | 低 | 指标卡、柱状图、折线图、饼图、迷你图 |
| L2 结构比较 | 多系列、多维度或需要辅助说明 | 中 | 分组柱状图、堆叠图、面积图、雷达图、直方图 |
| L3 关系探索 | 需要理解变量关系、分布、流转、层级或多编码 | 较高 | 散点图、气泡图、热力图、漏斗图、双轴图 |
| L4 复杂分析 | 需要路径、网络、多层级、交互探索或强业务口径 | 高 | 桑基图、透视图、复杂看板、多图联动 |

## 4. 判断维度

复杂度由以下维度共同决定：

| 维度   | 低复杂度          | 高复杂度                    |
| ---- | ------------- | ----------------------- |
| 字段数量 | 1 个维度 + 1 个指标 | 多维度、多指标、多层级或流向字段        |
| 视觉编码 | 位置、长度、角度等单一编码 | 位置、颜色、大小、形状、路径同时编码      |
| 交互依赖 | 静态即可读         | 依赖筛选、下钻、联动、缩放或 Tooltip  |
| 业务口径 | 指标含义直观        | 需要目标、阈值、归因、路径或规则解释      |
| 误读风险 | 低             | 双轴、面积、气泡、相关性、流量宽度等高误读风险 |
| 实现成本 | 标准图表配置即可      | 需要自定义布局、复杂数据转换或多引擎适配    |

## 5. 图表默认复杂度

| 图表        | 默认等级 | 升级条件                         |
| --------- | ---- | ---------------------------- |
| 指标卡       | L1   | 同时包含目标差、趋势、状态解释时升为 L2        |
| 柱状图 / 条形图 | L1   | 分组、堆叠、Top N + Others 时升为 L2  |
| 折线图       | L1   | 多系列、参考线、活动区间、密集点位时升为 L2      |
| 饼图 / 环图   | L1   | 类目过多或嵌套构成时升为 L2，不建议继续升级      |
| 面积图       | L2   | 堆叠、多系列或累计关系时保持 L2，密集时需降噪     |
| 雷达图       | L2   | 维度过多或对象过多时不建议使用              |
| 直方图       | L2   | 多组分布对比或阈值区间时升为 L3            |
| 散点图       | L3   | 增加气泡大小、颜色分组、趋势线时保持 L3 或升为 L4 |
| 气泡图       | L3   | 同时编码颜色、大小、分组和标签时升为 L4        |
| 热力图       | L3   | 多层级行列、交互钻取或大矩阵时升为 L4         |
| 漏斗图       | L3   | 多漏斗对比、阶段拆解或路径混合时升为 L4        |
| 双轴图       | L3   | 指标口径差异大或趋势强解释时升为 L4 风险       |
| 桑基图       | L4   | 默认 L4，需要流向字段和业务解释            |
| 透视图       | L4   | 默认 L4，需要交互、聚合和层级口径           |

## 6. 选型规则

### 6.1 优先低复杂度

当 L1 图表可以回答问题时，不使用 L2-L4。

示例：

| 用户问题 | 推荐 | 不推荐 |
|---|---|---|
| 谁最高 | 排序柱状图 | 雷达图、气泡图 |
| 最近趋势如何 | 折线图 | 双轴图、桑基图 |
| 各渠道占比 | 环图或堆叠条形图 | 热力图 |
| 是否达标 | 指标卡 + 目标线 | 复杂看板 |

### 6.2 复杂图表必须给理由

使用 L3 / L4 图表时，AI 必须说明：

- 为什么简单图表不足。
- 需要哪些字段。
- 用户需要理解哪些视觉编码。
- 是否存在误读风险。
- 是否有更简单的替代方案。

### 6.3 高复杂度需要交互承接

L3 / L4 图表通常需要：

- Tooltip 解释完整字段。
- 图例或编码说明。
- 筛选、缩放、下钻或高亮。
- 异常、目标、路径等业务口径说明。
- 空状态和数据不足时的兜底方案。

## 7. 降级策略

| 复杂原因 | 降级方式 |
|---|---|
| 系列过多 | 默认展示重点系列，其余进入图例筛选 |
| 类目过多 | Top N + Others、搜索、分页或改用表格 |
| 指标单位不同 | 拆成两个图，避免强行双轴 |
| 关系不明确 | 先用散点或表格验证，再输出结论 |
| 路径过多 | 聚合小流量、只展示主路径 |
| 维度过多 | 分面、小多图、下钻或透视交互 |

## 8. AI 自查清单

- 是否标记了 `complexity_level`？
- 是否说明复杂度理由？
- 是否检查 L1 / L2 图表能否回答问题？
- 使用 L3 / L4 时，是否说明必要字段和误读风险？
- 是否提供更简单的替代方案？
- 是否为复杂图表提供 Tooltip、图例、筛选或下钻承接？
- 是否避免为了视觉效果选择复杂图表？
