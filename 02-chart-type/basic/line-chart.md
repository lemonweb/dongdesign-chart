# 折线图 Line Chart

## 1. 文档定位

本文定义折线图在数据可视化 Wiki 中的选型、数据结构、视觉构成、设计原则和交互规则。

折线图主要用于回答：

> 某个指标是否随时间或有序维度发生变化？变化方向、波动幅度、异常点和多系列趋势差异是什么？

本文不重新定义坐标轴、图例、标签、Tooltip、颜色和字体规范。相关视觉规则应读取 `01-design-language/`。

## 2. 图表定义

折线图是由直角坐标系、数据点和连接线组成的统计图表，常用于表示数值随连续时间间隔或有序类别发生的变化。

在折线图中：

- 横轴通常表示连续时间或有序类别。
- 纵轴表示连续数值指标。
- 数据点表示每个维度位置上的指标值。
- 折线连接相邻数据点，用方向表示正负变化，用斜率表示变化程度。

折线图强调趋势判断，不强调单个数值本身。

## 3. 属性摘要

| 属性 | 规则 |
|---|---|
| 图表类型 | 统计图表 / 基础图表 |
| 图形形态 | 线条，可按需要显示数据点 |
| 核心功能 | 趋势、波动、分布、异常发现、多系列趋势比较 |
| 适合的数据 | 两个连续字段，或一个有序分类字段 + 一个连续数值字段 |
| 数据与图形映射 | 横轴映射时间或有序维度，纵轴映射连续指标，线条连接相邻数据点 |
| 数据体量 | 单条线的数据点应大于 2；同一图中指标线不建议超过 5 条 |

## 4. 适用场景

折线图适用于以下场景：

| 场景 | 使用条件 | 价值 |
|---|---|---|
| 趋势分析 | 指标随时间或连续变量变化 | 判断上升、下降、波动、周期性 |
| 比较趋势 | 多个数据系列共享同一时间或有序维度 | 比较不同类别的变化方向和相对表现 |
| 异常发现 | 数据中可能存在峰值、谷值、断崖、突增或突降 | 帮助用户定位异常点或异常时段 |
| 预测展望 | 有连续历史数据作为基础 | 为趋势外推、目标判断或预警提供依据 |

典型业务问题：

- 销售额最近 30 天是否持续增长？
- 转化率在活动前后是否出现异常波动？
- 不同渠道的访问量趋势是否一致？
- 某个指标是否存在周期性高峰或低谷？

## 5. 不适用场景

| 不适用场景 | 原因 | 替代方向 |
|---|---|---|
| 无序分类比较 | 折线会暗示类别之间存在连续关系 | 柱状图、条形图 |
| 部分与整体构成 | 折线不表达占比和组成结构 | 饼图、环图、堆叠图 |
| 强调单个精确数值 | 折线图重点是变化关系，不是数值卡片 | 指标卡、表格 |
| 系列数量过多 | 多条线容易交叉遮挡，降低趋势识别效率 | 筛选、分面、小多图 |
| 维度顺序不可靠 | 顺序错误会制造虚假的趋势路径 | 先清洗排序，或改用分类比较图 |

## 6. 数据结构

### 6.1 基础字段

| 字段角色 | 是否必需 | 说明 |
|---|---|---|
| 维度字段 | 必需 | 映射到横轴，应为连续时间、连续数值或有序类别 |
| 指标字段 | 必需 | 映射到纵轴，应为连续数值 |
| 分组字段 | 可选 | 用于生成多条折线，表达多系列对比 |
| 标记字段 | 可选 | 用于标记异常点、最大值、最小值、阈值、活动区间或目标线 |

### 6.2 数据要求

- 单条折线至少需要 3 个数据点。
- 横轴必须有明确顺序，不得使用无序分类直接连线。
- 纵轴应保持同一度量口径，不同单位指标不应强行放在同一纵轴。
- 多系列折线应共享同一横轴维度。
- 同一图中的折线数量不建议超过 5 条。

### 6.3 数据示例

```json
[
  { "date": "2026-05-01", "metric": "成交金额", "value": 120 },
  { "date": "2026-05-02", "metric": "成交金额", "value": 136 },
  { "date": "2026-05-03", "metric": "成交金额", "value": 118 }
]
```

字段映射：

| 语义 | 字段 |
|---|---|
| 横轴 | `date` |
| 纵轴 | `value` |
| 系列 | `metric` |

## 7. 视觉构成

| 构成元素 | 作用 | 是否必需 |
|---|---|---|
| 横轴 xAxis | 表示时间、连续变量或有序类别 | 必需 |
| 纵轴 yAxis | 表示连续数值指标 | 必需 |
| 数据点 symbol | 表示单个数据位置 | 可选，悬停或关键值时建议显示 |
| 折线 line | 连接相邻数据点，表达趋势路径 | 必需 |
| 图例 legend | 区分多系列折线 | 多系列时必需 |
| 标签 label | 标记最大值、最小值、异常点或关键点 | 可选 |
| Tooltip | 展示悬停点的精确值和上下文 | 建议启用 |
| dataZoom | 控制长时间序列的可视范围 | 数据点密集时启用 |
| 辅助线 / 辅助区间 | 表示阈值、目标值、活动阶段或异常区间 | 按业务需要启用 |

折线图不默认包含面积填充。若需要通过面积强调“分量”或累计量，应判断是否改用面积图。

## 8. 设计原则

### 8.1 控制系列数量

同一图中折线数量应克制。多系列用于比较趋势差异，但系列过多会造成遮挡、交叉和颜色辨识困难。

执行规则：

- 1 条线用于单指标趋势。
- 2-3 条线适合重点对比。
- 4-5 条线应确认用户确实需要同时比较。
- 超过 5 条线时，应优先使用筛选、分面、小多图或默认只展示重点系列。

### 8.2 使用简洁线条

折线应保持轻量、清晰，默认线宽可按组件规范使用 `2.5px`。

执行规则：

- 不使用过粗线条遮盖数据点。
- 不使用过细线条导致趋势不可辨认。
- 不滥用虚线、点划线或复杂线型。
- 虚线仅用于预测值、目标线、对比基准或非实测数据。

### 8.3 保持数据表达清晰

折线图不应给每个点都添加标签。过多标签会破坏趋势阅读。

优先标记：

- 最大值。
- 最小值。
- 异常点。
- 当前值。
- 目标值。
- 与业务事件相关的关键点。

### 8.4 避免过度装饰

折线图应突出数据趋势，不应依赖装饰制造视觉效果。

禁止默认使用：

- 渐变线条。
- 投影。
- 高饱和背景。
- 无业务含义的发光、描边和装饰图形。

### 8.5 准确使用色彩

单系列折线应使用主强调色或业务指定色。多系列折线应使用分类色板，并保证系列之间可区分。

执行规则：

- 单系列不需要图例时，可通过标题或指标说明解释含义。
- 多系列必须提供图例。
- 不同系列颜色应绑定分类色板变量。
- 不用颜色同时表达过多语义，避免用户混淆。

### 8.6 组件还原规格

当折线图用于 VibeDesign / Relay / Figma 设计验证时，应优先还原图表组件库的外观细节，而不是只满足“有折线和坐标轴”。

标准基础折线图建议规格：

| 元素                       | 规则                                                           |
| ------------------------ | ------------------------------------------------------------ |
| plot area                | 不加独立卡片边框，不使用额外背景色；通过坐标轴和网格定义绘图区                              |
| 折线                       | 默认平滑曲线，线宽 `2.5px`，端点圆角，绑定 `分类色板/item-*`                      |
| 默认 symbol                | 非 hover 状态不铺满所有点；只在 hover 或关键点显示                             |
| hover symbol / markPoint | `8 × 8px` 圆形，系列色填充，`1px` 白色描边                                |
| X 轴                      | 底部 `1px` 轴线，时间标签抽样显示                                         |
| Y 轴                      | 不默认显示左轴实线；默认从 `0` 起步，仅显示刻度标签和水平网格                            |
| 轴标签                      | 固定 `.textSmall 12/18 400`；生成时必须显式设置，禁止回退为 `16px`             |
| 网格线                      | 约 `5` 条，线性等距，浅色虚线或极弱实线，弱于折线                                  |
| 图例                       | 右上角水平线型图例，shape 为 `10 × 2px` 线段                              |
| hover 指示线                | 垂直 `1px` 实线，绑定 `提示/tooltipAxisColor`，不得绘制为虚线                 |
| Tooltip                  | axis trigger，显示垂直实线指示线、hover 点和带阴影浮层                         |
| Tooltip 浮层               | `4px` 圆角，`12px 8px` padding，使用原生投影 `0 4 16 rgba(0,0,0,0.16)` |

如果目标工具无法复用真实图表组件，应至少按上述规格绘制；如果只能用 SVG 合成，应在验证清单中说明“当前是视觉还原稿，不是组件实例”。

## 9. 交互规则

| 交互 | 启用条件 | 系统反馈 |
|---|---|---|
| 悬停 Tooltip | 需要读取某个点的精确值 | 显示该点数值、时间、系列名和必要上下文 |
| 辅助线 | 悬停定位困难或横轴密集 | 显示垂直辅助线，帮助定位当前维度 |
| 点高亮 | 用户悬停或选中数据点 | 显示数据点、增强当前系列或当前点 |
| dataZoom | 时间跨度长、数据点密集 | 支持拖拽、滚轮或滑块缩放和平移 |
| 图例筛选 | 多系列折线 | 支持隐藏、显示或高亮某个系列 |
| 标记 | 存在阈值、目标、活动阶段或异常区间 | 显示辅助线、背景区间、关键标签 |

交互应帮助用户主动探索趋势、波动和关键点，不应把图表变成复杂操作面板。

### 9.1 Hover 展示顺序

折线图 hover 状态应按以下层级生成：

```txt
垂直 axisPointer
→ hover symbol
→ Tooltip container
→ Tooltip item icon / label / value
```

要求：

- `axisPointer` 使用 `提示/tooltipAxisColor`。
- `axisPointer` 必须是单根垂直 `1px` 实线；虚线只允许用于网格线、预测线、目标线等明确语义，不得用于普通 hover 辅助线。
- hover 点必须覆盖在线条上方，尺寸固定为 `8 × 8px`，描边固定 `1px`。
- Tooltip 避开 hover 点和折线路径。
- Tooltip icon 形态与图例一致，折线使用线段 icon。
- Tooltip shadow 在 Figma / Relay 中应使用原生 drop shadow / effects，不依赖 SVG filter。
- 多系列 hover 时，同一时间点的所有系列值按图例顺序展示。
- 多系列 Tooltip 内容必须按通用 Tooltip 规范使用两列结构：左侧 icon + label，右侧 value；系列行高 `18px`，行间距 `0px`，note 距系列列表 `4px`。
- Tooltip title 与 value 必须使用 `600` 字重，label 与 note 使用 `400` 字重。
- 在 Figma / Relay 生成时，Tooltip value 列必须按实际文本宽度右对齐；不得把 value 文本的 x 坐标误当作右边界，导致数值列被裁切。

## 11. G2 / ECharts 映射

| 语义 | G2 | ECharts |
|---|---|---|
| 图表类型 | `line()` / line mark | `series.type = "line"` |
| 横轴维度 | `encode("x", dimensionField)` | `xAxis` + `encode.x` |
| 纵轴指标 | `encode("y", valueField)` | `yAxis` + `encode.y` |
| 多系列 | `encode("color", seriesField)` | 多个 `series` 或 `dataset` + `encode` |
| 数据点 | `point` / `shape` / style | `showSymbol`、`symbol`、`symbolSize` |
| Tooltip | `interaction("tooltip")` | `tooltip.trigger = "axis"` 或 `"item"` |
| 图例 | `legend` | `legend` |
| 缩放 | brush / slider 交互 | `dataZoom` |
| 标记 | annotation | `markPoint`、`markLine`、`markArea` |

AI 生成折线图设计或代码时，应输出：

```txt

分析目标：趋势 / 波动 / 异常 / 多系列比较
字段映射：
  - 横轴：
  - 纵轴：
  - 系列：
视觉规则：
  - 线条：
  - 坐标轴：
  - 图例：
  - 标签：
交互规则：
  - Tooltip：
  - 缩放：
  - 标记：
风险与限制：
  - 是否存在无序分类？
  - 系列数量是否超过 5？
  - 是否需要替代图表？
```

## 13. 禁止项

AI 不得：

- 将无序分类字段直接连成折线。
- 在同一图中默认展示超过 5 条折线。
- 为每个数据点默认显示标签。
- 用折线图表达部分与整体的构成比例。
- 用折线图替代指标卡来强调单个精确数值。
- 在不同单位指标之间强行共用同一纵轴。
- 默认添加面积填充、渐变、投影或无业务含义的装饰。
- 忽略横轴排序。
- 在设计验证场景中用普通折线外观替代组件库折线细节。
- 用圆点或色块图例替代折线图线型图例。
- 用无阴影、无内部结构的白框替代标准 Tooltip。
- 为了减少绘制成本省略 hover 指示线和 hover symbol。
- 把 hover 指示线绘制成虚线或用大量小矩形拼接。
- 把 Tooltip 系列之间拉出额外大间距，或让 value 列与 label 列错位 / 裁切。

## 14. AI 自查清单

生成折线图前检查：

- 横轴是否为连续时间、连续变量或有序类别？
- 纵轴是否为连续数值指标？
- 数据点数量是否大于 2？
- 多系列是否共享同一横轴？
- 系列数量是否不超过 5？
- 是否存在需要标记的最大值、最小值、异常点或阈值？
- Tooltip 是否能展示精确值和必要上下文？
- hover 指示线是否为单根垂直 `1px` 实线？
- Tooltip 是否使用 `label_column + value_column` 两列结构，系列行间距为 `0px`，value 未裁切？
- Tooltip title 和 value 是否为 `600` 字重？
- 数据点密集时是否启用 dataZoom 或采样策略？
- 是否引用坐标轴、图例、标签、Tooltip 的上游规范？
- 是否没有把面积图规则误用为折线图默认规则？
