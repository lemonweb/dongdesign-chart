# 面积图 Area Chart

## 1. 文档定位

本文定义面积图在数据可视化 Wiki 中的选型、数据结构、视觉构成、设计原则、交互规则和代码映射。

面积图主要用于回答：

> 某个指标是否随时间或有序维度发生变化？变化过程中整体量级是否扩大或收缩？多系列之间的累计量、占比或贡献结构是否发生变化？

面积图的大部分形态、坐标轴和交互规则继承 `line-chart.md`。本文只补充面积图相对折线图的差异点：面积填充、量感表达、累计关系、堆叠关系和透明度控制。

本文不重新定义坐标轴、图例、标签、Tooltip、颜色和字体规范。相关视觉规则应读取 `../01-design-language/`。

## 2. 图表定义

面积图是由直角坐标系、折线和折线下方的面积填充组成的统计图表，常用于表示数值随连续时间间隔或有序类别变化时的量级变化。

在面积图中：

- 横轴通常表示连续时间、连续变量或有序类别。
- 纵轴表示连续数值指标。
- 折线表示指标的变化路径。
- 面积填充表示折线与基线之间的数量范围。
- 多系列面积图可通过堆叠表达总量和各部分贡献。

面积图既表达趋势，也强调“量感”。如果只需要判断走势，应优先使用折线图；如果需要强调累计规模、总体体量或构成变化，才使用面积图。

## 3. 属性摘要

| 属性 | 规则 |
|---|---|
| 图表类型 | 统计图表 / 基础图表 |
| 图形形态 | 折线 + 面积填充 |
| 核心功能 | 趋势、量级变化、累计量、结构贡献、占比变化 |
| 适合的数据 | 两个连续字段，或一个有序分类字段 + 一个连续数值字段 |
| 数据与图形映射 | 横轴映射时间或有序维度，纵轴映射连续指标，面积表达指标与基线之间的数量范围 |
| 数据体量 | 单个系列数据点应大于 2；普通多系列面积图不建议超过 3 个系列；堆叠面积图不建议超过 5 个系列 |

## 4. 与折线图的关系

面积图和折线图共享以下规则：

| 共享项 | 继承规则 |
|---|---|
| 横轴 | 必须是连续时间、连续变量或有序类别 |
| 纵轴 | 必须是连续数值指标 |
| 折线 | 表示趋势路径，默认线宽可继承折线图的 `2.5px` |
| 数据点 | 默认不铺满所有点，仅在 hover 或关键点显示 |
| Tooltip | 继承折线图的 axis trigger、垂直 hover 指示线和两列内容结构 |
| dataZoom | 长时间序列或数据点密集时启用 |
| 图例 | 多系列时必须提供图例，并支持筛选或高亮 |

面积图相对折线图的差异：

| 差异项 | 面积图规则 |
|---|---|
| 视觉重点 | 从“线的变化”扩展为“趋势 + 体量” |
| 图形构成 | 必须包含面积填充 |
| 基线含义 | 面积必须有明确基线，通常从 `0` 开始 |
| 多系列表达 | 更适合表达累计量、结构贡献或占比变化 |
| 视觉风险 | 面积遮挡、透明度叠加和堆叠顺序会影响判断 |

## 5. 适用场景

面积图适用于以下场景：

| 场景 | 使用条件 | 价值 |
|---|---|---|
| 体量趋势 | 指标随时间变化，且量级本身重要 | 同时判断趋势方向和规模变化 |
| 累计变化 | 关注总量随时间增长或收缩 | 强化整体量级和增长区间 |
| 构成变化 | 多个子项构成一个总量 | 观察各部分对整体的贡献 |
| 占比变化 | 各部分之和固定为 100% | 观察结构比例如何随时间变化 |
| 阶段差异 | 不同时间段的量级差异明显 | 通过面积大小增强阶段对比 |

典型业务问题：

- 最近 30 天成交金额的总体规模是否扩大？
- 不同渠道对总访问量的贡献如何变化？
- 各品类销售额在整体中的占比是否发生迁移？
- 活动期前后，总体流量和主要来源结构是否有变化？

## 6. 不适用场景

| 不适用场景 | 原因 | 替代方向 |
|---|---|---|
| 只需要比较趋势方向 | 面积填充会增加视觉重量，可能干扰趋势判断 | 折线图 |
| 无序分类比较 | 面积会暗示连续变化和累计关系 | 柱状图、条形图 |
| 精确比较多个系列 | 面积遮挡和叠加会降低精确读数能力 | 折线图、分面图 |
| 系列数量过多 | 多层面积会造成遮挡、混色和阅读困难 | 筛选、小多图、堆叠柱状图 |
| 指标不应累加 | 堆叠面积会暗示总量关系 | 折线图或并列对比图 |
| 存在大量负值 | 面积跨越基线后含义复杂，容易误读 | 折线图、正负柱状图 |

## 7. 数据结构

### 7.1 基础字段

| 字段角色 | 是否必需 | 说明 |
|---|---|---|
| 维度字段 | 必需 | 映射到横轴，应为连续时间、连续变量或有序类别 |
| 指标字段 | 必需 | 映射到纵轴，应为连续数值 |
| 分组字段 | 可选 | 用于生成多系列面积、堆叠面积或百分比堆叠面积 |
| 标记字段 | 可选 | 用于标记最大值、最小值、异常点、阈值、活动区间或目标线 |
| 总量字段 | 可选 | 堆叠面积中可显式提供总量，用于校验各子项之和 |

### 7.2 数据要求

- 单个系列至少需要 3 个数据点。
- 横轴必须有明确顺序，不得使用无序分类直接绘制面积。
- 面积图默认要求纵轴从 `0` 开始，确保面积大小有可解释的基线。
- 多系列面积图应共享同一横轴维度。
- 普通多系列面积图不建议超过 3 个系列。
- 堆叠面积图不建议超过 5 个系列。
- 堆叠面积图中的系列必须可以相加，且总量有业务意义。
- 百分比堆叠面积图中，每个横轴位置的系列合计必须为 `100%`。

### 7.3 数据示例

```json
[
  { "date": "2026-05-01", "channel": "自然流量", "value": 120 },
  { "date": "2026-05-01", "channel": "付费流量", "value": 80 },
  { "date": "2026-05-02", "channel": "自然流量", "value": 136 },
  { "date": "2026-05-02", "channel": "付费流量", "value": 96 },
  { "date": "2026-05-03", "channel": "自然流量", "value": 118 },
  { "date": "2026-05-03", "channel": "付费流量", "value": 104 }
]
```

字段映射：

| 语义 | 字段 |
|---|---|
| 横轴 | `date` |
| 纵轴 | `value` |
| 系列 | `channel` |

## 8. 视觉构成

| 构成元素 | 作用 | 是否必需 |
|---|---|---|
| 横轴 xAxis | 表示时间、连续变量或有序类别 | 必需 |
| 纵轴 yAxis | 表示连续数值指标 | 必需 |
| 折线 line | 表示趋势路径和面积上边界 | 必需 |
| 面积 area | 表示指标与基线之间的数量范围 | 必需 |
| 数据点 symbol | 表示单个数据位置 | 可选，悬停或关键值时建议显示 |
| 图例 legend | 区分多系列面积 | 多系列时必需 |
| 标签 label | 标记最大值、最小值、异常点或关键点 | 可选 |
| Tooltip | 展示悬停点的精确值和上下文 | 建议启用 |
| dataZoom | 控制长时间序列的可视范围 | 数据点密集时启用 |
| 辅助线 / 辅助区间 | 表示阈值、目标值、活动阶段或异常区间 | 按业务需要启用 |

面积图必须包含面积填充。如果没有面积填充，应归类为折线图。

## 9. 图表类型

### 9.1 基础面积图

基础面积图展示单个指标随时间或有序维度变化时的量级变化。

适用条件：

- 只有一个指标系列。
- 用户既关注趋势，也关注指标规模。
- 纵轴可从 `0` 开始，面积含义明确。

### 9.2 多系列面积图

多系列面积图将多个系列以不同颜色的面积展示在同一坐标系中。

适用条件：

- 系列数量较少，建议不超过 3 个。
- 系列之间遮挡较轻。
- 用户关注不同系列的量级和趋势差异。

使用多系列面积图时，应通过透明度降低遮挡风险；如果遮挡明显，应改用折线图或分面面积图。

### 9.3 堆叠面积图

堆叠面积图将多个系列沿纵轴方向累加，用于表达总量变化和各系列对总量的贡献。

适用条件：

- 各系列可以相加。
- 总量具有业务意义。
- 用户需要同时观察整体规模和组成变化。

堆叠面积图不适合精确比较非底部系列的绝对变化，因为非底部系列的基线会随下方系列变化。

### 9.4 百分比堆叠面积图

百分比堆叠面积图将每个横轴位置的总量归一化为 `100%`，用于表达结构占比随时间变化。

适用条件：

- 用户关注结构比例，而不是绝对数值。
- 每个时间点的系列合计为 `100%`。
- Tooltip 需要同时显示占比和原始值，避免丢失规模信息。

## 10. 设计原则

### 10.1 明确面积的语义

面积图中的填充区域不是装饰，而是数据表达的一部分。生成面积图前必须判断填充区域表达的是绝对量、累计量还是占比。

执行规则：

- 单系列面积表示该指标与基线之间的数量范围。
- 堆叠面积表示总量及各部分贡献。
- 百分比堆叠面积表示结构占比变化。
- 如果面积没有明确含义，应改用折线图。

### 10.2 保持基线可信

面积大小依赖基线。默认情况下，面积图纵轴应从 `0` 开始。

执行规则：

- 基础面积图默认使用 `0` 作为填充基线。
- 堆叠面积图必须从 `0` 开始累计。
- 如果因业务原因不从 `0` 开始，必须在图表说明中显式标注。
- 不得通过截断纵轴夸大面积变化。

### 10.3 控制系列数量

面积图比折线图更容易出现遮挡和混色，因此系列数量应更克制。

执行规则：

- 基础面积图适合 1 个系列。
- 普通多系列面积图建议 2-3 个系列。
- 堆叠面积图最多不建议超过 5 个系列。
- 超过 5 个系列时，应使用筛选、分面、小多图或默认只展示重点系列。

### 10.4 使用低干扰填充

面积填充应弱于折线，不能遮挡坐标轴、网格线、标签和 Tooltip。

执行规则：

- 折线颜色绑定分类色板，面积填充使用同色低透明度。
- 单系列面积透明度建议控制在 `12%-24%`。
- 多系列面积透明度建议控制在 `10%-18%`。
- 堆叠面积可使用实色或较高透明度，但必须保持系列边界清晰。
- 面积填充不使用高饱和渐变、投影或发光效果。

### 10.5 准确使用色彩

面积图颜色既要表达系列，也要避免视觉混色。

执行规则：

- 单系列使用主强调色或业务指定色。
- 多系列使用分类色板，并保证线条、面积、图例、Tooltip icon 颜色一致。
- 面积填充颜色应由线条色派生，不额外引入无语义颜色。
- 叠加面积不可通过随机透明度制造不稳定的混合色。

### 10.6 保持数据表达清晰

面积图不应给每个点都添加标签。标签过多会破坏趋势和面积阅读。

优先标记：

- 最大值。
- 最小值。
- 当前值。
- 异常点。
- 目标值。
- 与业务事件相关的关键点。

### 10.7 组件还原规格

当面积图用于 VibeDesign / Relay / Figma 设计验证时，应优先继承折线图组件规格，并补充面积填充规则。

标准基础面积图建议规格：

| 元素 | 规则 |
|---|---|
| plot area | 不加独立卡片边框，不使用额外背景色；通过坐标轴、网格和面积定义绘图区 |
| 折线 | 默认平滑曲线，线宽 `2.5px`，端点圆角，绑定 `分类色板/item-*` |
| 面积填充 | 与折线同色，使用低透明度填充；填充到 `0` 基线 |
| 默认 symbol | 非 hover 状态不铺满所有点；只在 hover 或关键点显示 |
| hover symbol / markPoint | `8 × 8px` 圆形，系列色填充，`1px` 白色描边 |
| X 轴 | 底部 `1px` 轴线，时间标签抽样显示 |
| Y 轴 | 默认从 `0` 起步；仅显示刻度标签和水平网格 |
| 轴标签 | 固定 `.textSmall 12/18 400`；生成时必须显式设置，禁止回退为 `16px` |
| 网格线 | 约 `5` 条，线性等距，浅色虚线或极弱实线，弱于折线和面积 |
| 图例 | 右上角水平线型图例；面积图仍使用线型图例，不使用大色块替代 |
| hover 指示线 | 垂直 `1px` 实线，绑定 `提示/tooltipAxisColor`，不得绘制为虚线 |
| Tooltip | axis trigger，显示垂直实线指示线、hover 点和带阴影浮层 |
| Tooltip 浮层 | `4px` 圆角，`12px 8px` padding，使用原生投影 `0 4 16 rgba(0,0,0,0.16)` |

如果目标工具无法复用真实图表组件，应至少按上述规格绘制；如果只能用 SVG 合成，应在验证清单中说明“当前是视觉还原稿，不是组件实例”。

## 11. 交互规则

| 交互 | 启用条件 | 系统反馈 |
|---|---|---|
| 悬停 Tooltip | 需要读取某个点的精确值 | 显示该点数值、时间、系列名和必要上下文 |
| 辅助线 | 悬停定位困难或横轴密集 | 显示垂直辅助线，帮助定位当前维度 |
| 点高亮 | 用户悬停或选中数据点 | 显示数据点、增强当前系列或当前点 |
| 面积高亮 | 用户悬停或选中某个系列 | 当前系列线条和面积增强，其他系列弱化 |
| dataZoom | 时间跨度长、数据点密集 | 支持拖拽、滚轮或滑块缩放和平移 |
| 图例筛选 | 多系列面积图 | 支持隐藏、显示或高亮某个系列 |
| 标记 | 存在阈值、目标、活动阶段或异常区间 | 显示辅助线、背景区间、关键标签 |

交互应帮助用户主动探索趋势、量级和构成变化，不应把面积图变成复杂操作面板。

### 11.1 Hover 展示顺序

面积图 hover 状态应按以下层级生成：

```txt
垂直 axisPointer
→ hover symbol
→ 当前系列线条 / 面积高亮
→ Tooltip container
→ Tooltip item icon / label / value
```

要求：

- `axisPointer` 使用 `提示/tooltipAxisColor`。
- `axisPointer` 必须是单根垂直 `1px` 实线；虚线只允许用于网格线、预测线、目标线等明确语义。
- hover 点必须覆盖在线条上方，尺寸固定为 `8 × 8px`，描边固定 `1px`。
- Tooltip 避开 hover 点和面积边界。
- Tooltip icon 形态与图例一致，面积图使用线段 icon，不使用大面积色块。
- 多系列 hover 时，同一时间点的所有系列值按图例顺序展示。
- 堆叠面积图 Tooltip 应额外显示总量。
- 百分比堆叠面积图 Tooltip 应同时显示占比和原始值。

## 13. G2 / ECharts 映射

| 语义 | G2 | ECharts |
|---|---|---|
| 图表类型 | `area()` / area mark | `series.type = "line"` + `areaStyle` |
| 横轴维度 | `encode("x", dimensionField)` | `xAxis` + `encode.x` |
| 纵轴指标 | `encode("y", valueField)` | `yAxis` + `encode.y` |
| 多系列 | `encode("color", seriesField)` | 多个 `series` 或 `dataset` + `encode` |
| 面积填充 | area style / shape | `areaStyle` |
| 堆叠面积 | `transform({ type: "stackY" })` | 多个 series 设置相同 `stack` |
| 百分比堆叠 | normalize / percent transform | 数据预处理为占比 + `stack` |
| 数据点 | point / shape / style | `showSymbol`、`symbol`、`symbolSize` |
| Tooltip | `interaction("tooltip")` | `tooltip.trigger = "axis"` |
| 图例 | `legend` | `legend` |
| 缩放 | brush / slider 交互 | `dataZoom` |
| 标记 | annotation | `markPoint`、`markLine`、`markArea` |

### 13.1 G2 示例

基础面积图：

```js
chart
  .area()
  .data(data)
  .encode('x', 'date')
  .encode('y', 'value')
  .encode('color', 'metric')
  .style('fillOpacity', 0.18);

chart
  .line()
  .data(data)
  .encode('x', 'date')
  .encode('y', 'value')
  .encode('color', 'metric')
  .style('lineWidth', 2.5);
```

堆叠面积图：

```js
chart
  .area()
  .data(data)
  .encode('x', 'date')
  .encode('y', 'value')
  .encode('color', 'channel')
  .transform({ type: 'stackY' });
```

### 13.2 ECharts 示例

基础面积图：

```js
option = {
  tooltip: { trigger: 'axis' },
  legend: {},
  xAxis: { type: 'category', data: dates },
  yAxis: { type: 'value', min: 0 },
  series: [
    {
      name: '成交金额',
      type: 'line',
      smooth: true,
      data: values,
      showSymbol: false,
      symbolSize: 8,
      lineStyle: { width: 2.5 },
      areaStyle: { opacity: 0.18 }
```

## 15. 禁止项

AI 不得：

- 在没有面积语义时把折线图加填充伪装成面积图。
- 将无序分类字段直接绘制为面积图。
- 在同一普通面积图中默认展示超过 3 个重叠系列。
- 在堆叠面积图中叠加不能相加的指标。
- 在百分比堆叠面积图中只显示占比，不保留原始值上下文。
- 默认给每个数据点显示标签。
- 使用高饱和渐变、投影、发光或无业务含义的装饰面积。
- 使用大面积不透明填充遮挡网格线、轴标签或其他系列。
- 截断纵轴后仍用面积大小暗示总量变化。
- 用面积图替代柱状图表达无序分类比较。
- 用面积图替代指标卡强调单个精确数值。
- 在设计验证场景中只画一条线而省略面积填充。
- 用圆点或大色块图例替代面积图的线型图例。
- 把 hover 指示线绘制成虚线或省略 hover symbol。
- 让 Tooltip value 列错位、裁切或丢失堆叠总量信息。

## 16. AI 自查清单

生成面积图前检查：

- 横轴是否为连续时间、连续变量或有序类别？
- 纵轴是否为连续数值指标？
- 数据点数量是否大于 2？
- 面积填充是否有明确语义？
- 纵轴是否从 `0` 开始？
- 是否明确选择基础面积、多系列面积、堆叠面积或百分比堆叠面积？
- 普通多系列面积是否不超过 3 个系列？
- 堆叠面积是否不超过 5 个系列？
- 堆叠系列是否可以相加？
- 百分比堆叠是否每个横轴位置合计为 `100%`？
- 面积透明度是否低于线条视觉权重？
- Tooltip 是否能展示精确值、系列名和必要上下文？
- 堆叠面积 Tooltip 是否显示总量？
- hover 指示线是否为单根垂直 `1px` 实线？
- 数据点密集时是否启用 dataZoom 或采样策略？
- 是否引用坐标轴、图例、标签、Tooltip 的上游规范？
- 是否没有把面积图规则误用为折线图默认规则？
