# 饼图 Pie Chart

## 1. 文档定位

本文定义饼图在数据可视化 Wiki 中的选型、数据结构、视觉构成、设计原则和交互规则。

饼图主要用于回答：

> 一个整体由哪些部分组成？每个分类在整体中占多少比例？是否需要突出某个主要分类？

本文不重新定义颜色、图例、标签、Tooltip 和字体规范。相关视觉规则应读取 `01-design-language/`。

## 2. 图表定义

饼图用于表示不同分类在整体中的占比关系。

饼图将一个圆形按照分类占比切分为多个扇区，整个圆形代表数据总量，每个扇区表示一个分类占总体的比例，所有扇区占比之和应等于 100%。

饼图通过扇区角度和面积表达占比大小，适合观察整体构成，不适合精确比较多个接近数值。

## 3. 属性摘要

| 属性 | 规则 |
|---|---|
| 图表类型 | 统计图表 / 基础图表 |
| 图形形态 | 圆形、扇区、可选环形 |
| 核心功能 | 组成、占比、简单分类对比 |
| 适合的数据 | 一个分类字段 + 一个连续数值字段 |
| 数据与图形映射 | 分类字段映射到标签和颜色，数值字段映射到扇区角度 / 面积 |
| 数据体量 | 2-6 个分类为宜，最多不超过 10 个 |

## 4. 适用场景

饼图适用于以下场景：

| 场景 | 使用条件 | 价值 |
|---|---|---|
| 占比分析 | 需要展示某一类别在整体中的比例 | 帮助用户理解部分与整体的关系 |
| 简单分类 | 分类数量较少，且各分类之间差异较明显 | 快速呈现每个分类的占比 |
| 突出重点 | 需要强调某个主要类别或最大分类 | 通过颜色、排序或标签强化重点 |

典型业务问题：

- 各渠道销售额占总销售额的比例是多少？
- 预算在不同成本项之间如何分配？
- 某个核心品类是否占据主要份额？
- 用户来源中哪个渠道贡献最大？

## 5. 不适用场景

| 不适用场景 | 原因 | 替代方向 |
|---|---|---|
| 类别过多 | 扇区过小，颜色和标签难以区分 | 条形图、柱状图、Treemap |
| 趋势变化 | 饼图不表达时间顺序和变化方向 | 折线图、面积图 |
| 精确数值比较 | 人对角度和弧度差异不敏感 | 柱状图、条形图、表格 |
| 多组占比对比 | 多个饼图之间难以比较扇区差异 | 堆叠柱状图、百分比堆叠条形图 |
| 总量不完整 | 所有分类不能构成一个整体 | 分类比较图，不使用饼图 |

## 6. 数据结构

### 6.1 基础字段

| 字段角色 | 是否必需 | 说明 |
|---|---|---|
| 分类字段 | 必需 | 映射到扇区、颜色、标签和图例 |
| 数值字段 | 必需 | 映射到扇区角度和面积 |
| 占比字段 | 可选 | 如果数据已包含百分比，应校验总和是否接近 100% |
| 标记字段 | 可选 | 用于突出重点分类、最大分类或“其他”分类 |

### 6.2 数据要求

- 分类数量至少为 2 个。
- 推荐分类数量为 2-6 个。
- 分类数量最多不超过 10 个。
- 所有分类必须构成同一个整体。
- 数值字段不能为负数。
- 如果存在多个很小的分类，应合并为“其他”。
- “其他”分类应放在最后，通常作为最小扇区处理。

### 6.3 数据示例

```json
[
  { "category": "Technology", "value": 84.48 },
  { "category": "Car", "value": 58.36 },
  { "category": "Airlines", "value": 45.24 },
  { "category": "Energy", "value": 60.64 },
  { "category": "Estate", "value": 56.91 }
]
```

字段映射：

| 语义 | 字段 |
|---|---|
| 分类 | `category` |
| 数值 | `value` |
| 占比 | 由 `value / sum(value)` 计算 |

## 7. 视觉构成

| 构成元素 | 作用 | 是否必需 |
|---|---|---|
| 圆形整体 | 表示 100% 总量 | 必需 |
| 扇区 slice | 表示分类占比 | 必需 |
| 扇区颜色 | 区分类别 | 必需 |
| 分类标签 | 显示分类名称 | 建议启用 |
| 数值标签 | 显示原始值 | 可选 |
| 百分比标签 | 显示占比 | 建议启用 |
| 引导线 | 连接扇区与外置标签 | 标签外置时启用 |
| 图例 legend | 解释颜色与分类关系 | 分类较多或不显示标签时必需 |
| Tooltip | 展示悬停扇区的分类、数值、占比 | 建议启用 |

饼图可以扩展为环形图。环形图允许在中心区域放置总量、核心指标或标题，但不应改变“部分与整体”的表达语义。

## 8. 设计原则

### 8.1 清晰呈现占比

饼图应帮助用户快速识别最大分类、最小分类和主要组成。

执行规则：

- 推荐从最大扇区开始排列。
- 扇区可按顺时针或逆时针依次排列，但同一项目中应保持一致。
- 多个非常小的类别应合并为“其他”。
- “其他”分类应放在最后。

### 8.2 控制分类数量

饼图的分类数量必须克制。

执行规则：

- 2-6 个分类为推荐范围。
- 7-10 个分类需要确认每个分类都必须单独展示。
- 超过 10 个分类不得默认使用饼图。
- 分类过多时，应改用条形图、柱状图或合并长尾分类。
- 图例过多时，应使用分页图例或改用更适合分类比较的图表。

### 8.3 合理使用环形图

环形图适合在保留占比关系的同时，为中心信息留出空间。

执行规则：

- 环形图内径建议占外径的 `60% - 80%`。
- 内径过小会接近普通饼图，中心信息空间不足。
- 内径过大会让扇区过薄，降低扇区面积感知。
- 中心信息只放总量、核心指标或简短标题，不放复杂说明。

### 8.4 合理使用标签

标签用于降低用户在扇区、图例和数值之间来回匹配的成本。

执行规则：

- 分类较少时，可以显示外置标签。
- 标签应优先显示分类名和百分比。
- 是否显示原始数值取决于业务是否需要读数。
- 标签发生重叠时，应减少标签、改用 Tooltip 或使用图例。
- 不应为过小扇区强行显示完整标签。

### 8.5 使用扇区间隔区分类别

当颜色区分不足或扇区相邻关系较密集时，可以使用轻微扇区间隔增强可读性。

执行规则：

- 扇区间隔只用于辅助区分类别。
- 间隔不应大到破坏整体感。
- 间隔样式应保持统一。
- 不用阴影、描边或高饱和装饰替代清晰的分类色。

### 8.6 准确使用色彩

饼图依赖分类色表达类别差异。

执行规则：

- 每个分类应绑定分类色板变量。
- 相邻扇区颜色应有足够区分度。
- 重点分类可以使用强调色，但不得破坏整体分类映射。
- 不用单一颜色表达多个不同分类，除非通过标签和交互补充说明。

## 9. 交互规则

| 交互 | 启用条件 | 系统反馈 |
|---|---|---|
| 悬停 Tooltip | 需要读取精确值或完整分类名 | 显示分类、原始值、百分比 |
| 扇区高亮 | 用户悬停或选中某个分类 | 高亮当前扇区，弱化其他扇区 |
| 图例筛选 | 分类较多或需要聚焦某类 | 支持显示、隐藏或高亮分类 |
| 标签显示 | 分类较少且空间充足 | 显示分类名、百分比，可选原始值 |
| 中心信息 | 使用环形图并需要突出总量 | 在中心显示总量或核心指标 |

饼图交互应服务于“读清占比”和“定位分类”。如果需要复杂筛选、排序或多维分析，应考虑替代图表。

## 11. G2 / ECharts 映射

| 语义 | G2 | ECharts |

| 图表类型 | `interval()` + `coordinate({ type: "theta" })` 或 pie mark | `series.type = "pie"` |
| 分类字段 | `encode("color", categoryField)` | `encode.itemName` / `name` |
| 数值字段 | `encode("y", valueField)` | `encode.value` / `value` |
| 半径 | `radius` | `radius` |

| 标签 | `label` | `label` |
| 引导线 | `labelLine` | `labelLine` |
| Tooltip | `interaction("tooltip")` | `tooltip.trigger = "item"` |
| 图例 | `legend` | `legend` |

## 12. AI 输出格式

AI 生成饼图设计或代码时，应输出：

```txt
图表类型：饼图 / 环形图
分析目标：占比 / 组成 / 重点分类
字段映射：
  - 分类：
  - 数值：
  - 占比：
视觉规则：
  - 扇区排序：
  - 分类色：
  - 标签：
  - 图例：
交互规则：
  - Tooltip：
  - 高亮：
  - 图例筛选：
风险与限制：
  - 分类数量是否超过 10？
  - 是否存在长尾分类需要合并为“其他”？
  - 是否需要替代图表？
```

## 13. 禁止项

AI 不得：

- 用饼图展示时间趋势。
- 用饼图比较大量分类的精确数值。
- 在分类超过 10 个时默认使用饼图。
- 使用负数或不能构成整体的数据生成饼图。
- 将多个不属于同一整体的数据强行合并成饼图。
- 为过小扇区强行显示完整标签。
- 使用过多高饱和颜色导致分类混淆。
- 用三维饼图、阴影或夸张装饰增强视觉效果。
- 在多组占比对比场景中默认堆叠多个饼图。

## 14. AI 自查清单

生成饼图前检查：

- 数据是否表达部分与整体关系？
- 所有分类占比之和是否等于或接近 100%？
- 分类数量是否在 2-6 个推荐范围内？
- 分类数量是否不超过 10 个？
- 是否存在需要合并为“其他”的长尾分类？
- 是否需要展示百分比、原始值或两者都展示？
- 标签是否可能重叠？
- 是否需要图例分页？
- 是否需要使用环形图展示中心总量？
- 如果用户需要精确比较，是否应改用柱状图或条形图？
