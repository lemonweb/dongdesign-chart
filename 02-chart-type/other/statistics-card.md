# 指标卡 Statistics Card

## 1. 文档定位

本文定义指标卡在 dongDesign Chart 中的适用场景、数据结构、视觉构成、布局尺寸、状态表达、分组展示和 AI 输出规则。

指标卡用于展示单个核心指标的当前值、变化、目标差、轻量趋势或状态摘要。它不是完整图表的替代品，而是业务看板中的快速读数入口。

当多个指标需要按同一业务维度归纳展示时，应使用指标卡组。指标卡组负责表达“某个维度下多个指标的不同表现”，不负责替代明细表、完整趋势图或复杂分析面板。

## 2. 图表定义

指标卡通过数字、单位、标题、对比信息、趋势缩略和状态标签，帮助用户快速判断一个指标当前表现。

常见形式：

- 单指标数值卡。
- 数值 + 同比 / 环比变化。
- 数值 + 目标差 / 完成率。
- 数值 + 迷你趋势图。
- 数值 + 状态标签 / 风险提示。
- 指标卡组：分组标题 + 一个或多个指标卡。
- 公式指标卡组：用公式符号展示指标之间的拆解关系。

## 3. 属性摘要

| 属性 | 说明 |
|---|---|
| 图表类型 | 其他 / 指标组件 |
| 图形形态 | 数值卡、数值 + 迷你趋势、指标卡组、公式指标卡组 |
| 核心功能 | 快速读数、状态判断、目标差判断、分组归纳 |
| 适合的数据 | 1 个核心指标，可选对比值、目标值、趋势序列、状态字段；指标卡组可包含多个同层级指标 |
| 数据与图形映射 | 指标映射为主数字；对比值映射为子指标；趋势序列映射为迷你图；分组维度映射为分组标题 |
| 默认复杂度 | L1；包含目标、趋势、状态解释、分组或公式拆解时为 L2 |

## 4. 适用场景

| 场景 | 数据条件 | 图表价值 |
|---|---|---|
| 总览核心指标 | 有明确当前值和单位 | 快速建立业务状态 |
| 同比 / 环比摘要 | 有当前值和对比期值 | 判断变化方向和幅度 |
| 目标达成 | 有当前值和目标值 | 判断差距和完成率 |
| 轻量趋势提示 | 有短时间序列 | 判断走势但不展开分析 |
| 状态提醒 | 有状态规则或阈值 | 标记正常、预警、异常 |
| 分组归纳 | 多个指标共享同一分组维度 | 在一个维度下扫读多项指标表现 |
| 公式拆解 | 指标之间存在明确公式关系 | 展示指标拆分关系和组成逻辑 |

典型业务问题：

- 当前 GMV、订单量、转化率是否正常？
- 与上周、上月或去年同期相比变化如何？
- 当前值距离目标还有多远？
- 某个账期、渠道、区域或业务线下的多个指标表现如何？
- 一个核心指标由哪些可解释的指标项拆解而来？

## 5. 不适用场景

| 不适用条件 | 原因 | 替代方向 |
|---|---|---|
| 需要比较多个类别排序 | 单卡难以表达排序和差异 | 柱状图、条形图、表格 |
| 需要分析完整趋势 | 迷你趋势不承载精确读数 | 折线图、面积图 |
| 需要展示构成占比 | 指标卡不表达部分与整体 | 饼图、堆叠图、树图 |
| 指标口径不明确 | 主数字会被误读为正式结论 | 先补口径或标记 `needs_spec` |
| 无目标却判断达标 | 缺少判断依据 | 只展示当前值，不输出状态 |
| 公式关系不明确 | 公式指标卡会制造错误因果 | 改用普通指标卡组或表格 |
| 指标数量过多 | 卡片区域会出现大量空白、滚动或扫读负担 | 分组、筛选、折叠、表格或看板分页 |

## 6. 数据结构

### 6.1 单张指标卡

| 字段角色 | 是否必需 | 说明 |
|---|---|---|
| 指标名称 | 必需 | 业务可读名称，不使用字段名 |
| 当前值 | 必需 | 主数字 |
| 单位 | 必需 | 元、%、人、件、次等 |
| 时间范围 | 建议 | 当前值对应的统计周期 |
| 对比值 | 可选 | 同比、环比、上期、去年同期 |
| 对比方向 | 可选 | 上升、下降、持平；必须结合业务含义判断好坏 |
| 目标值 | 可选 | 目标、预算、计划、SLA |
| 趋势序列 | 可选 | 用于迷你趋势图 |
| 状态字段 | 可选 | normal / warning / danger / disabled / selected 等 |
| 操作字段 | 可选 | 选择、链接、下钻、展开、自定义扩展区 |

### 6.2 指标卡组

| 字段角色 | 是否必需 | 说明 |
|---|---|---|
| 分组名称 | 必需 | 分组维度的可读名称，例如账期、渠道、区域 |
| 分组描述 | 可选 | 用于说明分组口径、范围或特殊规则 |
| 分组链接 | 可选 | 跳转到详情页、流转箭头或相关页面 |
| 指标列表 | 必需 | 一个或多个指标卡 |
| 分组主题 | 可选 | 默认灰色主题；可按业务需要使用蓝色等主题 |
| 布局方式 | 必需 | 横向、纵向、网格或公式布局 |
| 滚动策略 | 可选 | 横向整体滚动、纵向内部滚动、纵向整体滚动 |

### 6.3 公式指标卡组

| 字段角色 | 是否必需 | 说明 |
|---|---|---|
| 结果指标 | 必需 | 公式左侧或顶部的主指标 |
| 公式项 | 必需 | 拆解指标列表 |
| 运算符 | 必需 | `=`、`+`、`-`、`×`、`÷` 等 |
| 分组名称 | 可选 | 有分组时承载公式整体标题 |
| 计算口径 | 必需 | 必须说明公式和时间口径 |

公式指标卡只用于展示已确认的指标拆解关系。若缺少公式口径，不得生成公式布局。

## 7. 视觉构成

### 7.1 单张指标卡构成

| 构成元素 | 作用 | 是否必需 |
|---|---|---|
| 指标名称 | 说明指标含义 | 必需 |
| 主指标 | 展示当前值，包含币种符号、数值、单位或百分号 | 必需 |
| 子指标 | 展示同比、环比、同期、持平、目标差等 | 可选 |
| 迷你趋势图 | 展示轻量趋势 | 可选 |
| 多选 / 单选控件 | 当指标卡可选择时显示 | 可选 |
| 边框 | 区分卡片边界、悬停和选中状态 | 可选 |
| 扩展区域 | 承载定制业务内容，通常放在底部并用分割线隔开 | 可选 |
| 操作入口 | 承载链接、下钻、展开或更多操作 | 可选 |

执行规则：

- 指标名称必须清晰显示，必要时可用虚线注释、说明 icon 或 Tooltip 补充口径。
- 单张指标卡默认内容顺序固定为“指标名称 → 主指标 → 子指标”。主指标和子指标之间不得插入单位说明、时间说明、口径说明、扩展槽、空白占位或其他自定义元素。
- 主指标必须包含可读单位。金额可拆为币种符号、数值和数量级单位；单位必须作为主指标的一部分出现，不得另起一行放在主指标和子指标之间。
- 子指标通常展示同比 / 环比 / 同期等信息，用于说明增长、下降或持平。
- 同比、环比等子指标的升降不得使用箭头表示，应使用 `+` / `-` 表达变化值；持平可写为“持平”或 `0`，不得用箭头替代符号。
- 迷你趋势图是辅助信息，不得遮挡主指标或挤压主指标读数。
- 扩展区域不属于默认指标卡结构；只有业务明确要求时才允许出现，且必须放在子指标之后并用分割线隔开，不得出现在主指标和子指标之间。

### 7.2 指标卡组构成

| 构成元素 | 作用 | 是否必需 |
|---|---|---|
| 分组标题 | 展示分组名称 | 必需 |
| 分组描述 | 解释分组口径或补充说明 | 可选 |
| 插槽 / 链接 | 跳转、流转箭头或辅助功能 | 可选 |
| 指标卡列表 | 承载组内指标 | 必需 |
| 分组背景 | 形成组内归属关系 | 建议 |
| 滚动条 | 当内容超出容器时提供访问路径 | 可选 |

执行规则：

- 指标卡组至少包含分组标题和一个指标卡。
- 分组标题必须清晰展示，不能只用颜色或位置暗示。
- 组内指标卡应保持相同高度、相同信息结构和相近子指标数量。
- 指标数量不足时不应留下明显空白，应调整每行数量、卡片宽度或分组数量。

## 8. 设计原则

### 8.1 信息层级

- 主数字是第一视觉层级，必须易读。
- 单位必须跟随主数字组成主指标读数组；时间范围、统计口径和说明信息应进入标题说明、Tooltip 或卡片外部上下文，不得插入主指标和子指标之间。
- 对比信息应说明基准，例如“较上周”“同比去年同期”。
- 同比、环比和目标差的变化值统一使用 `+` / `-` 符号表达，不使用上升 / 下降箭头；方向字段可在数据中保留，但视觉展示以符号、文字和语义色为准。
- 状态色必须来自语义色，并配合文字说明。
- 多张指标卡并列时，应保持标题、数字、单位、对比信息的位置一致。
- 指标卡不应承载过多解释文本，复杂说明进入 Tooltip、详情或下钻页。

### 8.2 指标命名

指标名称必须满足以下规则：

| 原则 | 执行方式 |
|---|---|
| 业务相关性 | 名称应与业务目标和用户需求紧密相关 |
| 用户友好 | 使用非技术性语言，避免只有开发或数据人员能理解的术语 |
| 简洁明了 | 尽量使用简短描述性名称 |
| 避免歧义 | 名称不得与其他指标混淆 |
| 一致性 | 同一页面内保持命名风格一致 |
| 可扩展性 | 考虑未来可能扩展的指标，避免频繁重命名 |
| 上下文相关 | 与产品特性、用户行为或时间维度相关时，应在名称中体现 |

### 8.3 数值格式

- 数值过大时，应将个位换算为万、百万、亿等缩写单位，避免主数字过长导致趋势图错位。
- 同一组指标卡内应保持单位、精度、千分位和缩写策略一致。
- 金额、百分比、人数、件数等不同单位不得混写为同一种格式。
- 空值、无数据或未计算完成时，应显示明确占位，不得伪造为 `0`。
- vibeDesign 生成指标卡时，主指标大数值必须使用本地字体 `JDZhengHT-EN-Regular`。不得自动选择 `京东正黑 V2.1`、Bold、Medium 或其他非 Regular 字重替代；若运行环境缺少该字体，必须声明字体缺失并停止使用该样式，不得静默替换为普通系统字体。
- 默认指标卡主指标数值字号为 `24px`，不得为了强调而擅自放大到 `32px`、`36px` 或更大。需要大尺寸指标卡时必须由尺寸规格显式覆盖。
- 主指标单位与主数值不是同一文字样式。数值使用大号 `JDZhengHT-EN-Regular`，单位使用单位文本样式，字号应小于主数值，并与主数值做基线或视觉底部对齐。
- 主指标中的币种、数值和单位应作为一个读数组合排版。币种符号可跟随数值字体或按金额符号样式缩小；数量级单位、百分号、次、人、件等单位必须贴近数值，推荐间距为 `4px`，不得因拆成多个文本节点导致单位远离数值或垂直居中漂移。
- 默认子指标字号为 `14px`，label 与 value 使用同一字号，value 可通过字重和语义色强化。

### 8.4 局部间距与排版

指标卡内部间距应优先遵循组件结构，而不是平均分布。默认指标卡必须按照以下硬约束生成：

| 间距对象 | 默认值 | 规则 |
|---|---:|---|
| 卡片上内边距 | `12px` | 标题区距离卡片上边界 |
| 卡片左右内边距 | `16px` | 主信息区和右侧趋势图均不得贴边 |
| 卡片底内边距 | `16px` | 子指标或默认内容距离卡片下边界 |
| 指标名与口径下划线 / 注释线 | `2px` | 仅在需要解释口径时出现 |
| 标题与说明 icon | `4px` | icon 紧跟标题，不独立漂浮 |
| 标题区与主指标 | `2px` | 标题、注释线之后直接进入主指标，不插入其他元素 |
| 主数值与单位 | `4px` | 单位贴近主值，单位字号小于主数值，并与主数值基线或视觉底部对齐 |
| 主指标与子指标 | `2px` | 主指标下方直接进入同比 / 环比等子指标，不插入说明文本或空白插槽 |
| 子指标 label 与 value | `4px` | 例如“环比 -1.76%”中 label 和 value 的固定间距 |
| 同比组与环比组 | `12px` | 多个子指标水平排列时的组间距 |
| 主信息区与迷你趋势图 | `16px` | 左右布局时保证趋势图不挤压主值 |
| 选择控件到卡片上 / 右边距 | `6px` | 单选 / 多选控件使用 |

执行规则：

- 子指标必须以“label + value”为最小组，组内间距固定为 `4px`。
- 多个子指标水平排列时，应先保持组内紧凑，再通过 `12px` 组间距区分不同子指标；当剩余空间不足时，必须按子指标组自动换行，不得压缩文字、截断 value 或与迷你图重叠。
- 主指标和子指标必须包裹在同一个主内容容器中；该容器与右侧迷你图插槽组成自适应布局关系。不得让 `metric-main`、`submetrics` 和迷你图直接作为同级散落在卡片根节点上。
- 主体信息区应优先使用稳定的左右 flex 布局：`display: flex`、`align-items: stretch`、`justify-content: space-between`、`gap: 16px`、`width: 100%`。左侧主内容容器使用可收缩弹性宽度，右侧迷你图插槽使用稳定外接框。
- 默认不得通过窄屏媒体查询把主体信息区改为纵向堆叠，例如 `flex-direction: column`。响应式适配应优先发生在卡片组层级或通过左侧子指标换行解决，避免破坏主内容容器与右侧迷你图插槽的稳定关系。
- 当主体信息区剩余空间不足时，优先让左侧子指标在主内容容器内部换行；不得通过缩小间距、压缩字号或挤压右侧迷你图解决空间不足。
- 主指标数值和单位必须分层设置字体与字号，排版时优先使用基线对齐；当设计工具不支持精确基线时，使用视觉底部对齐，避免单位垂直居中或顶对齐。
- 主指标和子指标必须相邻排列，默认垂直间距为 `2px`；不得用“主值单位”“统计周期”“行业说明”等文本行或透明占位扩大间距。
- 迷你趋势图应放在稳定外接框内，外接框与主信息区保持 `16px` 间距；不得为了放大趋势图压缩主数值、单位或子指标。
- 指标卡高度必须由内容自适应，不得写死固定高度。子指标换行、字体放大或容器变窄时，卡片高度应自然增长，底部内边距仍保持 `16px`。
- 如业务明确要求扩展槽，扩展槽只能放在子指标之后，且必须通过分割线与默认内容区分；默认指标卡不得生成扩展槽。

### 8.5 布局与尺寸

指标卡提供大小两种尺寸和两类布局：

| 布局 | 使用条件 | 规则 |
|---|---|---|
| 左右布局 | 主指标和趋势图需要并排展示 | 主指标在左，迷你趋势在右；适合横向摘要 |
| 上下布局 | 趋势图需要获得更宽空间 | 指标信息在上，趋势图在下；通常趋势图区域更宽，可展示 X 轴趋势 |
| 大尺寸 | 需要展示更多子指标、趋势或扩展区 | 可承载更多信息，但仍保持主指标优先 |
| 小尺寸 | 看板空间有限或多卡并列 | 减少可选内容，保证主指标和必要子指标可读 |

尺寸与伸缩规则：

- 单张指标卡不建议过度横向拉伸；当宽度超过 `480px` 时，应优先增加列数、分组或改为完整图表。
- 单张指标卡不建议过度纵向拉伸；高度增加应服务于趋势区或扩展区，而不是制造空白。
- 默认指标卡高度应由内容、换行和内边距共同决定；不得通过固定高度制造裁切、空白或遮挡。
- 迷你趋势图应拥有稳定外接框，不因数值长度变化错位。
- 卡片宽高变化时，主数字、标题、子指标和迷你图的相对位置必须稳定。

### 8.6 分组展示

- 指标卡组用于展示多个具有分组属性的指标，重点突出业务数据，可通过指标变化趋势快速判断是否存在经营异常。
- 指标卡组支持默认灰色主题和蓝色主题；主题只用于强化分组背景，不应覆盖状态语义色。
- 分组指标卡支持横向、纵向和网格布局。
- 横向布局适合同一维度下少量指标横向扫读；纵向布局适合窄容器或指标自然按行阅读；网格布局适合多指标均衡展示。
- 公式指标卡可使用等号、乘号等公式符号表达指标拆解关系，但不支持自定义指标关系；必须基于已确认公式。

## 9. 交互规则

| 交互 | 启用条件 | 系统反馈 |
|---|---|---|
| Hover | 需要补充口径、公式或精确对比 | 显示 Tooltip；可显示悬停边框 |
| 单选 / 多选 | 指标卡可作为筛选或对比对象 | 显示选中边框和选择控件 |
| 禁用 | 指标不可选择、无权限或不可用 | 降低可交互反馈，不触发选择 |
| 点击下钻 | 指标可进入分析详情 | 跳转趋势、构成或明细 |
| 状态说明 | 存在阈值或异常规则 | 展开规则解释 |
| 时间切换 | 看板支持周期切换 | 更新当前值、对比值和迷你趋势 |
| 横向滚动 | 横向卡片超出容器且不宜压缩 | 显示横向滚动条，保持卡片宽度 |
| 纵向内部滚动 | 单个分组内指标较多 | 分组标题保持可见，组内列表滚动 |
| 纵向整体滚动 | 多个分组整体超出容器 | 整体区域滚动，保留分组之间的关系 |

交互状态规则：

- 指标卡可被选择时，可提供默认、悬停、选中、禁用、选中且禁用等状态。
- 默认状态不展示强边框；选中状态可使用蓝色边框或选中控件；禁用状态不得表现为可点击。
- 单选和多选都应明确选择范围，不能让用户误以为选中单卡会影响整个分组。
- 分组卡组横向布局支持整体滚动；纵向布局既可支持内部滚动，也可支持区域整体滚动。

## 10. 与视觉语言的关系

- 主数字、标题、说明文本读取 `01-design-language/typography.md`。
- 状态色和趋势色读取 `01-design-language/color.md` 和 `01-design-language/theme-token.md`。
- Tooltip 读取 `01-design-language/tooltip.md`。
- 迷你趋势读取 `02-chart-type/other/mini-chart.md`。
- 布局伸缩、滚动和 Frame 结构读取 `01-design-language/layout.md`。
- 目标差、状态标签和 callout 可读取 `09-visual-enhancement/`。

本文不重新定义全局字体、颜色、圆角、边框和阴影 token。涉及具体样式时应优先引用上游视觉语言。

## 11. G2 / ECharts 映射

指标卡通常不是标准图表引擎 series，而是 UI 组件 + 可选迷你图。

| 语义 | 实现建议 |
|---|---|
| 主数字 | DOM / Text / Figma Text |
| 单位与币种 | DOM / Text，和主数字同行或贴近展示 |
| 子指标 | DOM / Text + semantic color |
| 迷你趋势 | G2 / ECharts sparkline 或 SVG path |
| Tooltip | 统一 Tooltip 组件或浮层 |
| 状态 | 语义色 token + 标签文本 |
| 选择控件 | UI Checkbox / Radio / selected state |
| 分组标题 | UI Group Header / DOM Text |
| 分组滚动 | 容器滚动，不改变卡片内部比例 |

工程实现约束：

- 迷你图的采样、归一化和基线规则必须读取 `02-chart-type/other/mini-chart.md`。
- 分组卡不应把每张卡实现为完整大图表实例；只在需要趋势时为局部迷你图启用轻量绘制。
- 公式指标卡必须由后端或已确认配置提供公式关系，前端不得自行推断。

## 13. 禁止项

- 禁止没有单位的主数字。
- 禁止没有目标或阈值时判断“达标 / 未达标”。
- 禁止在同比、环比、目标差等变化值中使用上升 / 下降箭头；必须使用 `+` / `-` 或“持平”表达。
- 禁止只用红绿颜色表达好坏，不说明业务方向。
- 禁止把上升都写成正向、下降都写成负向。
- 禁止 vibeDesign 主指标大数值使用 `京东正黑 V2.1`、Bold、Medium 或系统字体替代 `JDZhengHT-EN-Regular`。
- 禁止默认指标卡主指标数值字号超过 `24px`，除非有明确大尺寸规格覆盖。
- 禁止主指标单位直接沿用主数值的大字号和京东正黑样式；单位必须使用更小字号，并与数值基线或视觉底部对齐。
- 禁止在主指标和子指标之间添加单位说明、时间说明、口径说明、行业说明、扩展槽、空白占位或透明辅助元素。
- 禁止把主指标单位另起一行作为“主值单位”说明；单位必须并入主指标读数组。
- 禁止将主指标、子指标和迷你图作为卡片根节点的散列同级元素；主指标和子指标必须先归入主内容容器，再与右侧迷你图插槽建立自适应关系。
- 禁止主体信息区缺少稳定 flex 关系；不得用绝对定位或散列 grid 子项替代 `metric-content` 与右侧插槽的左右自适应关系。
- 禁止用媒体查询将主体信息区改成纵向堆叠；不得为 `.metric-body` 生成 `flex-direction: column`。
- 禁止子指标 label 与 value 间距大于或小于 `4px`，导致“同比 / 环比 / 行业均值”等读数组松散或粘连。
- 禁止标题区到主指标、主指标到子指标的默认垂直间距偏离 `2px`，除非有明确尺寸规格覆盖。
- 禁止默认子指标字号小于或大于 `14px`，导致可读性或布局判断失真。
- 禁止固定指标卡高度；卡片必须随子指标换行和内容高度自适应。
- 禁止子指标空间不足时保持单行挤压、截断、压缩字号或与迷你图重叠；必须按子指标组自动换行。
- 禁止在一张指标卡里塞入完整分析报告。
- 禁止用指标卡替代需要比较、趋势或构成分析的图表。
- 禁止数值过长仍保留完整个位，导致主数字、单位和迷你趋势错位。
- 禁止单张指标卡超过 `480px` 后继续横向拉伸来填满版面。
- 禁止通过拉高卡片制造空白；若需要更多信息，应使用扩展区、下钻或完整图表。
- 禁止默认开启扩展槽；扩展槽只有业务明确要求时才能生成，且只能位于子指标之后。
- 禁止在指标数量不足时保留大块空白卡组区域；应调整数量、列数或布局。
- 禁止在公式关系未确认时生成公式指标卡。
- 禁止把分组主题色当作状态色使用。

## 14. AI 自查清单

- 是否明确指标名称、当前值、单位和时间范围？
- 是否说明对比基准？
- 同比、环比等变化值是否使用 `+` / `-` 或“持平”，并避免使用箭头？
- 是否有目标或阈值支撑状态判断？
- 如果使用迷你趋势图，是否仅作为辅助趋势，不替代完整趋势分析？
- 是否对大数值做了合理单位缩写？
- vibeDesign 主指标大数值是否使用本地字体 `JDZhengHT-EN-Regular`，并避免使用 `京东正黑 V2.1` / Bold / Medium？
- 默认主指标数值字号是否为 `24px`？
- 主指标单位是否使用独立字体 / 字号，并与主数值基线或视觉底部对齐？
- 主指标单位是否并入主指标读数组，而不是作为“主值单位”说明行插入？
- 主指标和子指标之间是否没有插槽、说明行、透明占位或其他额外元素？
- 主指标和子指标是否包裹在同一个主内容容器中，并与右侧迷你图插槽形成自适应关系？
- 主体信息区是否使用 `display:flex; align-items:stretch; justify-content:space-between; gap:16px; width:100%` 或等价布局？
- 是否避免使用媒体查询把主体信息区改为 `flex-direction: column`？
- 标题区到主指标、主指标到子指标是否保持 `2px` 默认垂直间距？
- 子指标字号是否为 `14px`，label 与 value 是否保持 `4px` 间距，子指标组间距是否保持 `12px`？
- 指标卡高度是否由内容自适应，而不是固定高度？
- 子指标空间不足时是否按子指标组自动换行，且没有文字截断或与迷你图重叠？
- 是否选择了合适的左右 / 上下布局和大小尺寸？
- 是否避免单卡过度横向或纵向拉伸？
- 如果使用指标卡组，是否包含分组标题、指标列表和明确布局？
- 如果使用公式指标卡，是否有已确认公式口径？
- 如果启用选择状态，是否区分默认、悬停、选中和禁用？
- 如果内容超出容器，是否声明横向或纵向滚动策略？
- 是否引用了字体、颜色、Tooltip、布局和迷你图规范？
- 是否提供下钻或详情承接复杂解释？
