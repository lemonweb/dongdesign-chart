# 迷你图 Mini Chart

## 1. 文档定位

本文定义迷你图在 dongDesign Chart 知识库中的选型、数据结构、视觉构成、尺寸规格、交互行为和代码映射。

迷你图主要用于回答：

> 当前指标的最近变化方向是什么？是否有明显波动、峰值、低谷或异常？

迷你图通常嵌入统计卡片、表格单元格、列表项、看板摘要区或移动端紧凑模块中。它不是完整分析图表的替代品，而是帮助用户快速扫读趋势和状态的辅助图形。

本文不重新定义颜色、字体、坐标轴、Tooltip、标签和卡片规范。相关视觉规则应读取 `01-design-language/` 与统计卡片类规范。

## 2. 图表定义

迷你图是一类小尺寸、弱坐标、轻标注的摘要图表。它通过极简图形呈现趋势、波动、分布或完成状态，常见形态包括迷你折线、迷你面积、迷你柱、迷你进度条和迷你环形进度。

在迷你图中：

- 图形服务于主指标，不独立承担完整解释任务。
- 默认不显示完整坐标轴、刻度线、网格线和图例。
- 必须保留核心数据语义，例如趋势方向、正负变化、完成度或最近值。
- 必须能在小尺寸下保持清晰，不依赖密集文本读取。

迷你图强调“速读”，不适合承载复杂比较、精确查询或多维分析。

## 3. 属性摘要

| 属性 | 说明 |
| --- | --- |
| 图表类型 | 辅助图表 / 摘要图表 |
| 图形形态 | 迷你折线、迷你面积、迷你柱、进度条、环形进度 |
| 主要功能 | 趋势摘要、波动提示、状态指示、完成度展示 |
| 适合的数据 | 小规模时间序列、少量分类序列、目标完成率、状态比例 |
| 数据与图形映射 | 时间 / 顺序映射到 X 方向，数值映射到 Y 高度、柱高、长度或角度 |
| 数据体量 | 单个迷你图建议 `5-30` 个点；超过 `50` 个点应聚合或降采样 |

## 4. 适用场景

| 场景 | 使用条件 | 价值 |
| --- | --- | --- |
| 指标卡趋势 | 主指标需要展示最近变化 | 快速判断上升、下降、波动或趋稳 |
| 表格内趋势 | 每行对象有一段短序列 | 在不展开详情的情况下比较走势 |
| 列表摘要 | 每个对象有轻量状态 | 帮助用户扫读异常和变化方向 |
| 目标完成度 | 存在目标值和当前值 | 显示完成进度、达成或未达成状态 |
| 小屏看板 | 空间有限但需要趋势提示 | 减少坐标轴和文字占用 |

典型业务问题：

- 这个指标最近 7 天是在上升还是下降？
- 哪个商品的销量波动最大？
- 哪个渠道的转化率最近出现异常下跌？
- 当前目标完成进度是多少？
- 列表中哪些对象值得进一步查看？

## 5. 不适用场景

| 不适用场景 | 原因 | 替代方向 |
| --- | --- | --- |
| 需要精确读取坐标值 | 迷你图默认无完整坐标轴 | 折线图、柱状图、表格 |
| 需要多系列复杂对比 | 小尺寸下颜色和线形难以区分 | 完整折线图、分面图 |
| 需要解释复杂分布 | 迷你图空间不足 | 直方图、箱线图、散点图 |
| 类别数量很多 | 标签无法承载 | 条形图、柱状图、表格 |
| 需要展示完整分析结论 | 迷你图只做摘要 | 图表卡片、分析面板 |

## 6. 数据结构

### 6.1 趋势迷你图

```json
[
  { "date": "2026-06-01", "value": 120 },
  { "date": "2026-06-02", "value": 132 },
  { "date": "2026-06-03", "value": 128 }
]
```

字段约束：

| 字段角色 | 是否必需 | 说明 |
| --- | --- | --- |
| 顺序字段 `x` / `time` | 必需 | 时间、序号或业务顺序 |
| 数值字段 `value` | 必需 | 映射到折线高度、面积高度或柱高 |
| 状态字段 `status` | 可选 | 标记异常、预测、缺失等状态 |
| 分组字段 `series` | 谨慎使用 | 迷你图默认不建议多系列 |
| 当前值 `currentValue` | 可选 | 与主指标联动 |
| 变化值 `delta` | 可选 | 与趋势色、趋势标识联动 |

### 6.2 完成度迷你图

```json
{
  "current": 72,
  "target": 100,
  "rate": 0.72,
  "status": "normal"
}
```

字段约束：

| 字段角色 | 是否必需 | 说明 |
| --- | --- | --- |
| 当前值 `current` | 必需 | 当前进度 |
| 目标值 `target` | 必需 | 完成度基准 |
| 完成率 `rate` | 可计算 | `current / target`，范围默认 `0-1` |
| 状态 `status` | 可选 | 正常、预警、达成、超额 |

### 6.3 数据要求

- 趋势迷你图必须有明确顺序；无序分类不能直接画成迷你折线。
- 数据点少于 `3` 个时，不应生成趋势线，可改用文本变化值或指标卡。
- 单个迷你图建议 `5-30` 个点；超过 `50` 个点应聚合、抽样或只展示最近窗口。
- 缺失值必须显式声明处理方式：断线、插值、跳过或置零。
- 多个迷你图并列比较时，必须统一时间窗口、采样间隔和 Y 轴归一策略。

## 7. 形态类型

### 7.1 迷你折线 Sparkline

迷你折线用于展示短周期趋势和波动，是默认形态。

适用条件：

- X 方向存在时间或顺序。
- 用户主要关注趋势方向、波动和异常点。
- 不需要读取每个点的精确值。

执行规则：

- 默认不显示点标记。
- 可标记最后一个点、最高点、最低点或异常点。
- 线宽默认 `1.5px`，最小不低于 `1px`。
- 不显示坐标轴、刻度和网格，除非进入展开态。

### 7.2 迷你面积 Mini Area

迷你面积用于强调趋势的整体量感，通常在指标卡中作为背景趋势。

适用条件：

- 需要增强趋势存在感。
- 数据均为非负值或有明确基线。
- 图形不会干扰主指标文本。

执行规则：

- 面积填充透明度建议 `8%-16%`。
- 面积填充不得喧宾夺主，不能压过主指标。
- 若存在负值，不应默认使用面积图，除非显式显示零基线。

### 7.3 迷你柱 Mini Bar

迷你柱用于展示短序列的离散周期值，例如最近 7 天、12 个月或多个状态桶。

适用条件：

- 每个点是独立周期或短分类。
- 用户关注局部高低和波动，而不是连续趋势。
- 数据点数量通常 `5-20`。

执行规则：

- 柱宽最小 `2px`，柱间距 `1-2px`。
- 顶部圆角最大 `1px`。
- 不显示完整 X 轴标签。
- 负值迷你柱必须显示零基线或上下分离方向。

### 7.4 迷你进度条 Mini Progress

迷你进度条用于展示目标完成率或比例状态。

适用条件：

- 存在明确目标值或总量。
- 用户关注达成状态，而不是趋势过程。

执行规则：

- 背景轨道使用弱化色，前景填充使用状态色或主强调色。
- 进度条高度建议 `4-8px`。
- 超过 `100%` 时必须声明超额表达方式：截断到 100%、延伸标记或显示超额标签。

### 7.5 迷你环形进度 Mini Donut

迷你环形进度用于在极小空间内展示单个比例。

适用条件：

- 只展示一个完成率或占比。
- 周围有文本说明当前指标。

执行规则：

- 环形尺寸不小于 `24px`。
- 环宽建议为直径的 `12%-18%`。
- 不承载超过一个比例，不做多段复杂环图。

## 8. 尺寸与布局

### 8.1 推荐尺寸

| 使用位置 | 推荐尺寸 | 说明 |
| --- | --- | --- |
| 指标卡右下角趋势 | `96 x 32` / `120 x 36` | 与主指标形成辅助关系 |
| 表格单元格 | `72 x 24` / `88 x 28` | 不撑高行高 |
| 列表项摘要 | `80 x 28` / `120 x 32` | 适合快速扫读 |
| 移动端卡片 | `100% x 32-48` | 宽度自适应，高度稳定 |
| 迷你环形 | `24 x 24` / `32 x 32` | 与文本同行展示 |

### 8.2 布局规则

- 迷你图应使用稳定外接框，不因数据值变化撑开容器。
- 图形必须在外接框内保留至少 `2px` 安全边距，避免线条或柱体被裁切。
- 指标卡中的迷你图不能遮挡标题、主数值、同比环比或状态标签。
- 表格内迷你图必须垂直居中，不应改变表格行高。
- 多个迷你图并排时，应统一外接框尺寸和基线位置。

### 8.3 与主指标关系

迷你图通常和主指标共同出现，信息层级必须明确：

```txt
标题 / 指标名
主数值
同比环比 / 状态
迷你趋势图
```

执行规则：

- 主数值永远比迷你图更高优先级。
- 迷你图不应抢占主数值颜色或字号注意力。
- 趋势颜色应与 `delta` 语义一致：上升、下降、持平、异常。
- 当主指标已有明确趋势箭头时，迷你图只补充波动形态，不重复写大量文字。

## 9. 样式生成规则

### 9.1 坐标与比例

迷你图通常隐藏坐标轴，但内部仍必须使用稳定坐标计算。

执行规则：

- 趋势类迷你图使用局部 `min-max` 归一化作为默认 Y 域。
- 多个迷你图用于横向比较时，应使用统一 Y 域或明确声明 `scaleMode = "local"`。
- 当业务关注是否高于 0 或目标值时，应固定 `baseline = 0` 或 `baseline = target`。
- 不得因为隐藏坐标轴就随意手绘线条；VibeDesign 和 VibeCode 必须共享 `scaleMode`、`domain` 和采样窗口。
- 极端值会压缩主体趋势时，可启用异常点标记或 winsorize，但必须声明。

### 9.2 色彩

迷你图色彩必须克制，默认只表达趋势或状态。

执行规则：

- 普通趋势使用主题主强调色或 `分类色板/item-1`。
- 上升 / 下降使用趋势语义色，且必须遵循业务含义：有些指标上升不一定是好。
- 背景面积或轨道使用低透明度弱化色。
- 不按每个点随机上色。
- 单个迷你图中同时出现的颜色不应超过 `2` 种，异常点或阈值线除外。

### 9.3 线、点、柱

| 图形 | 默认规则 |
| --- | --- |
| 折线 | 线宽 `1.5px`，默认无点，线帽 round |
| 面积 | 填充透明度 `8%-16%`，边界线同趋势色 |
| 柱体 | 最小宽度 `2px`，间距 `1-2px`，顶部圆角最大 `1px` |
| 进度条 | 高度 `4-8px`，轨道圆角不超过高度一半 |
| 环形 | 尺寸不小于 `24px`，环宽为直径 `12%-18%` |

### 9.4 标签

迷你图默认不显示坐标轴标签和全量数据标签。

允许显示的标签：

- 最后一个值。
- 最高值 / 最低值。
- 异常点。
- 完成率文本。
- 阈值说明。

执行规则：

- 标签必须使用 `.textSmall 12/18` 或更紧凑的组件内文字规格。
- 表格内迷你图默认不显示标签，避免破坏行高。
- 标签不能和主指标、状态标签或图形本身重叠。

### 9.5 状态

迷你图需要支持以下状态：

| 状态 | 表现 |
| --- | --- |
| normal | 默认主题色 |
| positive | 使用趋势上升或业务正向色 |
| negative | 使用趋势下降或业务负向色 |
| warning | 使用预警色，必要时标记异常点 |
| disabled | 降低透明度，保留结构 |
| loading | 使用骨架线 / 骨架条，不显示假数据 |
| empty | 显示空态短线或空文案，不生成随机趋势 |

空态和加载态不得用随机数据伪造趋势。

## 10. 交互规则

| 交互 | 启用条件 | 系统反馈 |
| --- | --- | --- |
| Hover Tooltip | 需要读取点位值 | 展示当前点时间、值、变化和状态 |
| 最后点提示 | 指标卡摘要 | 默认可强调最后一个点，但不弹出复杂浮层 |
| 点击跳转 | 迷你图代表可下钻指标 | 跳转到完整图表或详情页 |
| 表格行联动 | 迷你图位于表格中 | hover 行时高亮对应迷你图 |
| Brush / Zoom | 默认不启用 | 迷你图空间不足，不承载复杂交互 |

执行规则：

- 迷你图 Tooltip 应简短，避免遮挡主指标和邻近卡片。
- 表格内迷你图 hover 时，优先让整行状态保持一致。
- 若用户需要缩放、框选、图例筛选，应升级为完整图表。

## 12. G2 / ECharts 映射

### 12.1 统一配置语义

无论使用 G2、ECharts、SVG、Canvas、DOM 还是设计节点，迷你图都应先生成以下统一配置，再转换为具体引擎配置。

| 语义 | 取值 / 规则 | 说明 |
| --- | --- | --- |
| `chartType` | `mini-chart` | 固定为迷你图 |
| `variant` | `sparkline` / `mini-area` / `mini-bar` / `mini-progress` / `mini-donut` | 明确形态 |
| `dataMapping.x` | 时间或顺序字段 | 趋势类必需 |
| `dataMapping.y` | 数值字段 | 趋势类必需 |
| `dataMapping.current` | 当前值 | 完成度类必需 |
| `dataMapping.target` | 目标值 | 完成度类必需 |
| `scaleMode` | `local` / `shared` / `zero-baseline` / `target-baseline` | 控制 Y 域 |
| `window` | 最近 `n` 个点或时间范围 | 控制采样窗口 |
| `showAxis` | 默认 `false` | 迷你图默认隐藏轴 |
| `showPoint` | `none` / `last` / `extreme` / `anomaly` | 点标记策略 |
| `showLabel` | `false` / `last` / `extreme` / `progress` | 标签策略 |

### 12.2 G2

迷你折线：

```js
chart
  .line()
  .data(data)
  .encode("x", "date")
  .encode("y", "value")
  .axis(false)
  .legend(false)
  .style("lineWidth", 1.5);
```

迷你柱：

```js
chart
  .interval()
  .data(data)
  .encode("x", "date")
  .encode("y", "value")
  .axis(false)
  .legend(false)
  .style("radiusTopLeft", 1)
  .style("radiusTopRight", 1);
```

G2 映射规则：

- 默认关闭 `axis`、`legend` 和网格。
- 外部容器负责尺寸，图表不得自适应撑开卡片。
- `scale` 必须根据 `scaleMode` 明确设置，不得依赖引擎默认域导致多个迷你图不可比。
- Tooltip 按需启用，内容保持短格式。

### 12.3 ECharts

迷你折线：

```js
option = {
  grid: { left: 2, right: 2, top: 2, bottom: 2 },
  xAxis: { type: "category", show: false, data: data.map(d => d.date) },
  yAxis: { type: "value", show: false, min: "dataMin", max: "dataMax" },
  tooltip: { show: true, trigger: "axis" },
  series: [
    {
      type: "line",
      data: data.map(d => d.value),
      showSymbol: false,
      symbolSize: 4,
      lineStyle: { width: 1.5 },
      areaStyle: undefined
    }
  ]
};
```

迷你面积：

```js
option.series[0].areaStyle = {
  opacity: 0.12
};
```

迷你柱：

```js
option = {
  grid: { left: 2, right: 2, top: 2, bottom: 2 },
  xAxis: { type: "category", show: false, data: data.map(d => d.date) },
  yAxis: { type: "value", show: false },
  tooltip: { show: true, trigger: "axis" },
  series: [
    {
      type: "bar",
      data: data.map(d => d.value),
      barMinWidth: 2,
      barGap: "20%",
      itemStyle: { borderRadius: [1, 1, 0, 0] }
    }
  ]
};
```

ECharts 映射规则：

- `grid` 边距应控制在 `2-4px`，避免浪费小图空间。
- 默认 `xAxis.show = false`、`yAxis.show = false`。
- `showSymbol` 默认 `false`，仅最后点或异常点使用标记。
- 多个迷你图并列比较时，必须显式设置相同 `yAxis.min/max`。
- 不应开启 `dataZoom`、完整 legend、复杂 markLine，除非进入展开态。

    "y": "value",
    "status": "status optional",
    "delta": "delta optional"
  },
  "constraints": {
    "pointCount": "5-30 recommended, max 50 before aggregation",
    "showAxis": false,
    "showLegend": false,
    "showGrid": false,

    "width": "72-120 depending container",

## 14. 禁止项

AI 生成迷你图时禁止：

- 禁止把迷你图当成完整图表承载复杂分析。
- 禁止默认显示完整坐标轴、刻度、网格和图例。
- 禁止在无序分类数据上生成迷你折线。
- 禁止数据点少于 `3` 个时强行生成趋势线。
- 禁止用随机假数据填充 loading 或 empty 状态。
- 禁止多个并排迷你图使用不同 Y 域却不声明，造成趋势不可比。
- 禁止为每个点显示标签。
- 禁止在单个迷你图内使用过多颜色。
- 禁止在卡片里让迷你图遮挡主指标、标题或状态信息。
- 禁止开启复杂交互如框选、缩放、完整图例筛选；需要时应升级为完整图表。
- 禁止使用 3D、投影、发光、大面积渐变等装饰效果。

## 15. AI 自查清单

生成迷你图前，AI 必须检查：

- [ ] 迷你图是否只是辅助主指标，而不是替代完整分析图表？
- [ ] 当前数据是否存在明确时间或顺序字段？
- [ ] 数据点数量是否在 `5-30` 的推荐范围内？
- [ ] 数据点少于 `3` 个时是否改用文本或指标状态？
- [ ] 是否明确选择了 `sparkline`、`mini-area`、`mini-bar`、`mini-progress` 或 `mini-donut`？
- [ ] 是否声明 `scaleMode`，并说明多个迷你图是否可比较？
- [ ] 是否默认隐藏坐标轴、网格和图例？
- [ ] 是否只标记最后点、极值或异常点？
- [ ] 颜色是否绑定趋势语义和主题变量？
- [ ] 是否处理了 loading、empty、disabled 等状态？
- [ ] 迷你图是否在固定外接框内，不会撑开卡片或表格行？
- [ ] Tooltip 是否为紧凑内容，且不会遮挡主指标？
