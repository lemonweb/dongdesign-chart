# 03.气泡图 Bubble

## 1. 文档定位

气泡图用于在二维坐标关系的基础上，通过气泡大小表达第三个连续变量，是散点图的扩展关系分析图表。

本文档用于约束 AI 在生成气泡图时的图表判断、数据字段映射、视觉构成、交互行为和工程映射，避免把气泡图误用为普通散点图、精确数值比较图或多指标堆叠展示。

气泡图与散点图在坐标系、点位布局、分组颜色、Tooltip、框选和缩放等大体规范上保持一致；两者的主要差异在于气泡图必须存在明确的 `size` 字段，并且需要控制气泡半径、面积感知、遮挡和大小图例。

## 2. 图表定义

气泡图是将数据以不同大小的圆形气泡展现在平面直角坐标系上的统计图表。它至少需要三个变量：

- X 轴变量：决定气泡在横向坐标上的位置。
- Y 轴变量：决定气泡在纵向坐标上的位置。
- Size 变量：决定气泡的视觉大小，用于表达规模、权重、交易额、用户数、影响力等连续数值。

每个气泡表示一条数据记录。气泡的位置用于判断两个变量之间的关系、分布和异常点；气泡大小用于补充说明该记录在第三个变量上的规模差异。

气泡图的核心不是展示“更多指标”，而是在关系分析中补充一个规模维度。若第三变量无法解释业务问题，或用户只需要判断两个变量的关系，应优先使用散点图。

## 3. 属性摘要

| 属性 | 说明 |
| --- | --- |
| 图表类型 | 统计图表 |
| 图形形态 | 不同大小的圆形气泡 |
| 主要功能 | 关系、分布、规模比较、异常发现 |
| 适合的数据 | 两个连续字段 + 一个连续规模字段；可选一个分组字段 |
| 数据与图形映射 | X 轴映射第一变量；Y 轴映射第二变量；气泡面积或半径映射第三个规模变量；颜色映射可选分组 |
| 数据体量 | 至少 10-20 个数据点；推荐 20-200 个气泡；超过 200 个时需要谨慎处理遮挡、透明度、缩放或聚合 |

## 4. 适用场景

气泡图适用于以下场景：

- 在两个变量关系中同时观察第三个规模变量。
- 判断高价值对象在二维坐标空间中的位置分布。
- 比较不同对象在关系、表现和规模上的综合差异。
- 发现同时具有异常位置和异常规模的数据点。
- 对不同分组在二维关系空间中的分布和规模差异进行比较。

典型问题包括：

- 不同商品的转化率和客单价之间是否存在关系，同时 GMV 规模如何？
- 不同渠道的流量成本和转化率表现如何，用户规模是否集中在某些区域？
- 不同城市的人均消费和复购率之间是否相关，城市销售规模是否形成头部集中？
- 哪些活动既偏离整体表现，又具有较大交易规模或用户影响？

## 5. 不适用场景

以下场景不应优先使用气泡图：

- 只有两个变量需要分析时，应使用散点图。
- 第三个变量不是连续规模字段，或业务上不需要用大小表达时，不应强行使用气泡图。
- 需要精确比较第三变量大小时，应使用柱状图、条形图、表格或排序列表。
- 数据点过多且气泡相互严重遮挡时，应考虑散点图、热力图、密度图、分面图或聚合视图。
- 只需要展示单个指标的时间趋势时，应使用折线图或面积图。
- 只需要比较离散分类数值大小时，应使用柱状图或条形图。
- 只需要表达整体与部分关系时，应使用饼图、环图、堆叠图或矩形树图。
- 分类过多且必须通过颜色区分时，不应继续叠加颜色分组。

## 6. 数据结构

气泡图至少需要两个可映射到坐标位置的字段，以及一个可映射到气泡大小的连续规模字段。

### 6.1 基础气泡图

```json
[
  {
    "x": 53,
    "y": 48,
    "size": 1200
  },
  {
    "x": 67,
    "y": 52,
    "size": 3200
  }
]
```

### 6.2 分组气泡图

```json
[
  {
    "x": 53,
    "y": 48,
    "size": 1200,
    "category": "A"
  },
  {
    "x": 67,
    "y": 52,
    "size": 3200,
    "category": "B"
  }
]
```

字段约束：

- `x`：必填，连续数值字段，或具有明确顺序关系的分类字段。
- `y`：必填，连续数值字段。
- `size`：必填，连续数值字段，用于表达规模、权重或影响力。
- `category`：可选，用于分组对比，映射到颜色。
- `label`：可选，仅用于标记关键气泡、异常气泡或被选中气泡。

数据要求：

- `size` 必须为非负数值。若存在 0 值，应使用最小可见半径或在 Tooltip 中说明，不得让 0 值气泡完全不可发现。
- `size` 不应使用已被 X / Y 映射的同一字段，除非业务明确需要重复强调规模。
- `size` 的业务含义必须在图表标题、Tooltip 或大小图例中说明。
- 如果 `size` 值域跨度极大，应使用合理缩放方式控制最大和最小气泡差异，避免最大气泡遮挡主体分布。

## 7. 视觉构成

气泡图由以下部分构成：

- 坐标系：使用笛卡尔坐标系承载 X / Y 两个变量。
- 横轴：表示维度、连续变量或有序分类变量。
- 纵轴：表示指标或连续变量。
- 气泡：每个气泡表示一条数据记录，位置由 X / Y 决定，大小由 `size` 决定。
- 网格线：辅助用户判断气泡在坐标系中的相对位置。
- 颜色图例：仅当存在分组字段时显示。
- 大小图例：当 `size` 变量对理解结果重要时显示，用于解释气泡大小与数值规模的关系。
- 标签：默认不显示，仅在关键气泡、异常气泡或交互选中时显示。
- 提示框：用于展示单个气泡的完整字段信息，包括 X、Y、Size、分组和标签。

## 8. 设计原则

### 8.1 数据量

气泡图比散点图更容易出现遮挡，因此推荐数据量应更克制。

- 最小数据量：至少 10-20 个数据点。
- 推荐数据量：20-200 个气泡。
- 高数据量：超过 200 个气泡时，需要通过透明度、缩放、框选、分面、筛选或聚合能力降低遮挡。

当数据点超过 500 个且需要观察整体密度时，不应继续使用气泡图，应优先考虑散点图、热力图或密度图。

### 8.2 维度控制

气泡图最多同时表达 X、Y、Size 和 Category 四类信息。默认不再叠加更多视觉编码。

- X / Y 是主分析变量，必须保持最高优先级。
- Size 是第三变量，用于表达规模，不用于表达分类。
- Category 只用于少量分组对比，建议控制在 2-4 个之间。
- 不得同时使用颜色深浅、形状、描边样式等额外通道表达更多变量。

### 8.3 气泡尺寸

气泡尺寸必须保证既可见又不过度遮挡。默认使用半径映射，输出时必须说明映射的是半径还是面积。

| 数据规模 | 推荐半径范围 |
| --- | --- |
| 少数据集，50 个气泡以下 | 6-24px |
| 中数据集，50-200 个气泡 | 4-18px |
| 高数据集，200 个气泡以上 | 3-12px |

执行规则：

- 最小气泡半径不得低于 `3px`，避免气泡不可点击或不可读。
- 最大气泡半径不得大到遮挡主要分布区域，默认不超过 `24px`。
- 气泡大小应优先使用面积感知映射，避免用户误读大小差异。
- 如果引擎只支持半径映射，应在代码或输出中明确 `sizeScale`，并控制最大半径。
- 当气泡重叠严重时，应优先降低透明度、启用缩放或分面，不应继续增大最大半径。

### 8.4 颜色与透明度

分组气泡图使用颜色区分类别。颜色只用于表达分组，不用于表达数值大小。

- 分组颜色必须绑定图表分类色变量。
- 气泡默认可使用适度透明度，让重叠区域仍可被感知。
- 透明度不得低到破坏分类色识别。
- 若没有分组字段，所有气泡应使用同一主色或主题规定的数据点颜色。

### 8.5 大小图例

当 `size` 字段承担重要业务解释时，应提供大小图例。

大小图例应说明：

- `size` 字段名称。
- 最小、中位或代表值、最大值的气泡大小。
- 单位或业务含义。

如果空间有限，可以不显示大小图例，但 Tooltip 必须展示 `size` 的完整值和单位。

### 8.6 标签显示

气泡图默认不显示全部标签。标签只允许在以下场景出现：

- 标记最大规模气泡。
- 标记异常气泡。
- 标记业务关键气泡。
- 鼠标悬停或选中时展示。
- 框选后展示选中气泡的摘要。

不得为所有气泡同时显示标签。若需要展示所有对象名称，应使用表格、排行榜或可搜索列表辅助呈现。

## 9. 交互规则

### 9.1 悬停

鼠标悬停在气泡上时，应高亮当前气泡，并通过 Tooltip 展示 X 值、Y 值、Size 值、分组、标签和单位。

Tooltip 中必须明确展示 `size` 字段名称，不得只展示“大小”或默认字段名。

### 9.2 框选

用户可以使用鼠标框选图表中的一部分气泡。框选后：

- 选中气泡保持主色或高亮状态。
- 未选中气泡置灰并降低透明度。
- 未选中气泡建议使用 `20%` 黑色透明度或等效弱化样式。
- 框选区域内的气泡应保持位置和大小清晰可见。

### 9.3 图例筛选

当存在分组字段时，颜色图例可用于筛选或高亮对应类别。被关闭的类别不应继续参与当前视图判断。

当存在大小图例时，大小图例默认只作说明，不默认承担筛选能力。若需要按规模筛选，应使用独立筛选器或范围控件。

### 9.4 缩放与平移

当气泡分布密集、重叠明显或需要查看局部关系时，可以提供缩放和平移能力。

缩放后气泡尺寸应保持可读，不应因为坐标缩放导致气泡半径无限放大或缩小。默认气泡半径应保持屏幕像素尺寸，坐标位置随视图缩放变化。

## 10. 与视觉语言的关系

气泡图依赖以下视觉规范：

- 坐标轴：使用 `01-design-language/axis.md` 中的坐标轴、刻度、网格线规则。
- 坐标轴布局：必须继承直角坐标系布局规格，建立独立 `yAxisLabelRail`、`plotArea`、`xAxisLabelBand`。默认 `Y` 轴标签轨道 `50px`，标签右边缘距绘图区左边界 `10px`；`X` 轴标签顶部距底部轴线 `10px`，标签带高度至少 `28px`。
- 图例：存在分组字段时，使用 `01-design-language/legend.md` 中的图例规则；存在大小图例时，应与颜色图例保持层级清晰，不得混淆分类色和规模大小。
- 色彩系统：分组颜色必须绑定图表分类色变量，不使用临时颜色。
- 主题变量：气泡颜色、标签、轴线、网格线、Tooltip、选中态和弱化态应绑定主题变量。
- 字体排版：坐标轴、图例、标签、Tooltip 使用可视化系统约定的 12 / 14 / 16 字号，不新增临时字号。

执行约束：

- 散点 / 气泡图不得单独发明坐标轴间距，也不得把轴标签放入绘图区内部。
- 数据点坐标必须以 `plotArea` 边界计算；不得以包含轴标签轨道的外层 chartFrame 计算。
- 网格线必须裁切在 `plotArea` 内，不得延伸到轴标签轨道、标题区或图例区。
- 气泡半径应在 `plotArea` 内参与边界判断，避免最大气泡被图表边缘裁切。
- 如果气泡图文档未明确某个细节，应按 `05-rules/README.md` 使用 G2 / ECharts 常规布局作为兜底，并在输出中说明 `fallback_reference`。

## 11. G2 / ECharts 映射

### 11.1 G2

```js
chart
  .point()
  .data(data)
  .encode("x", "x")
  .encode("y", "y")
  .encode("size", "size")
  .encode("color", "category")
  .scale("size", {
    range: [6, 24]
  });
```

基础映射：

- `x` -> X 轴字段。
- `y` -> Y 轴字段。
- `size` -> 气泡大小字段，必填。
- `color` -> 分组字段，可选。
- `tooltip` -> 气泡的完整字段信息。

G2 输出时必须显式声明 `size` 映射范围，不得依赖默认点大小。

### 11.2 ECharts

```js
option = {
  xAxis: {
    type: "value"
  },
  yAxis: {
    type: "value"
  },
  tooltip: {
    trigger: "item"
  },
  series: [
    {
      type: "scatter",
      symbolSize: function (value) {
        return value[2];
      },
      data: data.map((item) => [item.x, item.y, item.size])
    }
  ]
};
```

ECharts 中气泡图通常仍使用 `scatter` series，通过 `symbolSize` 表达 `size` 字段。分组气泡图应按 `category` 拆分为多个 `series`，或通过 `dataset` + `encode` 维护字段映射。

`symbolSize` 不应直接返回原始业务值。必须经过缩放函数映射到可控半径范围，例如 `4-18px` 或 `6-24px`。

## 12. AI 输出格式

当 AI 生成气泡图时，应输出以下结构：

```json
{
  "chartType": "bubble",
  "purpose": "relationship | distribution | scale-comparison | outlier-detection",
  "dataMapping": {
    "x": "连续变量或有序分类字段",
    "y": "连续变量字段",
    "size": "连续规模字段",
    "category": "可选分组字段",
    "label": "可选标签字段"
  },
  "style": {
    "bubbleShape": "circle",
    "sizeEncoding": "area-aware scale preferred",
    "radiusRange": "根据数据量选择 6-24 / 4-18 / 3-12",
    "opacity": "存在遮挡时可降低透明度",
    "showLabel": false,
    "showSizeLegend": "size 字段承担重要解释时启用",
    "axisLayout": "yAxisLabelRail 50px; yAxisLabelRightPadding 10px; xAxisLabelBand >= 28px; xAxisLabelTopOffset 10px"
  },
  "interaction": {
    "tooltip": true,
    "brush": true,
    "legendFilter": "存在 category 字段时启用",
    "zoom": "气泡密集或遮挡明显时启用"
  }
}
```

## 13. 禁止项

- 禁止在没有两个可映射到 X / Y 的字段时生成气泡图。
- 禁止在没有明确 `size` 字段时生成气泡图；此时应使用散点图。
- 禁止把无序分类字段强行映射到 X / Y 连续坐标。
- 禁止使用气泡图表达整体占比关系。
- 禁止使用气泡面积进行需要精确读数的大小比较。
- 禁止将原始业务数值直接作为气泡半径。
- 禁止在未说明第三变量含义时使用气泡大小编码。
- 禁止为所有气泡同时显示标签。
- 禁止在类别超过 4 个时继续使用颜色强行分组。
- 禁止在气泡严重重叠时继续增大最大半径。
- 禁止同时用颜色深浅和气泡大小表达两个连续变量，导致用户无法判断主次。
- 禁止将气泡直接连成折线，除非数据存在明确顺序且图表目标已经转为趋势表达。

## 14. AI 自查清单

生成气泡图前，AI 必须检查：

- 是否至少存在两个可映射到 X / Y 的字段？
- 是否存在明确、连续、非负的 `size` 字段？
- `size` 字段是否对当前业务问题有解释价值？
- X / Y 是否具有连续数值或明确顺序关系？
- 当前任务是否是在分析关系、分布、规模差异或异常点？
- 数据点数量是否至少达到 10-20 个？
- 数据量是否适合直接显示，是否需要透明度、框选、缩放、筛选或聚合？
- 气泡半径范围是否明确，是否避免最大气泡遮挡主体分布？
- 分组类别是否控制在 2-4 个之间？
- 是否只在必要时显示标签？
- 是否需要颜色图例、大小图例、Tooltip 和框选交互？
- 是否已将颜色、轴线、网格、文字和交互状态绑定到主题变量？
