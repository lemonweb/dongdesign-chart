# 02.散点图 Scatter

## 1. 文档定位

散点图用于描述两个变量之间的关系、分布、趋势和异常点，是统计图表中的基础关系分析图表。

本文档用于约束 AI 在生成散点图时的图表判断、数据字段映射、视觉构成、交互行为和工程映射，避免将散点图误用为趋势图、分类比较图或占比图。

## 2. 图表定义

散点图是将数据以点的形式展现在平面直角坐标系上的统计图表。它至少需要两个变量，一个沿 X 轴绘制，另一个沿 Y 轴绘制。每个点在 X、Y 轴上都有一个确定的位置，表示一条数据记录在两个变量上的取值。

当大量数据点叠加后，散点图可以呈现数据集的整体分布，帮助用户分析两个变量之间是否存在相关性、相关性的强弱、趋势方向、聚集区域和异常点。必要时，可以使用颜色、透明度或分组字段表达附加变量，但不得破坏 X / Y 两个核心变量的主导地位。

## 3. 属性摘要

| 属性 | 说明 |
| --- | --- |
| 图表类型 | 统计图表 |
| 图形形态 | 圆点 |
| 主要功能 | 关系、分布、趋势 |
| 适合的数据 | 两个连续字段；或一个有序分类字段 + 一个连续字段 |
| 数据与图形映射 | X 轴映射维度或连续变量；Y 轴映射指标或连续变量；点的位置由 X / Y 坐标共同决定 |
| 数据体量 | 至少 10-20 个数据点；50-500 个数据点通常最适合观察关系并避免过度绘制 |

## 4. 适用场景

散点图适用于以下场景：

- 分析两个变量之间是否存在相关关系。
- 判断变量之间相关性的强弱和方向。
- 观察数据在二维坐标空间中的整体分布。
- 发现离群点、异常值或局部聚集区域。
- 对不同分组在同一二维空间中的分布差异进行比较。

典型问题包括：

- 价格和销量是否存在关系？
- 成本升高是否伴随转化率下降？
- 不同类别的数据是否形成明显聚类？
- 是否存在偏离整体趋势的异常样本？

## 5. 不适用场景

以下场景不应优先使用散点图：

- 只需要展示单个指标的时间趋势时，应使用折线图。
- 只需要比较离散分类的数值大小时，应使用柱状图或条形图。
- 只需要表达整体与部分关系时，应使用饼图、环图或堆叠类图表。
- 只需要展示单变量分布时，应使用直方图、箱线图或密度图。
- 数据点过少且不足以形成关系判断时，不应强行生成散点图。
- 分类过多且必须通过颜色区分时，不应继续堆叠分类颜色。

## 6. 数据结构

散点图至少需要两个可映射到坐标位置的字段。

### 6.1 基础散点图

```json
[
  {
    "sample": "点1",
    "value": 48
  },
  {
    "sample": "点2",
    "value": 52
  }
]
```

### 6.2 分组散点图

```json
[
  {
    "sample": "点1",
    "series": "指标A",
    "value": 48
  },
  {
    "sample": "点1",
    "series": "指标B",
    "value": 67
  }
]
```

字段约束：

- `sample` / `x`：必填，表示样本、记录或连续 X 值。若只有样本字段，生成器可按样本顺序映射到 X 轴位置。
- `value` / `y`：必填，连续数值字段。
- `series` / `category`：可选，用于多指标或分组对比，映射到颜色。
- `label`：可选，仅用于标记关键点、异常点或被选中点。
- `size`：谨慎使用。若点大小承载第三个连续变量，应优先判断是否需要升级为气泡图。

默认数据规则：

- 基础散点图默认只需要 `1` 个指标字段。
- 当只有 `1` 个指标字段时，所有点使用同一主色，不得按样本 / 维度逐点套用分类色板。
- 当存在 `2` 个及以上指标系列，或存在明确 `series/category` 分组字段时，才使用分类色区分系列，并可显示图例。
- 若业务明确提供连续 `x` 与连续 `y` 两个字段，可按二维关系散点图处理；否则默认使用样本顺序作为 X 轴。

## 7. 视觉构成

散点图由以下部分构成：

- 坐标系：使用笛卡尔坐标系承载 X / Y 两个变量。
- 横轴：表示维度、连续变量或有序分类变量。
- 纵轴：表示指标或连续变量。
- 数据点：每个点表示一条数据记录。
- 网格线：辅助用户判断点在坐标系中的相对位置。
- 图例：仅当存在分组字段时显示。
- 标签：默认不显示，仅在关键点、异常点或交互选中时显示。
- 提示框：用于展示单个点的完整字段信息。

## 8. 设计原则

### 8.1 数据量

散点图理论上没有绝对数据上限，但数据量过少会导致关系判断失去意义。

- 最小数据量：至少 10-20 个数据点。
- 推荐数据量：50-500 个数据点。
- 高数据量：超过 500 个点时，需要通过透明度、采样、缩放、框选或聚合能力降低过度绘制。

### 8.2 维度控制

分组维度建议控制在 2-4 个之间。维度过多会导致颜色难以区分、图例负担增加、标签重叠，并削弱用户对整体分布的判断。

### 8.3 点尺寸

点尺寸应根据数据量果断选择，不使用模糊区间。

| 数据规模 | 点直径 |
| --- | --- |
| 少数据集，50 个点以下 | 8px |
| 中数据集，50-100 个点 | 6px |
| 高数据集，100-1000 个点 | 4px |

当点数量较多且重叠严重时，可以降低透明度，但不应继续增大点尺寸。

### 8.4 分组颜色

分组散点图使用颜色区分类别。颜色只用于表达分组，不用于表达数值大小。若同时需要表达数值大小，应使用 `size` 字段并明确判断是否更适合气泡图。

执行规则：

- 单指标基础散点图使用单一主色或 `分类色板/item-1`。
- 禁止在单指标散点图中按每个样本分配不同分类色。
- 多指标散点图按指标系列分配分类色，图例 label 使用指标业务名。
- 分组字段不存在时，不显示颜色图例。

### 8.5 标签显示

散点图默认不显示全部标签。标签只允许在以下场景出现：

- 标记异常点。
- 标记业务关键点。
- 鼠标悬停或选中时展示。
- 框选后展示选中点的摘要。

不得为所有点同时显示标签。

## 9. 交互规则

### 9.1 悬停

鼠标悬停在数据点上时，应高亮当前点，并通过 Tooltip 展示 X 值、Y 值、分组、标签等完整信息。

### 9.2 框选

用户可以使用鼠标框选图表中的一部分数据点。框选后：

- 选中点保持主色或高亮状态。
- 未选中点置灰并降低透明度。
- 未选中点建议使用 `20%` 黑色透明度或等效弱化样式。
- 框选区域内的数据应保持清晰可见。

### 9.3 图例筛选

当存在分组字段时，图例可用于筛选或高亮对应类别。被关闭的类别不应继续参与当前视图判断。

### 9.4 缩放与平移

当数据量较大或点分布密集时，可以提供缩放和平移能力，帮助用户查看局部关系。

## 10. 与视觉语言的关系

散点图依赖以下视觉规范：

- 坐标轴：使用 `01-design-language/axis.md` 中的坐标轴、刻度、网格线规则。
- 坐标轴布局：必须继承直角坐标系布局规格，建立独立 `yAxisLabelRail`、`plotArea`、`xAxisLabelBand`。默认 `Y` 轴标签轨道 `50px`，标签右边缘距绘图区左边界 `10px`；`X` 轴标签顶部距底部轴线 `10px`，标签带高度至少 `28px`。
- 图例：存在分组字段时，使用 `01-design-language/legend.md` 中的图例规则。
- 色彩系统：分组颜色必须绑定图表分类色变量，不使用临时颜色。
- 主题变量：点颜色、标签、轴线、网格线、Tooltip、选中态和弱化态应绑定主题变量。
- 字体排版：坐标轴、图例、标签、Tooltip 使用可视化系统约定的 12 / 14 / 16 字号，不新增临时字号。

执行约束：

- 散点 / 气泡图不得单独发明坐标轴间距，也不得把轴标签放入绘图区内部。
- 数据点坐标必须以 `plotArea` 边界计算；不得以包含轴标签轨道的外层 chartFrame 计算。
- 网格线必须裁切在 `plotArea` 内，不得延伸到轴标签轨道、标题区或图例区。
- 如果散点图文档未明确某个细节，应按 `05-rules/README.md` 使用 G2 / ECharts 常规布局作为兜底，并在输出中说明 `fallback_reference`。

## 11. G2 / ECharts 映射

### 11.1 G2

```js
chart
  .point()
  .data(data)
  .encode("x", "x")
  .encode("y", "y")
  .encode("color", "category")
  .style("r", 3);
```

基础映射：

- `x` -> X 轴字段。
- `y` -> Y 轴字段。
- `color` -> 分组字段，可选。
- `size` -> 点大小，可选；使用前必须判断是否为气泡图。
- `tooltip` -> 点的完整字段信息。

### 11.2 ECharts

```js
option = {
  xAxis: {
    type: "value"
  },
  yAxis: {
    type: "value"
  },
  tooltip: {
    trigger: "item"
  },
  series: [
    {
      type: "scatter",
      data: data.map((item) => [item.x, item.y])
    }
  ]
};
```

分组散点图应按 `category` 拆分为多个 `series`，或通过 `dataset` + `encode` 维护字段映射。

## 12. AI 输出格式

当 AI 生成散点图时，应输出以下结构：

```json
{
  "chartType": "scatter",
  "purpose": "relationship | distribution | trend | outlier-detection",
  "dataMapping": {
    "x": "样本顺序、连续变量或有序分类字段",
    "y": "连续变量字段 / 指标字段",
    "series": "可选指标系列或分组字段",
    "label": "可选标签字段"
  },
  "style": {
    "pointShape": "circle",
    "pointSize": "根据数据量选择 8 / 6 / 4",
    "singleMetricColor": "use item-1 only",
    "multiMetricColor": "color by series",
    "opacity": "高密度数据可降低透明度",
    "showLabel": false,
    "axisLayout": "yAxisLabelRail 50px; yAxisLabelRightPadding 10px; xAxisLabelBand >= 28px; xAxisLabelTopOffset 10px"
  },
  "interaction": {
    "tooltip": true,
    "brush": true,
    "legendFilter": "存在分组字段时启用"
  }
}
```

## 13. 禁止项

- 禁止在没有两个可映射变量的情况下生成散点图。
- 禁止把无序分类字段强行映射到 X / Y 连续坐标。
- 禁止用散点图表达整体占比关系。
- 禁止用散点图替代明确的时间趋势折线图。
- 禁止为所有数据点同时显示标签。
- 禁止在类别超过 4 个时继续使用颜色强行分组。
- 禁止在点密集重叠时继续增大点尺寸。
- 禁止在未说明第三变量含义时使用点大小编码。
- 禁止将散点直接连成折线，除非数据存在明确顺序且图表目标已经转为趋势表达。

## 14. AI 自查清单

生成散点图前，AI 必须检查：

- 是否至少存在两个可映射到 X / Y 的字段？
- X / Y 是否具有连续数值或明确顺序关系？
- 当前任务是否是在分析关系、分布、趋势或异常点？
- 数据点数量是否至少达到 10-20 个？
- 数据量是否适合直接显示，是否需要透明度、框选或缩放？
- 分组类别是否控制在 2-4 个之间？
- 是否只在必要时显示标签？
- 是否需要图例筛选、Tooltip 和框选交互？
- 是否已将颜色、轴线、网格、文字和交互状态绑定到主题变量？
