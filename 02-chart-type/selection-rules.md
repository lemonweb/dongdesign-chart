# 图表选型规则 Chart Selection Rules

## 1. 文档目的

本文件用于指导 AI 和设计师在数据可视化设计中快速选择合适的图表类型。

本文件只负责回答：

> 当前需求应该优先考虑哪些图表？

具体图表的设计细则、视觉规范和代码实现，不在本文展开，应继续查阅对应文档。

---

## 2. 使用原则

图表选型必须先判断业务问题，再判断数据结构，最后选择图表。

正确顺序：

```txt
业务问题
→ 分析意图
→ 数据结构
→ 候选图表
→ 排除不适合图表
→ 查阅具体图表文档
→ 输出最终方案
```

错误示例：

```
看到“占比”就直接用饼图看到“趋势”就直接用折线图看到“三个指标”就直接用气泡图看到“多维指标”就直接用雷达图
```

---

## 3. 选型前必须识别的信息

AI 在选图前，必须识别以下 4 类信息：

### 3.1 业务目标

用户到底想解决什么问题？

常见目标：

```
看趋势看比较看占比看排名看分布看关系看转化看流向看地域差异看多维画像看异常监控
```

### 3.2 主要分析意图

每个图表方案必须有一个主要意图：

```
primary_intent
```

例如：

```
趋势 Trend比较 Comparison占比 Proportion关系 Relationship分布 Distribution流程 Flow地理 Geo多维评估 Multivariate
```

### 3.3 次要分析意图

很多图表同时承担多个表达目标，因此还需要识别：

```
secondary_intents
```

例如：

```
堆叠柱状图：primary_intent: comparisonsecondary_intents: proportion, composition堆叠面积图：primary_intent: trendsecondary_intents: proportion, composition, volume气泡图：primary_intent: relationshipsecondary_intents: comparison, distribution, ranking
```

### 3.4 数据结构

至少判断以下字段类型：

```
category：类目字段time：时间字段value：数值字段group：分组字段size：规模字段source / target：流向字段geo：地理字段status：状态字段hierarchy：层级字段
```

---

## 4. 图表选型总流程

AI 必须按照以下流程选图：

```
Step 1：理解业务问题Step 2：判断 primary_intentStep 3：识别 secondary_intentsStep 4：识别数据字段结构Step 5：查阅 02-chart-type/capability-matrix.mdStep 6：筛选 1-3 个候选图表Step 7：排除明显不适合的图表Step 8：查阅候选图表的具体文档Step 9：输出推荐方案、字段映射和不推荐方案
```

---

## 5. 分析意图与候选图表

本表只用于快速选型。  
最终选择必须继续查阅 `02-chart-type/capability-matrix.md` 和具体图表文档。

|分析意图|典型问题|优先候选图表|
|---|---|---|
|比较 Comparison|谁更高？谁更低？差异多大？|柱状图、条形图、分组柱状图、热力图|
|趋势 Trend|指标随时间如何变化？|折线图、面积图、堆叠面积图、迷你趋势图|
|占比 Proportion|各部分占整体多少？|环图、饼图、百分比堆叠柱状图、矩形树图|
|排名 Ranking|Top / Bottom 是谁？|条形图、排行榜、柱状图、表格|
|分布 Distribution|数据集中在哪里？是否有异常？|直方图、箱线图、散点图、热力图|
|关系 Relationship|两个或多个变量是否相关？|散点图、气泡图、热力图、桑基图|
|流程 Flow|阶段转化或路径流向如何？|漏斗图、桑基图、路径流转图|
|地理 Geo|不同地区表现如何？|区域地图、气泡地图、迁徙地图、条形图|
|多维评估 Multivariate|多个维度的能力结构如何？|雷达图、评分卡、热力图、多指标表格|
|监控 Monitoring|当前状态是否正常？是否异常？|指标卡、折线图、告警列表、热力图|

## 6. 多意图场景选型

当需求同时包含多个分析意图时，不要只根据单一关键词选图。

### 6.1 趋势 + 占比

典型问题：

```
不同渠道的流量占比随时间如何变化？
```

候选图表：

```
堆叠面积图百分比堆叠面积图百分比堆叠柱状图
```

判断：

```
关注绝对体量 + 构成变化 → 堆叠面积图只关注比例变化 → 百分比堆叠面积图时间点较少 → 百分比堆叠柱状图
```

---

### 6.2 比较 + 构成

典型问题：

```
不同城市的订单量以及订单状态构成如何？
```

候选图表：

```
堆叠柱状图百分比堆叠柱状图分组柱状图
```

判断：

```
关注总量差异 + 内部构成 → 堆叠柱状图只关注构成比例 → 百分比堆叠柱状图需要精确比较子类目 → 分组柱状图
```

---

### 6.3 关系 + 规模

典型问题：

```
商品价格、销量和库存之间有什么关系？
```

候选图表：

```
散点图气泡图热力图
```

判断：

```
两个数值变量 → 散点图两个变量关系 + 第三个变量规模 → 气泡图点位过多或重叠严重 → 热力图 / 密度图
```

---

### 6.4 排名 + 趋势

典型问题：

```
Top 10 商品最近 30 天销售趋势如何？
```

候选图表：

```
排行榜 + 迷你趋势图条形图 + 折线图小多图
```

判断：

```
空间有限 → 排行榜 + Sparkline需要详细分析 → 条形图 + 折线图每个对象趋势都重要 → 小多图
```

---

### 6.5 流程 + 状态构成

典型问题：

```
活动从报名到审核再到成交，每个阶段流失多少，各状态占比如何？
```

候选图表：

```
漏斗图堆叠柱状图漏斗图 + 堆叠柱状图
```

判断：

```
主目标是阶段转化 → 漏斗图主目标是阶段内状态构成 → 堆叠柱状图两者都重要 → 组合图表
```

## 7. 快速排除规则

以下规则用于快速排除明显不合适的图表。  
具体限制请查阅对应图表文档。

### 7.1 不要默认使用饼图

以下情况优先避免饼图 / 环图：

```
分类过多
需要精确比较
需要展示趋势
存在负数
多个对象之间要比较构成
```

优先替代：

```
条形图
百分比堆叠柱状图
矩形树图
表格
```

---

### 7.2 不要滥用雷达图

以下情况优先避免雷达图：

```
维度过多
对象过多
指标未标准化
需要精确比较
需要表达排名
```

优先替代：

```
热力图评分卡分组条形图多指标表格
```

---

### 7.3 不要滥用双轴图

以下情况优先避免双轴图：

```
两个指标没有明确业务关系
只是为了节省空间
容易制造虚假相关
坐标轴单位不清晰
```

优先替代：

```
上下排列两个趋势图
柱线混合图
指标卡 + 趋势图
```

---

### 7.4 不要用复杂图表表达简单问题

简单比较优先使用：

```
柱状图
条形图
表格
```

简单趋势优先使用：

```
折线图
迷你趋势图
```

简单占比优先使用：

```
环图条形图
百分比堆叠图
```

复杂图表只有在业务目标明确、数据结构匹配、阅读价值高于理解成本时才使用。

---

## 8. 展示空间对选型的影响

### 8.1 小卡片

优先使用：

```
指标卡迷你趋势图简化柱状图简化条形图环图进度条
```

避免使用：

```
复杂地图桑基图多系列折线图复杂热力图雷达图
```

---

### 8.2 标准分析卡片

优先使用：

```
柱状图条形图折线图面积图堆叠柱状图散点图漏斗图热力图
```

---

### 8.3 大屏 / 监控

优先使用：

```
指标卡实时折线图Top 排名地图热力图告警列表
```

注意：

```
大屏不等于可以过度装饰状态色必须服务业务含义远距离阅读清晰度优先
```

---

### 8.4 移动端

优先使用：

```
指标卡折线图条形图环图简化趋势图列表
```

避免使用：

```
复杂图例密集标签复杂地图桑基图多系列复杂图表
```

## 9. 图表复杂度优先级

AI 应优先选择复杂度较低但足以回答问题的图表。

|复杂度|图表类型|使用建议|
|---|---|---|
|Level 1|指标卡、柱状图、条形图、折线图、表格|默认优先|
|Level 2|面积图、堆叠柱状图、环图、热力图|有明确目标时使用|
|Level 3|散点图、气泡图、漏斗图、雷达图、箱线图|需要一定理解成本|
|Level 4|桑基图、迁徙地图、双轴图、复杂复合图|谨慎使用|
|Level 5|网络图、3D 图表、强装饰图表|默认避免|

原则：

```
能用 Level 1 解决，不使用 Level 2能用 Level 2 解决，不使用 Level 3复杂图表必须说明必要性
```

## 10. 选型后必须查阅的文档

图表初步选型后，AI 必须继续查阅对应文档。

### 10.1 查阅图表能力矩阵

```
02-chart-type/
└── capability-matrix.md
```

用于确认：

```
该图表是否支持当前 primary_intent是否支持 secondary_intents是否存在更合适的替代图表
```

### 10.2 查阅具体图表文档

例如：

```
基础图表 Basic-charts/01.柱状图 Bar-chart.md基础图表 Basic-charts/03.折线图 Line-chart.md统计图表 Statistical-charts/04.气泡图 Bubble-chart.md复合图表 Composite-charts/02.堆叠柱状图 Stacked-bar-chart.md
```

用于确认：

```
适用场景不适用场景数据字段要求视觉规则交互规则AI warning
```

### 10.3 查阅视觉语言文档

```
01-design-language/
```

用于确认：

```
颜色字体坐标轴网格线图例标签Tooltip状态
```

涉及代码生成时，必须查阅：


## 12. 全局禁止项

AI 在图表选型中禁止以下行为：

```
禁止仅根据目录名称判断图表用途
禁止随机选择图表
禁止未经判断直接使用饼图
禁止为了视觉丰富使用复杂图表
禁止在没有时间字段时使用趋势图
禁止在没有 source / target 字段时使用桑基图禁止在指标未标准化时使用雷达图
禁止在没有业务关系时使用双轴图
禁止生成没有字段映射说明的图表方案
禁止跳过 02-chart-type/capability-matrix.md 和具体图表文档
```
