# 01.柱线混合图 Bar-line Chart

## 1. 文档定位

柱线混合图用于在同一笛卡尔坐标系中组合柱状图和折线图，表达同一维度下两个相关指标的对比、趋势和联动关系。

本文档用于约束 AI 在生成柱线混合图时的图表判断、数据字段映射、视觉构成、交互行为和工程映射，避免把柱线混合图误用为任意两个指标的拼接图，或在不需要复合表达时增加阅读成本。

柱线混合图的基础构成来自柱状图、折线图和直角坐标系规范。柱体负责表达分类或时间维度下的数值量级，折线负责表达趋势、比率、效率、目标线或衍生指标。若两个指标单位和值域差异明显，柱线混合图通常会同时成为双轴图；若两个指标单位一致，也可以使用同一 Y 轴。

## 2. 图表定义

柱线混合图是在同一横轴维度上叠加柱体系列和折线系列的复合图表。

常见结构：

- X 轴：时间、分类或有序维度。
- 柱体系列：表达量级、规模、计数、金额、成交量等绝对值。
- 折线系列：表达趋势、比率、转化率、效率、目标值、均值或对比基准。
- Y 轴：可为单轴，也可根据单位差异拆分为左右双轴。

柱线混合图的重点是让用户同时观察“量”和“变化 / 效率 / 比率”的关系。例如成交金额用柱体展示规模，转化率用折线展示效率，用户可判断高规模是否伴随高转化。

## 3. 属性摘要

| 属性 | 规则 |
|---|---|
| 图表类型 | 复合图表 |
| 图形形态 | 柱体 + 折线 |
| 核心功能 | 规模对比、趋势联动、绝对值与比率对照、目标 / 实际对比 |
| 适合的数据 | 一个共享维度字段 + 至少两个相关指标字段 |
| 数据与图形映射 | 横轴映射共享维度；柱体映射绝对量或主指标；折线映射趋势、比率、效率、目标或辅助指标 |
| 数据体量 | 分类或时间点不宜过多；同图建议 1-2 个柱体系列 + 1-2 条折线，最多不超过 5 个总系列 |

## 4. 适用场景

柱线混合图适用于以下场景：

| 场景 | 使用条件 | 价值 |
|---|---|---|
| 绝对量 + 比率 | 一个指标是规模值，另一个指标是比例或效率 | 同时观察规模和质量 |
| 量级 + 趋势 | 柱体展示每期数值，折线展示连续变化 | 兼顾分类比较和趋势判断 |
| 目标 / 实际对照 | 柱体展示实际值，折线展示目标、均值或基准 | 判断达成情况和偏离程度 |
| 活动或周期复盘 | 时间维度下同时关注成交规模和转化效率 | 识别高量低效或低量高效区间 |
| 多指标联动 | 两个指标有明确业务因果、约束或解释关系 | 辅助判断指标之间是否同向变化 |

典型业务问题：

- 每日成交金额变化如何，成交转化率是否同步提升？
- 各渠道流量规模和转化率是否匹配？
- 活动期间订单量上涨时，客单价是否发生变化？
- 不同品类销售额和毛利率之间是否存在冲突或机会？

## 5. 不适用场景

以下场景不应优先使用柱线混合图：

| 不适用场景 | 原因 | 替代方向 |
|---|---|---|
| 两个指标无明确业务关系 | 会制造虚假的联动解读 | 分开展示两个图表 |
| 只有单一指标 | 不需要复合图表 | 柱状图或折线图 |
| 两个指标都只是分类比较 | 折线会暗示不存在的连续关系 | 分组柱状图、条形图 |
| 指标过多 | 图例和 Tooltip 复杂，用户难以判断主次 | 看板拆分、分面、小多图 |
| 需要精确比较每个指标 | 复合图更适合关系观察，不适合精确查数 | 表格、指标卡 + 趋势图 |
| 横轴为无序分类且折线指标无顺序含义 | 折线连接会暗示连续路径 | 柱状图、点图、分组图 |

## 6. 数据结构

### 6.1 基础字段

| 字段角色 | 是否必需 | 说明 |
|---|---|---|
| 维度字段 | 必需 | 映射到 X 轴，应为时间、有序分类或可稳定排序的业务分类 |
| 柱体指标 | 必需 | 映射到柱体，通常为金额、数量、订单数、流量等绝对量 |
| 折线指标 | 必需 | 映射到折线，通常为比率、效率、均值、目标值或辅助趋势指标 |
| 系列字段 | 可选 | 用于多个柱体系列或多个折线系列 |
| 单位字段 | 推荐 | 明确每个指标的单位、统计口径和展示精度 |

### 6.2 数据要求

- 柱体指标和折线指标必须共享同一 X 轴维度。
- 柱体指标和折线指标必须存在明确业务关系。
- 同一图中总系列数量不建议超过 5 个。
- 若横轴是时间，必须按时间顺序排列。
- 若横轴是分类，必须说明分类顺序依据，例如业务顺序、指标排序或固定枚举顺序。
- 不同单位指标不得强行放在同一 Y 轴；需要时应升级为双轴柱线混合图。

### 6.3 数据示例

```json
[
  { "date": "2026-05-01", "gmv": 120, "conversionRate": 0.042 },
  { "date": "2026-05-02", "gmv": 136, "conversionRate": 0.047 },
  { "date": "2026-05-03", "gmv": 118, "conversionRate": 0.039 }
]
```

字段映射：

| 语义 | 字段 |
|---|---|
| 横轴 | `date` |
| 柱体 | `gmv` |
| 折线 | `conversionRate` |

## 7. 视觉构成

| 构成元素 | 作用 | 是否必需 |
|---|---|---|
| 横轴 xAxis | 承载共享时间、分类或有序维度 | 必需 |
| 左侧 Y 轴 | 默认承载柱体指标 | 必需 |
| 右侧 Y 轴 | 承载单位不同的折线指标 | 按需启用 |
| 柱体 seriesBar | 表达主指标的绝对量或分类比较 | 必需 |
| 折线 seriesLine | 表达辅助指标的趋势、比率或基准 | 必需 |
| 图例 legend | 区分柱体和折线系列 | 必需 |
| Tooltip | 聚合同一维度下的柱体值和折线值 | 建议启用 |
| 网格线 grid | 辅助读取主数值轴 | 建议启用 |
| 标签 label | 标记关键柱体或折线点 | 可选 |

视觉层级：

```txt
grid / axis
→ bar series
→ line series
→ hover symbol / interaction layer
→ Tooltip
```

折线必须绘制在柱体上方，避免被柱体遮挡。柱体不应使用过高透明度削弱分类比较，但在折线可读性受影响时，可以适度降低柱体饱和度或选择更克制的柱体颜色。

## 8. 设计原则

### 8.1 明确主次关系

柱线混合图必须明确主指标和辅助指标。

- 默认柱体为主指标，表达规模或数量。
- 默认折线为辅助指标，表达趋势、比率、效率、目标或基准。
- 如果折线指标才是主分析对象，应确认是否更适合折线图 + 柱体辅助背景，或拆分为两个图表。

### 8.2 谨慎使用双轴

当柱体和折线单位不同或值域差异明显时，可以使用左右双轴。

执行规则：

- 左轴默认对应柱体，右轴默认对应折线。
- 左右轴标题必须显示指标名称和单位。
- 左右轴刻度数量应保持一致，默认约 5 个刻度。
- 左右轴标签对齐规则必须继承坐标轴规范：左轴右对齐，右轴左对齐。
- 不得通过左右轴缩放制造虚假的同步趋势。

如果两个指标可以统一单位、归一化或转成同一口径，应优先统一尺度，而不是默认使用双轴。

### 8.3 控制系列数量

柱线混合图的阅读成本高于单一图表。

推荐组合：

- 1 个柱体系列 + 1 条折线。
- 1 个柱体系列 + 2 条折线。
- 2 个柱体系列 + 1 条折线。

不推荐：

- 多组分组柱 + 多条折线同时出现。
- 超过 5 个总系列。
- 柱体和折线都使用多个系列且没有明确主次。

### 8.4 图例形态必须区分图形语义

图例和 Tooltip icon 必须按系列真实形态绘制：

- 柱体系列使用 `rectangle`，建议 `10 × 10px`。
- 折线系列使用 `line`，建议 `10 × 2px`。
- 虚线目标或预测线应在图例中保持虚线语义。

不得为了统一外观把所有图例项都改成圆点或色块。

### 8.5 柱线配色与折线可读性

柱线混合图的系列配色可以遵循分类色板，但不能只按色板顺序机械取色。由于柱体通常占据更大的面积，折线必须保证在柱体、网格线和 hover 状态上方仍然清晰可读。

执行规则：

- 柱体颜色优先使用分类色板中的主系列色，默认可使用 `分类色板/item-1` 或业务指定主色。
- 折线颜色应与柱体颜色形成足够区分，优先选择分类色板中与柱体色具有补色关系、冷暖对比或显著明度 / 色相差异的颜色。
- 如果分类色板中存在合适的互补色或高区分色，必须优先从分类色板中选择，不新增临时颜色。
- 如果分类色板无法提供足够可读的折线颜色，允许定义一种自定义折线色，但必须在输出中说明 `custom_line_color`、使用原因和建议沉淀为主题变量的路径。
- 自定义折线色只能用于增强折线可读性，不得破坏整体视觉语言、品牌色约束或无障碍对比要求。
- 多条折线同时存在时，应先确认是否需要多折线；若确实需要，应优先使用分类色板中的高区分色组，而不是为每条线都新增自定义色。
- 折线颜色、图例 line shape、Tooltip line icon 和 hover symbol 必须保持一致。

不推荐：

- 柱体和折线使用相近蓝色、相近绿色或同色不同透明度，导致折线被柱体吞没。
- 为折线选择过浅颜色，使其在浅色网格线、柱体或背景上不可读。
- 只依靠线宽、点位或 Tooltip 弥补折线配色不足。

### 8.6 坐标轴和网格线

柱线混合图默认继承直角坐标系布局规格。

- X 轴位于底部，承载共享维度。
- 网格线默认跟随主数值轴，通常为左侧 Y 轴。
- 若启用右轴，右轴只显示标签和标题，不额外生成第二套密集网格线。
- 左右轴标题必须与对应系列建立清晰关系，可通过颜色、图例顺序和文案对应。
- 轴标签字号、颜色、间距必须继承坐标轴规范。

### 8.7 Hover 规则

柱线混合图 hover 应继承柱状图的类目高亮，而不是折线图的垂直指示线。

执行规则：

- axisPointer 使用 `shadow` / 类目高亮矩形。
- 高亮矩形宽度等于当前类目完整占位宽度。
- 高亮矩形填充 `rgba(0,0,0,0.03)`。
- 折线 hover 点可以显示在当前类目对应的折线位置。
- 不得同时显示类目高亮矩形和垂直 line 指示器。

## 9. 交互规则

| 交互 | 启用条件 | 系统反馈 |
|---|---|---|
| 悬停 Tooltip | 需要读取同一维度下多个指标 | 显示类目、柱体值、折线值、单位和必要说明 |
| 类目高亮 | 用户悬停某个 X 轴类目 | 显示完整类目占位宽度的高亮矩形 |
| 折线点高亮 | 悬停类目存在折线值 | 显示当前折线 hover symbol |
| 图例筛选 | 存在多个系列 | 支持隐藏、显示或高亮系列 |
| dataZoom / 滚动 | 时间跨度长或分类数量多 | 聚焦局部区间，保持柱体和折线联动 |
| 标记线 | 存在目标、均值、阈值或活动区间 | 显示目标线、参考线或背景区间 |

Tooltip 内容顺序建议：

```txt
当前时间 / 类目
柱体系列：指标名 + 数值 + 单位
折线系列：指标名 + 数值 + 单位
可选 note：口径说明或联动解读
```

Tooltip 必须使用两列结构，数值列右对齐；icon 必须与图例形态一致。

## 11. G2 / ECharts 映射

### 11.1 G2

```js
chart
  .interval()
  .data(data)
  .encode("x", "date")
  .encode("y", "gmv")
  .encode("color", "成交金额")
  .style("maxWidth", 40);

chart
  .line()
  .data(data)
  .encode("x", "date")
  .encode("y", "conversionRate")
  .encode("color", "成交转化率")
  .axis("y", { position: "right" });
```

基础映射：

- 柱体使用 `interval` mark。
- 折线使用 `line` mark。
- 两个 mark 必须共享同一 X 轴。
- 若启用双轴，必须显式配置左右轴位置、标题、单位和刻度。
- Tooltip 应按同一 X 维度聚合柱体和折线值。

### 11.2 ECharts

```js
option = {
  tooltip: {
    trigger: "axis",
    axisPointer: { type: "shadow" }
  },
  legend: {},
  xAxis: {
    type: "category",
    data: categories
  },
  yAxis: [
    {
      type: "value",
      name: "成交金额（万元）"
    },
    {
      type: "value",
      name: "成交转化率（%）"
    }
  ],
  series: [
    {
      name: "成交金额",
      type: "bar",
      yAxisIndex: 0,
      data: gmvValues,
      barMaxWidth: 40,
      itemStyle: { borderRadius: [1, 1, 0, 0] }
    },
    {
      name: "成交转化率",
      type: "line",
      yAxisIndex: 1,
      data: rateValues,
      showSymbol: false,
      lineStyle: { width: 2.5 }
    }
  ]
};
```

ECharts 中柱线混合图必须明确每个 series 的 `type` 和 `yAxisIndex`。若两个指标共用同一单位，可只保留一个 `yAxis`。

## 12. AI 输出格式

AI 生成柱线混合图时，应输出：

```json
{
  "chartType": "bar-line",
  "purpose": "volume-trend | volume-rate | target-actual | metric-correlation",
  "dataMapping": {
    "x": "共享时间或分类字段",
    "bar": "绝对量或主指标字段",
    "line": "比率、效率、目标或辅助趋势字段",
    "category": "可选系列字段"
  },
  "axis": {
    "mode": "single-axis | dual-axis",
    "leftYAxis": "柱体指标名称和单位",
    "rightYAxis": "折线指标名称和单位"
  },
  "style": {
    "bar": "rectangle legend; barMaxWidth <= 40; top radius 1px",
    "line": "line legend; lineWidth 2.5px; hover symbol 8px",
    "axisPointer": "shadow; rgba(0,0,0,0.03)"
  },
  "interaction": {
    "tooltip": true,
    "legendFilter": true,
    "dataZoom": "横轴密集时启用"
  }
}
```

## 13. 禁止项

- 禁止在两个指标无明确业务关系时使用柱线混合图。
- 禁止仅为了节省空间把两个图表强行合并。
- 禁止把无序分类字段连成折线。
- 禁止在不同单位指标之间强行共用同一 Y 轴。
- 禁止通过双轴缩放制造虚假的趋势相关性。
- 禁止超过 5 个总系列后仍继续叠加柱体和折线。
- 禁止将柱体和折线图例统一成同一种 shape。
- 禁止柱体和折线使用过于接近的颜色，导致折线在柱体上方不可读。
- 禁止在分类色板已有合适高区分色时随意新增自定义折线色。
- 禁止新增自定义折线色但不说明使用原因、色值和后续 token 沉淀建议。
- 禁止柱线混合图 hover 同时使用类目高亮矩形和垂直 line 指示器。
- 禁止省略左右轴单位、指标名或系列对应关系。
- 禁止用复合图替代清晰的上下排列图表。

## 14. AI 自查清单

生成柱线混合图前检查：

- 是否存在一个共享 X 轴维度？
- 柱体指标和折线指标是否存在明确业务关系？
- 柱体是否表达绝对量、规模或主指标？
- 折线是否表达趋势、比率、效率、目标或辅助指标？
- 是否需要双轴？如果需要，左右轴单位是否明确？
- 左右轴是否约 5 个等距刻度，是否避免制造虚假相关？
- 总系列数量是否不超过 5 个？
- 折线颜色是否与柱体形成足够区分，并优先从分类色板选择？
- 如果使用自定义折线色，是否说明 `custom_line_color`、原因和 token 沉淀建议？
- 图例 shape 是否按柱体 `rectangle`、折线 `line` 区分？
- Tooltip 是否聚合同一类目下的柱体和折线值？
- Hover 是否使用柱线混合图要求的 `shadow` 指示器？
- 是否引用柱状图、折线图、坐标轴、图例和 Tooltip 上游规范？
