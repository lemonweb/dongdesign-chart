# 02.双轴图 Dual-axis Chart

## 1. 文档定位

双轴图用于在同一笛卡尔坐标系中使用两个 Y 轴展示两个存在业务关系但单位或值域不同的指标。

本文档用于约束 AI 在生成双轴图时的图表判断、数据字段映射、左右轴关系、视觉构成、交互行为和工程映射，避免双轴图被滥用于无关指标拼接、虚假相关和不清晰的单位对照。

双轴图是一类轴结构，不等同于某一种具体图形。它可以表现为柱线混合图、双折线图、柱柱双轴图、面积线双轴图等。柱线混合图是双轴图的常见形态之一，但并非所有柱线混合图都必须使用双轴。

## 2. 图表定义

双轴图是在同一 X 轴维度下，同时使用左侧 Y 轴和右侧 Y 轴分别承载不同数值尺度的复合图表。

常见结构：

- X 轴：共享时间、分类或有序维度。
- 左侧 Y 轴：通常承载主指标、规模指标或柱体系列。
- 右侧 Y 轴：通常承载辅助指标、比率指标、效率指标或另一单位指标。
- 数据系列：可以是柱体、折线、面积、点或其他笛卡尔坐标图元组合。

双轴图的核心价值是同时观察两个相关指标在同一维度上的变化关系。它不是为了节省空间，而是为了比较两个指标的联动、偏离、同步或解释关系。

## 3. 属性摘要

| 属性 | 规则 |
|---|---|
| 图表类型 | 复合图表 / 坐标轴结构 |
| 图形形态 | 两套 Y 轴 + 一个共享 X 轴 + 两个或多个数据系列 |
| 核心功能 | 异单位对照、值域差异对照、趋势联动、主辅指标解释 |
| 适合的数据 | 一个共享维度字段 + 两个存在业务关系但单位或值域不同的指标 |
| 数据与图形映射 | X 轴映射共享维度；左 Y 轴映射主指标；右 Y 轴映射辅助指标；系列形态按图表语义选择 |
| 数据体量 | 适合中低复杂度对照；总系列数量不建议超过 4 个，最多不超过 5 个 |

## 4. 适用场景

双轴图适用于以下场景：

| 场景 | 使用条件 | 价值 |
|---|---|---|
| 异单位指标联动 | 两个指标单位不同但存在业务关系 | 同时观察规模和效率 |
| 值域差异较大 | 两个指标数量级差异明显，单轴会压扁一方 | 保留双方可读性 |
| 主辅指标解释 | 一个指标用于解释另一个指标变化 | 支持业务诊断 |
| 目标 / 实际 / 比率对照 | 绝对量与完成率、转化率、占比共同分析 | 判断结果和质量是否一致 |
| 周期复盘 | 同一时间轴下观察两个相关指标是否同步或背离 | 定位异常区间 |

典型业务问题：

- 成交金额增长时，成交转化率是否同步提升？
- 曝光量增加时，点击率是否下降？
- 订单数变化和客单价变化是否存在反向关系？
- 活动期间销售额和毛利率是否出现背离？

## 5. 不适用场景

以下场景不应优先使用双轴图：

| 不适用场景 | 原因 | 替代方向 |
|---|---|---|
| 两个指标没有明确业务关系 | 用户会误以为二者存在关联 | 分开展示两个图表 |
| 只是为了节省空间 | 双轴会增加认知成本 | 上下排列两个趋势图 |
| 两个指标可以统一单位或归一化 | 不必引入第二套坐标轴 | 单轴图、指数化折线、百分比变化图 |
| 需要精确比较两个指标大小 | 不同轴尺度无法直接比较高低 | 表格、指标卡、并列图 |
| 轴单位不清晰 | 用户无法判断系列与轴的对应关系 | 补充轴标题或拆分图表 |
| 系列过多 | 图例、轴、Tooltip 复杂度过高 | 分面、小多图、筛选 |

## 6. 数据结构

### 6.1 基础字段

| 字段角色 | 是否必需 | 说明 |
|---|---|---|
| 维度字段 | 必需 | 映射到共享 X 轴，应为时间、有序分类或稳定排序分类 |
| 左轴指标 | 必需 | 映射到左侧 Y 轴，通常为主指标或规模指标 |
| 右轴指标 | 必需 | 映射到右侧 Y 轴，通常为辅助指标、比率、效率或异单位指标 |
| 系列形态 | 必需 | 决定每个指标用柱体、折线、面积或点表示 |
| 单位 / 口径 | 必需 | 每个轴必须明确指标名称、单位和统计口径 |

### 6.2 数据要求

- 左右轴指标必须共享同一 X 轴维度。
- 左右轴指标必须存在明确业务关系，例如因果、约束、解释、转化、效率或目标关系。
- 左右轴指标单位或值域应确实存在差异；若不存在，应优先使用单轴图。
- 左右轴刻度数量应尽量一致，默认约 5 个刻度。
- 左右轴默认从 0 起步；若因业务需要使用非零起点，必须在轴标题、说明或输出中明确说明。
- 不得用双轴缩放来人为制造趋势同步。

### 6.3 数据示例

```json
[
  { "date": "2026-05-01", "gmv": 120, "conversionRate": 0.042 },
  { "date": "2026-05-02", "gmv": 136, "conversionRate": 0.047 },
  { "date": "2026-05-03", "gmv": 118, "conversionRate": 0.039 }
]
```

字段映射：

| 语义 | 字段 | 轴 |
|---|---|---|
| 横轴 | `date` | X 轴 |
| 主指标 | `gmv` | 左 Y 轴 |
| 辅助指标 | `conversionRate` | 右 Y 轴 |

## 7. 视觉构成

| 构成元素 | 作用 | 是否必需 |
|---|---|---|
| 横轴 xAxis | 承载共享时间、分类或有序维度 | 必需 |
| 左侧 Y 轴 leftYAxis | 承载主指标刻度、单位和标题 | 必需 |
| 右侧 Y 轴 rightYAxis | 承载辅助指标刻度、单位和标题 | 必需 |
| 主系列 primarySeries | 展示左轴指标 | 必需 |
| 辅助系列 secondarySeries | 展示右轴指标 | 必需 |
| 图例 legend | 建立系列、颜色、图形形态和轴的对应关系 | 必需 |
| Tooltip | 聚合同一 X 维度下左右轴指标值 | 建议启用 |
| 网格线 grid | 辅助读取主轴刻度 | 建议启用 |
| 轴标题 axisTitle | 说明左右轴指标和单位 | 必需 |

默认视觉组织：

- 左轴位于绘图区左侧，标签右对齐。
- 右轴位于绘图区右侧，标签左对齐。
- 左右轴标签与绘图区边界间距均为 `10px`。
- 网格线默认跟随主轴，避免左右轴各自生成两套网格线。
- 数据系列应通过颜色、图例 shape 和轴标题建立清晰对应关系。

## 8. 设计原则

### 8.1 双轴必须服务业务关系

双轴图必须回答“两个指标之间有什么关系”，而不是“我想同时看两个指标”。

可接受关系：

- 规模与效率，例如 GMV 与转化率。
- 流量与质量，例如曝光量与点击率。
- 结果与成本，例如成交金额与投放成本。
- 绝对值与占比，例如订单数与退款率。
- 实际表现与目标 / 基准，例如销售额与目标完成率。

若无法说明关系，应拆成两个图表。

### 8.2 左右轴单位必须清晰

双轴图的首要风险是用户不知道哪个系列对应哪个轴。

执行规则：

- 左右轴标题必须显示指标名称和单位。
- 图例项名称必须使用业务名称，不使用字段名。
- 图例颜色和系列颜色必须一致。
- 轴标题可与对应系列色建立弱关联，但不得让轴颜色喧宾夺主。
- Tooltip 必须展示每个指标的单位和口径。

### 8.3 轴尺度必须克制

左右轴可以不同尺度，但不能误导。

执行规则：

- 左右轴默认从 `0` 起步。
- 左右轴默认都使用约 5 个等距刻度。
- 不得为了让两条线看起来同步而手动调整轴最大值。
- 如果使用非零起点、对数轴或特殊尺度，必须显式说明。
- 如果两个指标可以统一成指数、百分比变化或标准化值，应优先考虑单轴替代方案。

### 8.4 系列形态应表达数据语义

双轴图中的系列形态不能随意选择。

| 指标语义 | 推荐形态 |
|---|---|
| 规模、数量、金额、订单数 | 柱体 |
| 比率、效率、趋势、目标完成率 | 折线 |
| 两个连续趋势指标 | 双折线 |
| 累计量或区间体量 | 面积，但需谨慎 |
| 离散事件或异常点 | 点或标记 |

如果一个指标是比率或效率，优先使用折线；如果一个指标是分类量级，优先使用柱体。

### 8.5 控制复杂度

双轴图属于高复杂度图表，应限制系列数量。

- 推荐：2 个指标，2 个系列。
- 可接受：3-4 个系列，但必须有明确分组和图例。
- 上限：不超过 5 个总系列。
- 超过上限时，应拆分为上下排列、分面或看板组合。

### 8.6 图例和 Tooltip 必须与图形形态一致

图例 shape 必须按系列真实形态选择：

- 柱体使用 `rectangle`。
- 折线使用 `line`。
- 散点或事件点使用 `dot`。
- 面积图一般使用 `rectangle`，必要时说明边界线语义。

Tooltip icon 必须与图例 shape 一致。不得用圆点或色块替代所有系列。

## 9. 交互规则

| 交互 | 启用条件 | 系统反馈 |
|---|---|---|
| 悬停 Tooltip | 需要读取同一维度下左右轴指标 | 展示 X 维度、左轴指标值、右轴指标值、单位和口径 |
| axisPointer | 按主图形形态选择 | 柱线混合使用 `shadow`；双折线使用垂直 `line` |
| 图例筛选 | 多系列或用户需要聚焦某指标 | 支持隐藏、显示或高亮系列 |
| dataZoom | 时间跨度长或分类数量多 | 左右轴系列必须同步缩放 |
| 标记线 / 标记区间 | 存在目标、阈值、活动阶段 | 标注对应轴和指标 |

Tooltip 内容顺序建议：

```txt
当前时间 / 类目
左轴指标：名称 + 数值 + 单位
右轴指标：名称 + 数值 + 单位
可选 note：口径说明、非零起点说明或尺度说明
```

axisPointer 选择规则：

- 柱线混合双轴图：使用柱线混合图规则，`shadow` / 类目高亮矩形。
- 双折线双轴图：使用折线图规则，单根垂直 `1px` 实线。
- 柱柱双轴图：使用柱状图规则，`shadow` / 类目高亮矩形。

## 10. 与视觉语言的关系

双轴图依赖以下视觉语言文档：

| 依赖文档 | 使用方式 |
|---|---|
| `01-design-language/theme-token.md` | 绑定系列色、坐标轴、图例、Tooltip 和交互状态变量 |
| `01-design-language/color.md` | 判断主系列、辅助系列、语义标记和弱化状态 |
| `01-design-language/typography.md` | 控制左右轴标签、轴标题、图例、Tooltip 的字号和字重 |
| `01-design-language/axis.md` | 继承异轴 / 双轴的左右标签、单位、刻度和布局规则 |
| `01-design-language/legend.md` | 按图形语义使用 `rectangle`、`line`、`dot` |
| `01-design-language/label.md` | 控制关键点、最大值、最小值和异常值标注 |
| `01-design-language/tooltip.md` | 使用同维度聚合 Tooltip，并保持 icon 与图例一致 |

执行约束：

- 双轴图不得单独发明右轴间距，应继承右侧异轴标签左内距 `10px`。
- 右轴标签必须左对齐，不得复用左轴的右对齐规则。
- 左右轴标题必须同时存在，除非图表上下文已经以等价方式明确说明。
- 网格线必须裁切在 `plotArea` 内，不得延伸到左右轴标签轨道。

## 11. G2 / ECharts 映射

### 11.1 G2

```js
chart
  .interval()
  .data(data)
  .encode("x", "date")
  .encode("y", "gmv")
  .encode("color", "成交金额")
  .axis("y", {
    position: "left",
    title: "成交金额（万元）"
  });

chart
  .line()
  .data(data)
  .encode("x", "date")
  .encode("y", "conversionRate")
  .encode("color", "成交转化率")
  .axis("y", {
    position: "right",
    title: "成交转化率（%）"
  });
```

基础映射：

- 每个轴必须声明指标名称、单位和位置。
- 每个系列必须明确绑定对应 Y 轴。
- 多 mark 必须共享同一 X 轴。
- Tooltip 必须按同一 X 维度聚合读数。

### 11.2 ECharts

```js
option = {
  tooltip: {
    trigger: "axis"
  },
  legend: {},
  xAxis: {
    type: "category",
    data: categories
  },
  yAxis: [
    {
      type: "value",
      name: "成交金额（万元）",
      position: "left"
    },
    {
      type: "value",
      name: "成交转化率（%）",
      position: "right"
    }
  ],
  series: [
    {
      name: "成交金额",
      type: "bar",
      yAxisIndex: 0,
      data: gmvValues
    },
    {
      name: "成交转化率",
      type: "line",
      yAxisIndex: 1,
      data: rateValues
    }
  ]
};
```

ECharts 中必须为每个 series 显式指定 `yAxisIndex`。若 `axisPointer` 类型与系列组合有关，应根据图形形态选择 `shadow` 或 `line`，不得统一套用默认值。

## 13. 禁止项

- 禁止在两个指标无明确业务关系时使用双轴图。
- 禁止仅为了节省空间使用双轴图。
- 禁止省略左右轴指标名称、单位或口径。
- 禁止不同单位指标强行共用同一 Y 轴。
- 禁止通过调整左右轴范围制造虚假的趋势同步。
- 禁止在双轴图中让左右轴刻度数量明显不一致。
- 禁止右轴标签复用左轴右对齐规则。
- 禁止图例 shape 与系列真实形态不一致。
- 禁止 Tooltip 只展示当前系列，导致用户无法读取同维度另一轴指标。
- 禁止超过 5 个总系列仍继续使用单个双轴图。
- 禁止在可以统一尺度或归一化时默认使用双轴。

## 14. AI 自查清单

生成双轴图前检查：

- 两个指标是否共享同一 X 轴维度？
- 两个指标是否存在明确业务关系？
- 是否确实存在单位差异或值域差异？
- 是否可以用单轴、归一化、上下排列或分面替代？
- 左右轴标题是否同时包含指标名称和单位？
- 每个系列是否明确绑定左轴或右轴？
- 左右轴刻度数量是否约 5 个且可解释？
- 是否默认从 0 起步？若不是，是否明确说明？
- 图例 shape 是否与柱体、折线、面积或点的真实形态一致？
- Tooltip 是否能同时读取左右轴指标值和单位？
- axisPointer 是否按图形组合选择，而不是机械套用？
- 总系列数量是否不超过 5 个？
