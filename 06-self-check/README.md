## 1. 目录定位

本目录用于存放标准示例、设计走查清单、代码走查清单和常见错误案例。
用于帮助设计师和 AI 判断生成结果是否符合数据可视化设计体系。

## 2. 使用时机

以下情况应读取本目录：

- 需要参考标准图表示例
- 需要检查设计稿是否合规
- 需要检查 G2 / ECharts 代码是否合规
- 需要识别常见错误
- 需要在交付前做最终自查

## 3. 读取顺序

建议按以下顺序读取：

1. `01.设计走查清单 Design-checklist.md`
2. `02.代码走查清单 Code-checklist.md`
3. `bad-cases.md`
4. `vibedesign-validation-examples.md`
5. 标准示例相关文档

## 4. 使用原则

- 示例只能作为参考，不替代设计规范。
- 自查清单用于发现问题，不负责定义完整规范。
- 如果发现问题，应回到对应规范文档修正。
- 代码问题应回到 `04-adaptation/` 校验。
- 设计问题应回到 `01-design-language/` 或 `02-chart-type/` 校验。

## 5. 不在本目录处理的内容

- 图表选型规则：查阅 `02-chart-type/`
- 视觉语言定义：查阅 `01-design-language/`
- 业务场景组合：查阅 `03-pattern/`
- 引擎实现标准：查阅 `04-adaptation/`
