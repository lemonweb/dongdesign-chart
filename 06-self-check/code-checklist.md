# 代码自查清单 Code Checklist

## 1. 文档定位

本文用于检查 AI 生成的 G2 / ECharts / SVG / Canvas 图表代码是否符合 Wiki 的图表选型、视觉语言和交互规范。

本文只做交付前检查，不替代具体规范。发现问题时，应回到对应文档修正。

## 2. 选型与字段

- [ ] 是否明确 `primary_intent`？
- [ ] 是否说明 `secondary_intents`？
- [ ] 是否输出字段映射：`x / y / color / size / tooltip`？
- [ ] 时间字段是否映射到时间轴或有序轴？
- [ ] 不同单位指标是否避免强行共用同一 Y 轴？
- [ ] 复杂图表是否说明必要性？

## 3. Token 与样式

- [ ] 系列色是否来自 `分类色板/item-*`、`有序色板/*`、`发散色板/*` 或 `语义色板/*`？
- [ ] 坐标轴颜色是否绑定 `坐标轴/*`？
- [ ] 图例文字是否绑定 `图例/legendTextColor`？
- [ ] Tooltip 背景和指示线是否绑定 `提示/*`？
- [ ] 是否没有写入 Wiki 未定义的颜色、字号、线宽或阴影？
- [ ] 是否同时说明绑定变量和当前模式显示值？

## 4. 字体排版

- [ ] 坐标轴标签是否显式设置为 `12px / 18px / 400`？
- [ ] 图例文字是否显式设置为 `12px / 18px / 400`？
- [ ] Tooltip 正文是否显式设置为 `12px / 18px / 400`？
- [ ] Tooltip 标题是否显式设置为 `12px / 18px / 600`？
- [ ] Tooltip value 是否显式设置为 `12px / 18px / 600`，且 label / note 保持 `12px / 18px / 400`？
- [ ] 是否没有使用非规范字号，例如 `10 / 11 / 13 / 15 / 18 / 20 / 24`？
- [ ] 是否没有使用负 letter spacing 或 viewport 字号缩放？

## 5. 坐标轴

- [ ] X 轴、Y 轴类型是否与字段类型一致？
- [ ] 时间轴标签是否抽样显示，保留首尾和等步长标签？
- [ ] Y 轴标签是否右对齐，且与绘图区有足够间距？
- [ ] 柱状图、折线图等笛卡尔坐标系图表的数值轴是否默认从 `0` 起步？
- [ ] Y 轴是否使用 `splitNumber = 4`，即 5 个刻度 / 4 个等距区间？
- [ ] 数值轴上界是否避免过大留白，`axisMax / dataMax` 默认是否不超过 `1.25`？
- [ ] 网格线 / 分隔线是否为 `1px` 虚线，而轴线和 hover 指示线仍为实线？
- [ ] 轴标题是否优先在 Y 轴标签上方显示；若使用竖排标题，是否有独立 title rail？
- [ ] 轴标题位于上方时，是否有 title band，且没有覆盖首个 Y 轴标签？
- [ ] 轴标题是否与图表数据、轴标签不重叠？
- [ ] 网格线 / 刻度数量是否克制，默认约 `5` 个，并保持线性等距？
- [ ] 是否没有使用数据最小值起点、密集刻度或无说明的非均匀刻度？
- [ ] 折线图 hover 指示线是否为 `1px` 实线？
- [ ] 多分面共享时间轴时，是否避免每个 plot 重复挤满 X 轴标签？

## 6. 图例

- [ ] 图例是否真的需要显示？
- [ ] 图例 label 是否使用业务名称，而不是字段名？
- [ ] 图例顺序是否与系列顺序一致？
- [ ] 折线图图例 shape 是否为 `10 × 2px` line，圆角 `2px`？
- [ ] 柱状 / 面积 / 饼图是否使用 `10 × 10px` rectangle？
- [ ] 散点 / 气泡图是否使用 `10 × 10px` dot？
- [ ] 图例 shape 与 Tooltip icon 是否保持同一语义？
- [ ] 图例筛选或高亮状态是否可见？

## 7. 数据点、标签与标注

- [ ] 折线图是否默认隐藏所有普通数据点？
- [ ] 数据点是否只在 hover、选中、最新值、最大值、最小值、异常点等场景显示？
- [ ] 散点 / 气泡图普通态是否没有默认白色描边？
- [ ] 是否避免给每个数据点都显示 label？
- [ ] 散点 / 气泡图显式开启全部标签时，是否显示所有点位标签，并开启防重叠和白色文字描边？
- [ ] 最新值 / 异常值标签是否不遮挡线条和坐标轴？
- [ ] 如果使用 markPoint / point mark，是否说明启用条件？

## 8. Tooltip 与交互

- [ ] Tooltip 是否包含时间 / 类目、系列名、数值和单位？
- [ ] 默认 hover 是否只显示垂直指示线；是否避免横向 crosshair 和默认字段名？
- [ ] hover 指示线是否只在 plot / chartHost 内显示，没有溢出标题区或容器？
- [ ] hover 点是否与折线真实位置重合，并来自引擎坐标转换、实际渲染 geometry，或与 series 同层同 scale 绘制？
- [ ] Tooltip 数值格式是否与轴标签和图例一致？
- [ ] Tooltip 顺序是否与图例顺序一致？
- [ ] Tooltip 是否使用 `label_column + value_column` 两列结构，系列行高 `18px`、行间距 `0px`，note 间距 `4px`？
- [ ] Tooltip title 和 value 是否加粗，且 value 列未被裁切？
- [ ] Tooltip 是否不会遮挡当前 hover 点和主要读数区域？
- [ ] hover、图例筛选、范围切换、缩放等交互是否不会改变数据含义？
- [ ] 空状态、加载态、异常态是否有处理方案？

## 9. 多引擎一致性

当同时支持 G2 和 ECharts 时，必须检查：

- [ ] 两个引擎的图表类型一致。
- [ ] 两个引擎的字段映射一致。
- [ ] 两个引擎的图例 shape、颜色和顺序一致。
- [ ] 两个引擎的坐标轴标签抽样一致。
- [ ] 两个引擎的 hover 指示线一致。
- [ ] 两个引擎的默认数据点显示策略一致。
- [ ] 两个引擎的 Tooltip 内容和单位一致。

## 10. 浏览器验证

- [ ] 页面首次打开时图表非空白。
- [ ] 桌面宽度下无文字重叠。
- [ ] 移动宽度下无文字溢出。
- [ ] Hover / touch 交互可触发。
- [ ] 切换筛选和时间范围后布局不跳动。
- [ ] 截图检查通过后才可标记完成。

## 11. 禁止通过项

以下任一问题存在时，不得声明 VibeCode 验证通过：

- 轴标题或轴标签与数据重叠。
- 图例 shape 与图表类型不匹配。
- 折线图默认显示所有点位 mark。
- Hover 指示线样式与规范不一致。
- 多引擎切换后视觉语义明显不一致。
- 只做语法检查，没有做浏览器视觉检查。
