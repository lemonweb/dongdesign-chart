# 设计自查清单 Design Checklist

## 1. 文档定位

本文用于检查 AI 生成或人工设计的图表设计稿是否符合 dongDesign Chart Wiki 的选型、视觉语言、布局、交互和增强规范。

本文只做交付前检查，不替代具体规范。发现问题时，应回到对应文档修正。

## 2. 业务问题与选型

- [ ] 是否明确图表要回答的业务问题？
- [ ] 是否明确分析目标：比较、趋势、占比、分布、关系、排名、流程或多维评估？
- [ ] 是否根据数据结构选择图表，而不是只根据图表名称或外观？
- [ ] 是否说明 `primary_intent` 和必要的 `secondary_intents`？
- [ ] 图表复杂度是否与问题复杂度匹配？
- [ ] L3 / L4 图表是否说明必要性和更简单替代方案？
- [ ] 用户指定图表时，是否检查其与数据结构是否匹配？

## 3. 数据与口径

- [ ] 标题、副标题或说明是否包含必要时间范围？
- [ ] 指标单位是否清晰，例如元、%、人、件、次、天？
- [ ] 筛选条件、样本范围或统计口径是否可见？
- [ ] 不同单位指标是否避免强行共用同一轴？
- [ ] 异常、风险、达标、恶化等判断是否有目标、阈值、历史基准或业务规则？
- [ ] 缺少数据证据时，是否标记 `inferred` 或 `needs_spec`？

## 4. 布局结构

- [ ] 是否建立清晰的标题区、图例区、绘图区、坐标轴区和页脚区？
- [ ] `plotAreaFrame` 是否作为唯一数据坐标空间？
- [ ] 标题、图例、坐标轴、Tooltip 和标注是否没有互相遮挡？
- [ ] 绘图区是否没有被图例、标签或说明文字压缩到不可读？
- [ ] X 轴标签过密时是否抽样、旋转、筛选或换图？
- [ ] Y 轴标签轨道是否给最长标签预留空间？
- [ ] 双轴图是否清楚表达系列与轴的对应关系？

## 5. 色彩与主题

- [ ] 色彩是否来自 `theme-token.md` 和 `color.md`？
- [ ] 分类数据是否使用分类色板，而非随机配色？
- [ ] 连续强弱是否使用顺序色板？
- [ ] 正负偏离是否有明确中心值并使用发散色板？
- [ ] 风险、异常、成功、警告是否使用语义色，并结合文字或符号说明？
- [ ] 同一业务对象在同一页面中颜色是否一致？
- [ ] 是否没有用品牌色或高饱和色表达普通辅助元素？

## 6. 字体与文本

- [ ] 坐标轴、图例、标签和 Tooltip 是否使用规范字号、行高和字重？
- [ ] 是否没有使用小于 `12px` 的图表文字？
- [ ] 文案是否使用业务名称，而不是数据库字段名？
- [ ] 数值格式是否统一，例如小数位、千分位、百分比和单位？
- [ ] 标题是否表达业务问题，不只是图表类型名称？
- [ ] 说明文字是否克制，不遮挡或挤压图表主体？

## 7. 坐标轴与图例

- [ ] 坐标轴类型是否与字段类型一致？
- [ ] 数值轴起点、刻度数量和上界是否符合 `axis.md`？
- [ ] 网格线是否为辅助结构，不抢占主视觉？
- [ ] 图例是否真的需要显示？
- [ ] 图例顺序是否与系列顺序一致？
- [ ] 图例 shape 是否与图表类型一致？
- [ ] 图例筛选或高亮状态是否可识别？

## 8. 标签、Tooltip 与标注

- [ ] 是否避免给所有数据点显示标签？
- [ ] 关键标签是否只用于最新值、最大值、最小值、异常点、目标差等重点？
- [ ] 标签是否不遮挡线条、柱体、点、轴和图例？
- [ ] Tooltip 是否包含维度、系列名、数值、单位和必要口径？
- [ ] Tooltip 是否不会遮挡当前 hover 点和主要读数？
- [ ] 参考线、区间和 callout 是否有明确业务含义？
- [ ] 标注数量是否控制在最重要的 `1-3` 个重点内？

## 9. 交互与状态

- [ ] hover、选中、筛选、联动是否有可见反馈？
- [ ] 柱状图 / 柱线混合图 hover 是否使用完整类目占位高亮矩形？
- [ ] 折线图 hover 指示线是否为 `1px` 实线，并限制在绘图区内？
- [ ] 下钻是否保留返回路径？
- [ ] 全局筛选和局部筛选作用范围是否清楚？
- [ ] 空状态、加载态、异常态是否有设计方案？

## 10. VibeDesign 可维护性

- [ ] 是否使用 `chartFrame / headerFrame / bodyFrame / plotAreaFrame` 等结构？
- [ ] 标题、图例、坐标轴、Tooltip 是否为独立 Frame 或可识别分组？
- [ ] 数据系列是否归属于 `seriesLayer`？
- [ ] 标注是否归属于 `annotationLayer`？
- [ ] 图层命名是否能表达结构、角色和数据语义？
- [ ] 修改标题、图例、轴标签或系列时，设计师是否能定位并维护？

## 11. 禁止通过项

以下任一问题存在时，不得声明设计验证通过：

- 图表选型与业务问题明显不匹配。
- 缺少必要字段仍输出确定图表。
- 异常、风险、达标或原因判断没有证据。
- 使用随机颜色、随机字号或图表库默认样式。
- 标题、图例、轴标签、Tooltip 或标注发生遮挡。
- VibeDesign 图层不可编辑或无结构。
- 复杂图表没有解释必要性和误读风险。
