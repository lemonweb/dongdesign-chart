# VibeDesign 验证示例 VibeDesign Validation Examples

## 1. 文档定位

本文提供一组用于验证 VibeDesign Skill 能力的提示词示例。

这些示例模拟真实用户在使用 Skill 时的表达方式：用户通常只描述业务目标、分析问题、展示场景或上传数据文件，不一定明确指定图表类型。因此验证重点不是“按指定图表生成”，而是检查 Skill 是否能：

- 根据诉求选择合理图表类型。
- 根据字段语义建立正确的数据编码。
- 输出符合 VibeDesign 要求的可编辑设计结构。
- 生成与 Wiki 视觉语言、图表规范和组件库细节一致的设计方案。

本文中的提示词可直接交给 AI / Skill / Relay / Figma Agent 使用。Mock 数据不是必需项；真实验证时可以让用户上传 CSV / Excel，并在提示词中说明数据文件用途。

## 2. 使用方式

建议验证流程：

1. 选择一个验证方案。
2. 将「用户提示词」原文交给 Skill。
3. 如有真实数据，附加上传 CSV / Excel；如无数据，可要求 AI 自行构造最小示例数据。
4. 检查 AI 是否输出了「预期图表类型」或同等合理的替代方案。
5. 检查是否输出 VibeDesign 可维护结构、Visual Fidelity Packet、构建指令和验证清单。
6. 若图表类型、编码、结构或视觉细节不符合预期，回到对应 Skill 或 Wiki 规范修正。

## 4. 验证方案

### VD-P01 趋势变化分析

用户提示词：

```txt
我想看最近一段时间核心流量指标的变化趋势，帮我生成一个适合放在分析看板里的 VibeDesign 图表方案。

如果我上传了 CSV，请你根据日期字段和流量相关指标自动判断应该展示哪些系列。
如果没有上传数据，请你先构造一组最小示例数据用于说明方案。

要求：
1. 不要只给静态截图，要输出可编辑的 VibeDesign 结构。
2. 需要说明你为什么选择这个图表类型。
3. 输出字段映射、图层结构、Visual Fidelity Packet 和验证清单。
4. 保留一个 hover 状态，用于检查 Tooltip 和指示线质量。
```

预期图表类型：

- 首选：折线图。
- 可接受：当用户强调累计量或趋势面积感时，可选面积图。
- 不推荐：柱状图、饼图、雷达图。

验证重点：

- 是否能识别时间字段作为 X 轴。
- 是否能把多个流量指标映射为多系列趋势。
- 折线图图例是否使用 `10×2px` line shape。
- 普通态是否隐藏全部数据点，只在 hover / 最新值 / 异常点显示 markPoint。
- hover 指示线是否为单根垂直 `1px` 实线。
- Tooltip 是否为可编辑两列结构，value 是否右对齐。
- X 轴首尾是否有维度留白，Y 轴是否默认从 `0` 起步。

失败信号：

- 用户没有指定图表类型时，AI 直接问“你想用什么图表”而不先给建议。
- 用饼图表达时间趋势。
- 使用单 SVG / 位图承载整张图，却声称是高质量 VibeDesign。

### VD-P02 类目对比分析

用户提示词：

```txt
我想比较不同商品类目的销售表现，请帮我生成一个 VibeDesign 图表方案。

我可能会上传一个 CSV，里面包含类目名称、销售额、订单量等字段。请你自动判断最适合的图表类型，并说明排序方式。

要求：
1. 适合放在标准分析卡片里。
2. 类目名称可能比较长，需要考虑标签可读性。
3. 需要包含 hover 状态和 Tooltip 结构。
4. 输出可编辑图层结构、变量绑定和验证清单。
```

预期图表类型：

- 首选：柱状图。
- 可接受：当类目名称较长或类目数量较多时，选条形图。
- 不推荐：折线图、饼图、热力图。

验证重点：

- 是否根据类目数量和名称长度选择柱状图或条形图。
- 是否说明排序策略，例如按销售额降序。
- 柱体是否归属到 `seriesLayer`，而不是散落在根画布。
- 类目轴是否保留首尾 half-band 留白。
- hover 是否使用完整类目占位宽度的浅色高亮矩形。
- 是否避免把折线图的垂直指示线套用到柱状图。

失败信号：

- 类目标签重叠却未选择条形图或标签抽样策略。
- 柱体默认加入无意义透明度。
- hover 同时出现高亮矩形和垂直 line 指示器。

### VD-P03 占比构成分析

用户提示词：

```txt
我想看本月销售额由哪些渠道贡献，各渠道占比是多少。请帮我生成一个适合管理层快速阅读的 VibeDesign 可视化方案。

如果上传 CSV，请自动识别渠道字段和金额字段。
如果渠道很多，请处理低占比项，不要让图表变得难读。

要求：
1. 需要说明图表选型依据。
2. 需要输出标签、图例和 Tooltip 的设计规则。
3. 输出 VibeDesign 可编辑结构和验证清单。
```

预期图表类型：

- 首选：环图 / 饼图。
- 可接受：当分类超过 6 个或差异较小时，使用条形占比图。
- 不推荐：折线图、散点图。

验证重点：

- 是否识别这是“整体-部分”的占比问题。
- 是否处理过多分类，例如合并低占比为“其他”。
- 图例 shape 是否使用 `rectangle 10×10 r2`。
- 标签是否避免拥挤，低占比项是否不强行外显。
- Tooltip 是否展示分类名、数值和占比。
- 是否避免用过多颜色导致视觉噪音。

失败信号：

- 分类很多时仍生成拥挤饼图。
- 百分比总和不接近 100% 且无说明。
- 图例和扇区颜色无法对应。

### VD-P04 金额与转化率联合分析

用户提示词：

```txt
我想同时看最近 12 个月的成交金额和成交转化率，判断销售增长是否真的来自转化效率提升。

如果我上传 CSV，请自动识别月份、金额指标和转化率指标。
请你选择最适合的图表类型，并生成 VibeDesign 方案。

要求：
1. 需要说明是否使用双轴，以及使用双轴的风险。
2. 图例必须能表达柱体和折线的差异。
3. Tooltip 需要同时展示金额和百分比。
4. 输出 Visual Fidelity Packet、图层结构和验证清单。
```

预期图表类型：

- 首选：柱线混合图。
- 可接受：双轴图，但必须明确左右轴语义。
- 不推荐：两个彼此孤立的图表，除非用户要求拆分对比。

验证重点：

- 是否把金额映射为柱体，把转化率映射为折线。
- 图例 shape 是否混合使用 `rectangle` 和 `line`。
- 是否避免把“左轴 / 右轴”写成图例项。
- 是否建立 `rightYAxisLabelRail`，右轴标签不进入绘图区。
- Tooltip 是否正确格式化金额和百分比。
- 折线 series 是否为完整 path / vector。

失败信号：

- 把百分比显示为 `0.048` 而不是 `4.8%`。
- 左右轴单位混淆。
- 两个系列图例都使用圆点。

### VD-P05 商品异常诊断

用户提示词：

```txt
我想找出高流量但低转化的商品，帮我生成一个适合商品运营诊断的 VibeDesign 图表方案。

如果上传 CSV，通常会包含商品名、流量指数、转化率、类目、销售额等字段。
请自动判断图表类型，并标出最值得关注的异常商品。

要求：
1. 需要说明字段如何映射到位置、颜色、大小或标注。
2. 需要有异常点标注。
3. 图表应适合设计师继续编辑。
4. 输出 annotationLayer、seriesLayer、Tooltip 结构和验证清单。
```

预期图表类型：

- 首选：散点图。
- 可接受：如果有销售额或商品规模字段，可选气泡图。
- 不推荐：饼图、单纯柱状图。

验证重点：

- 是否把流量和转化率映射为 X / Y 轴。
- 是否用颜色表示类目，或用气泡大小表示销售额。
- 散点图图例 shape 是否为 `dot 10×10 circle`。
- 异常点是否归属于 `annotationLayer`，并包含 label + leader line。
- 是否能说明异常判定逻辑，例如高于流量中位数且低于转化率中位数。
- Tooltip 是否包含商品名、类目和关键指标。

失败信号：

- 用柱状图只能比较销售额，无法回答“高流量低转化”。
- 标注覆盖点位且无法移动。
- 参考线、标签和数据点混在同一层。

### VD-P06 时段热度分析

用户提示词：

```txt
我想看一周内不同日期和时段的访问热度，找出用户最活跃的时间窗口。

如果上传 CSV，请自动识别星期、小时或时段字段，以及 UV / PV 指标。
请生成一个 VibeDesign 图表方案。

要求：
1. 图表需要清楚表达行、列和热度值。
2. 需要有连续色阶图例。
3. 保留一个高峰时段 Tooltip。
4. 输出矩阵结构、色彩映射和验证清单。
```

预期图表类型：

- 首选：热力图。
- 可接受：如果只有单日小时趋势，可选折线图。
- 不推荐：饼图、普通柱状图。

验证重点：

- 是否识别出“二维分类 + 数值强度”的矩阵问题。
- 是否建立 `matrixLayer`，行列标签是否独立成 band。
- 是否使用连续色阶图例，而不是离散分类图例。
- 色阶是否有最小值 / 最大值语义。
- 高峰 Tooltip 是否展示日期、时段和数值。
- 单元格间距是否一致，矩阵是否整体可移动。

失败信号：

- 用多系列柱状图导致时段比较困难。
- 行列标签与格子错位。
- 用无语义的多彩分类色表示连续热度。

### VD-P07 流程转化分析

用户提示词：

```txt
我想复盘一次营销活动，从曝光、点击、加购、提交订单到支付成功，看每一步流失在哪里。

如果上传 CSV，请自动识别阶段字段、人数指标和转化率字段。
请生成一个 VibeDesign 可编辑图表方案，并突出最明显的流失环节。

要求：
1. 需要说明为什么选择这个图表类型。
2. 阶段顺序不能被排序打乱。
3. 需要展示人数和相邻阶段转化率。
4. 输出流程标注、图层结构和验证清单。
```

预期图表类型：

- 首选：漏斗图。
- 可接受：如果阶段较多且需要精确比较，可用横向条形 + 转化标注。
- 不推荐：饼图、折线图。

验证重点：

- 是否识别流程转化语义，而不是简单类目比较。
- 阶段是否按业务顺序排列，不按人数重新排序。
- 漏斗宽度是否按人数递减。
- 转化率是否表达为相邻阶段转化率，而不是全部相对首阶段。
- 流失标注是否归属于 annotation / flow 层。
- 阶段 group 是否可整体移动和编辑。

失败信号：

- 阶段顺序被错误排序。
- 等宽矩形伪装漏斗。
- 转化率口径混乱。

### VD-P08 经营概览指标区

用户提示词：

```txt
我想做一个经营驾驶舱顶部概览区域，展示 GMV、支付订单量、转化率、客单价四个核心指标，并附带最近 7 天趋势。

如果上传 CSV，请自动识别这些指标及日期字段。
请生成一个 VibeDesign 方案，适合放在看板首屏。

要求：
1. 每个指标需要有当前值、环比变化和迷你趋势。
2. 不要把迷你趋势做成完整大图表。
3. 需要说明组件结构和复用方式。
4. 输出可编辑图层结构、变量绑定和验证清单。
```

预期图表类型：

- 首选：指标卡 + 迷你折线图。
- 可接受：指标卡 + 迷你柱状图。
- 不推荐：四张完整标准图表堆叠。

验证重点：

- 是否根据展示空间选择小型图表，而非完整坐标轴图表。
- 每张卡片是否包含 `title / value / delta / sparkline`。
- 迷你趋势是否弱化坐标轴、图例和 Tooltip。
- 指标值是否拥有最高视觉层级。
- 多张卡片尺寸、间距和排版是否一致。
- 上涨 / 下降状态色是否使用规范状态语义。

失败信号：

- 在小卡片里强行塞入完整 X / Y 轴。
- 趋势线遮挡指标值。
- 四张卡片结构不一致，难以复用。

### VD-P09 营销活动复盘看板

用户提示词：

```txt
我需要做一个营销活动复盘看板，面向运营负责人。

我想同时回答这些问题：
1. 活动整体 GMV 和订单量表现如何？
2. 从曝光到支付的转化漏斗是否健康？
3. 哪些渠道贡献最高？
4. 活动期间每天的成交趋势如何？

如果我上传 CSV，请你自动识别可用字段，并规划一个 VibeDesign 看板方案。
请不要只生成一个图表，而是根据问题拆成合理的图表组合。

要求：
1. 输出页面布局结构。
2. 为每个模块说明预期图表类型。
3. 输出 VibeDesign 图层组织方式和验证清单。
4. 说明哪些字段缺失会影响方案质量。
```

预期图表类型：

- 顶部：指标卡。
- 转化：漏斗图。
- 渠道贡献：环图 / 条形图。
- 趋势：折线图 / 柱线混合图。

验证重点：

- 是否能从多问题诉求拆解出多图组合。
- 是否能为不同问题选择不同图表类型。
- 是否建立页面级结构，而不是堆散图。
- 是否说明字段缺失和替代方案。
- 是否避免所有模块使用同一种图表。
- 是否在看板层面控制信息层级。

失败信号：

- 只生成一张万能图。
- 图表组合与问题不对应。
- 页面布局像图表素材堆叠，没有模块层级。

### VD-P10 上传 CSV 的自动选型验证

用户提示词：

```txt
我上传了一个 CSV，希望你帮我判断这份数据适合做哪些可视化，并生成一个 VibeDesign 方案。

请你先阅读字段名和样例数据，然后完成：
1. 判断每个字段的语义和数据类型。
2. 提出 2-3 个最适合的分析问题。
3. 为每个问题选择合适图表类型。
4. 选择其中一个最有价值的问题，生成完整 VibeDesign 图表方案。
5. 输出字段映射、图层结构、Visual Fidelity Packet 和验证清单。

如果字段不足以支持某个图表，请明确说明缺失字段，不要强行生成。
```

预期图表类型：

- 根据 CSV 字段自动决定。
- 常见输出：
  - 日期 + 指标：折线图。
  - 类目 + 指标：柱状图 / 条形图。
  - 分类 + 总量：环图 / 条形占比图。
  - 两个连续指标 + 分类：散点图 / 气泡图。
  - 阶段 + 人数：漏斗图。

验证重点：

- 是否先做字段语义判断，而不是直接生成图。
- 是否能提出合理分析问题。
- 是否说明图表类型选择依据。
- 是否能拒绝不成立的图表方案。
- 是否输出完整 VibeDesign 构建信息。
- 是否把 CSV 字段映射到正确视觉编码。

失败信号：

- 不看字段语义，随机选择图表。
- 所有 CSV 都默认生成柱状图。
- 缺少时间字段却强行生成时间趋势。
- 缺少阶段字段却强行生成漏斗图。

## 5. 通用验证标准

无论选择哪个方案，以下能力都应被检查：

- 是否主动完成图表选型，而不是把选择问题抛回给用户。
- 是否说明选型依据和不选其他图表的原因。
- 是否根据字段语义完成 x / y / color / size / tooltip 映射。
- 是否输出 VibeDesign 可维护结构，而不是单 SVG、截图或不可编辑位图。
- 是否包含 `chartFrame / headerFrame / bodyFrame / plotAreaFrame / seriesLayer / interactionLayer / tooltipFrame` 或与图表类型等价的结构。
- 是否遵守图例 shape 语义：折线用 line，柱 / 面 / 饼 / 热力用 rectangle，点图用 dot。
- 是否按图表类型使用正确 hover 反馈。
- 是否输出 Visual Fidelity Packet。
- 是否包含可执行的 Figma / Relay / Zero 构建指令。
- 是否列出风险、缺失字段和验证清单。

## 6. 推荐验证顺序

快速验证建议优先使用：

1. `VD-P10`：验证 CSV 场景下的字段推断与自动选型。
