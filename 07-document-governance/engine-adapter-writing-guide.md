# 引擎适配写法原则 Engine Adapter Writing Guide

## 1. 文档定位

本文定义 `04-adaptation/` 中主题模型、G2 适配、ECharts 适配和映射表的写法。

引擎适配文档负责把 dongDesign Chart 的设计语义转换为可实现、可检查的图表引擎配置关系。

## 2. 写作目标

适配文档应回答：

> 同一个图表设计语义在 G2 / ECharts 等引擎中分别如何配置，如何保证视觉结果和交互语义一致？

它不负责重新定义图表为什么这样设计。设计依据应来自 `01-design-language/` 和 `02-chart-type/`。

## 3. 标准章节

建议结构：

```txt
1. 文档定位
2. 输入与输出
3. 主题模型
4. 字段映射
5. 视觉元素映射
6. 交互映射
7. 多引擎一致性检查
8. 禁止项
9. AI 自查清单
```

## 4. 映射表写法

适配文档应优先使用映射表：

| 语义 | G2 | ECharts | 规范来源 |
|---|---|---|---|
| 横轴字段 | `encode('x', field)` | `xAxis` + `encode.x` | `axis.md` |
| 分类色 | scale / color | `color` / `visualMap` | `color.md` |

映射表必须说明“语义”，不得只列代码字段。

## 5. 代码示例写法

代码示例应保持最小可读，不写成完整业务项目。

要求：

- 示例字段使用通用命名，如 `date`、`category`、`value`、`series`。
- 样式必须通过 theme 或 token 映射，不随机写死。
- 示例中出现 fallback 时必须标记 `fallback_reference`。
- 不同引擎示例应保持同一数据语义。

## 6. 一致性规则

同时支持 G2 / ECharts 时，必须检查：

- 图表类型一致。
- 字段角色一致。
- 颜色、图例 shape、Tooltip 内容一致。
- 坐标轴刻度和标签抽样策略一致。
- hover、筛选、缩放和选中反馈语义一致。
- 空状态、加载态、异常态策略一致。

## 7. 禁止项

- 禁止直接使用图表库默认主题覆盖 dongDesign 规范。
- 禁止只给代码，不说明设计语义。
- 禁止 G2 / ECharts 使用不同字段口径。
- 禁止在适配文档中重新定义色彩、字体和图表选型规则。
- 禁止把某个引擎的能力限制写成设计规范。

## 8. 自查清单

- 是否说明输入设计语义和输出配置关系？
- 是否引用了视觉语言和图表类型规范？
- 是否包含 G2 / ECharts 的等价映射？
- 是否说明多引擎一致性检查项？
- 是否避免图表库默认样式泄漏？
- 是否明确 fallback 来源？
