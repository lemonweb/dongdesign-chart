# README 写法原则 README Writing Guide

## 1. 文档定位

本文定义各目录 `README.md` 的写法。

README 是目录入口，负责说明目录用途、使用时机、读取顺序和与其他目录的关系，不承载完整规范正文。

## 2. 写作目标

README 应回答：

> 这个目录负责什么？什么时候读？先读哪个文件？不要在这里找什么？

## 3. 标准章节

建议结构：

```txt
1. 目录定位
2. 使用时机
3. 读取顺序
4. 使用原则
5. 不在本目录处理的内容
```

如目录包含复杂子模块，可增加“目录结构”或“文件说明”。

## 4. 目录定位写法

定位应简短明确：

```txt
本目录用于管理单个图表组件的设计指南。
```

避免：

```txt
本目录内容丰富，包含很多有用的信息。
```

## 5. 读取顺序写法

读取顺序应使用稳定相对路径：

```markdown
1. `capability-matrix.md`
2. `selection-rules.md`
3. `complexity-level.md`
4. 具体图表组件文档
```

路径必须与真实文件名一致。

## 6. 文件说明写法

当目录文件较多时，可使用表格：

| 文件 | 用途 |
|---|---|
| `color.md` | 色彩决策规则 |
| `theme-token.md` | 主题变量和 resolved value |

README 不应复制目标文件的大段正文，只写入口说明。

## 7. 禁止项

- 禁止在 README 中维护具体样式值或复杂业务规则。
- 禁止 README 路径与真实文件名不一致。
- 禁止新增文件后不更新 README。
- 禁止让 README 替代 `ai-file-index.md`。
- 禁止写成宣传文案。

## 8. 自查清单

- 是否说明目录定位？
- 是否说明使用时机？
- 是否列出读取顺序？
- 是否说明不在本目录处理的内容？
- 路径是否真实存在且大小写一致？
- 是否避免复制完整规范正文？
