# AI 读取顺序

## 1. 读取目标

本文件用于指导 AI 在处理本 Wiki 相关任务时按稳定顺序读取上下文，避免跳过全局规则、误用图表类型或生成不符合设计体系的内容。

## 2. 默认读取顺序

1. `README.md`
2. `00.总览 Overview/01.使用说明 How-to-use.md`
3. `00.总览 Overview/05.Codex记忆迁移 Codex-memory-migration.md`
4. `00.总览 Overview/03.图表选型决策树 Decision-tree.md`
5. `00.总览 Overview/04.术语表 Glossary.md`
6. `01.视觉语言 Design Language/README.md`
7. `05.执行规则 Rules/README.md`

## 3. 按任务补充读取

### 3.1 图表选型任务

继续读取：

- `02.图表模式 Chart Type/01.能力矩阵 Capability-matrix.md`
- `02.图表模式 Chart Type/02.选型规则 Selection-rules.md`
- 对应具体图表文档

### 3.2 业务分析页面任务

继续读取：

- `03.业务场景 Pattern/README.md`
- 对应业务场景文档
- 相关图表类型文档

### 3.3 代码生成任务

继续读取：

- `04.引擎适配 Adaptation/01.通用主题模型 Theme-schema.md`
- `04.引擎适配 Adaptation/02.G2适配 G2-adapter.md` 或 `04.引擎适配 Adaptation/03.ECharts适配 ECharts-adapter.md`
- `06.示例自查 Self Check/02.代码自查清单 Code-checklist.md`

### 3.4 文档治理任务

继续读取：

- `07.文档治理 Document Governance/01.通用写作原则 General-writing-principles.md`
- 与目标目录对应的写法原则
- `07.文档治理 Document Governance/10.文档元信息与命名规范 Metadata-naming-guide.md`

## 4. 执行原则

- 先理解业务问题，再选择图表。
- 先读取视觉语言，再生成样式。
- 先判断图表选型，再读取具体图表组件文档。
- 代码生成必须读取引擎适配规范。
- 涉及持续维护、偏好、自动化工作流时，必须读取 Codex 记忆迁移文件。
- 不得随机使用颜色、样式、动效和图表类型。
