# 柱状图 Bar Chart

## 1. 文档定位

本文定义柱状图在数据可视化 Wiki 中的选型、数据结构、视觉构成、设计原则、交互规则和代码映射。

柱状图主要用于回答：

> 不同分类之间的指标大小有什么差异？哪个分类更高或更低？多组分类之间是否存在明显对比？

本文不重新定义坐标轴、图例、标签、Tooltip、颜色和字体规范。相关视觉规则应读取 `01.视觉语言 Design Language/`。

## 2. 图表定义

柱状图是由笛卡尔坐标系、分类轴、数值轴和矩形图元组成的统计图表。它通过矩形的高度表达连续数值的大小，用横向分类位置表达不同类别。

在柱状图中：

- 横轴表示分类维度或有序分类。
- 纵轴表示连续数值指标。
- 矩形表示每个分类对应的数值大小。
- 矩形高度越高，表示对应指标值越大。
- 分类或系列可映射到颜色，用于增强类别区分。

柱状图强调分类之间的数值比较，不强调连续趋势。

## 3. 属性摘要

| 属性 | 规则 |
|---|---|
| 图表类型 | 统计图表 / 基础图表 |
| 图形形态 | 矩形柱体 |
| 核心功能 | 分类对比、排序对比、分组对比、构成对比 |
| 适合的数据 | 两个连续字段，或一个有序分类字段 + 一个连续数值字段 |
| 数据与图形映射 | 分类字段映射到横轴位置，连续数值映射到矩形高度，分类或系列字段可映射到颜色 |
| 数据体量 | 分类维度数量应大于 2；同一图中指标或系列不建议超过 5 个 |

## 4. 适用场景

柱状图适用于以下场景：

| 场景 | 使用条件 | 价值 |
|---|---|---|
| 分类数值比较 | 一个分类维度 + 一个连续指标 | 快速判断不同分类的高低差异 |
| 排名分析 | 分类之间存在可排序的指标 | 识别 Top / Bottom、头部集中度和长尾 |
| 多组数据对比 | 一个主分类 + 一个系列分类 + 一个指标 | 比较不同组别在同一分类下的表现 |
| 组成对比 | 每个分类由多个子项构成 | 同时观察整体大小和内部构成 |
| 目标与实际对比 | 同一分类存在目标值和实际值 | 展示完成度、差距和达成情况 |

典型业务问题：

- 不同品牌的成交金额谁更高？
- 不同部门的绩效表现如何？
- 不同方案的转化效果是否有差异？
- 每个城市的销售额由哪些品类构成？
- 实际完成值与目标值之间的差距是多少？

## 5. 不适用场景

| 不适用场景 | 原因 | 替代方向 |
|---|---|---|
| 分类太多或字段太长 | 横轴标签需要旋转或截断，可读性下降 | 条形图、滚动、筛选、Top N |
| 连续趋势分析 | 柱状图强调分类比较，不适合表达连续变化路径 | 折线图、面积图 |
| 精确明细查询 | 柱体只能表达相对大小，不适合承载大量明细字段 | 表格、明细列表 |
| 占比结构为唯一重点 | 普通柱状图不直接表达整体占比关系 | 饼图、环图、百分比堆叠柱状图 |
| 系列过多 | 分组柱过密、颜色过多，会降低比较效率 | 筛选、分面、小多图 |

## 6. 数据结构

### 6.1 基础字段

| 字段角色 | 是否必需 | 说明 |
|---|---|---|
| 分类字段 | 必需 | 映射到横轴，用于定义每个柱体的位置 |
| 指标字段 | 必需 | 映射到纵轴，用于定义柱体高度 |
| 系列字段 | 可选 | 用于生成分组柱、堆叠柱或颜色分类 |
| 排序字段 | 可选 | 用于按指标值、业务顺序或自定义顺序排列分类 |
| 对比字段 | 可选 | 用于目标 / 实际、同比 / 环比等嵌套对比 |

### 6.2 数据要求

- 分类数量应大于 2，否则优先考虑指标卡、表格或简单文本说明。
- 单图中的指标或系列不建议超过 5 个。
- 分类字段必须具有明确含义，不应将无意义 ID 直接作为轴标签。
- 指标字段必须为连续数值，且应保持同一单位和统计口径。
- 分组柱要求每个主分类下的系列口径一致。
- 堆叠柱要求各子项可以相加，且总量具有业务意义。
- 嵌套柱要求内外层指标存在可比较关系，例如目标值与实际完成值。

### 6.3 数据示例

```json
[
  { "brand": "Apple", "metric": "成交金额", "value": 22 },
  { "brand": "华为", "metric": "成交金额", "value": 28 },
  { "brand": "小米", "metric": "成交金额", "value": 46 },
  { "brand": "OPPO", "metric": "成交金额", "value": 64 },
  { "brand": "VIVO", "metric": "成交金额", "value": 100 }
]
```

字段映射：

| 语义 | 字段 |
|---|---|
| 横轴分类 | `brand` |
| 纵轴数值 | `value` |
| 系列 / 图例 | `metric` |

## 7. 视觉构成

| 构成元素 | 作用 | 是否必需 |
|---|---|---|
| 横轴 xAxis | 表示分类维度 | 必需 |
| 纵轴 yAxis | 表示连续数值指标 | 必需 |
| 柱体 seriesBar | 表示每个分类对应的数值大小 | 必需 |
| 网格线 grid | 辅助读取数值位置 | 建议启用 |
| 图例 legend | 区分多系列或说明单系列指标 | 多系列时必需，单系列可选 |
| 标签 label | 标记重点柱体的数值 | 可选 |
| Tooltip | 展示悬停柱体的精确值和上下文 | 建议启用 |
| 排序菜单 | 支持按指标值或业务维度切换排序 | 按需启用 |
| 滚动条 / dataZoom | 处理分类数量较多或屏幕空间有限的场景 | 按需启用 |

柱状图的基础结构是“坐标轴 + 网格 + 柱体”。不应为了装饰额外添加卡片、背景图形或复杂容器。

## 8. 图表类型

### 8.1 基础柱状图

基础柱状图一般只展示一个维度和一个指标，适合简单的分类对比分析。

适用条件：

- 每个分类只有一个指标值。
- 用户主要关注分类之间的高低差异。
- 不需要展示组成结构或多系列关系。

### 8.2 分组柱状图

分组柱状图通过将不同系列的数据并列展示，形成“簇状”效果。每组内的柱体表示不同子类别的数据值，方便进行组内和组间对比。

适用条件：

- 存在一个主分类字段和一个系列字段。
- 用户需要比较同一分类下不同系列的表现。
- 系列数量应控制在 2-5 个。

### 8.3 堆叠柱状图

堆叠柱状图通过将不同类别的数据堆叠在同一个柱体上，展示整体与部分之间的关系。

适用条件：

- 子项可以相加并形成有意义的总量。
- 用户既需要观察整体大小，也需要理解内部构成。
- 图例和 Tooltip 必须清晰展示每个堆叠段的含义和值。

### 8.4 嵌套柱状图

嵌套柱状图通过在外层柱体中嵌套内层柱体，展示多层级或目标对比关系。常用于目标与实际完成情况的对比：外层柱体表示目标值，内层柱体表示实际完成值。

适用条件：

- 内外层指标存在明确对比关系。
- 内层值不应超过外层基准，除非业务上允许超额完成。
- 颜色、透明度或明度必须区分内外层。

## 9. 设计原则

### 9.1 清晰的数据呈现

柱体宽度应适中，避免过宽导致笨重，也避免过窄导致难以辨认。

执行规则：

- 柱体最大宽度建议为 `40px`。
- 同系列不同分类之间的最小间距建议为 `16px`。
- 分组柱中不同系列之间的间距固定为 `2px`。
- 柱体应垂直贴合横轴基线，禁止悬浮。
- 所有柱体必须共享同一数值轴基准。

### 9.2 合理的数据表达

柱状图不应默认给每个柱体都添加数值标签。过多标签会使图表杂乱，降低比较效率。

优先标记：

- 最大值。
- 最小值。
- 当前选中值。
- 异常值。
- 与业务目标相关的关键值。

### 9.3 避免过度装饰

柱状图应突出数据本身，不应依赖装饰制造视觉效果。

禁止默认使用：

- 大面积渐变柱体。
- 投影柱体。
- 立体柱体。
- 发光效果。
- 过大的圆角。
- 无业务含义的纹理、描边和背景装饰。

设计指南中的柱体仅保留轻微顶部圆角，默认顶部圆角为 `1px`。

### 9.4 准确使用色彩

单系列柱状图应使用主强调色或业务指定色。多系列柱状图应使用分类色板，并保证系列之间可区分。

执行规则：

- 单系列柱体绑定 `分类色板/item-1`。
- 多系列柱体按顺序绑定 `分类色板/item-*`。
- 图例颜色必须与柱体颜色一致。
- 不使用随机颜色或临时硬编码颜色。
- 同一图中颜色不应同时表达过多语义。

### 9.5 控制分类和系列数量

柱状图适合中低密度分类比较。分类过多时，应通过滚动、筛选、Top N 或横向条形图降低阅读负担。

执行规则：

- 分类数量较少时，直接展示全部柱体。
- 分类较多但仍需要完整查看时，启用滚动条或 dataZoom。
- 分类字段过长时，优先使用条形图，避免横轴标签大角度旋转。
- 系列超过 5 个时，不应继续增加分组柱，应使用筛选或分面。

### 9.6 组件还原规格

当柱状图用于 VibeDesign / Relay / Figma 设计验证时，应优先还原图表组件库的外观细节，而不是只满足“有柱体和坐标轴”。

标准基础柱状图建议规格：

| 元素 | 规则 |
|---|---|
| plot area | 不加独立卡片边框，不使用额外背景色；通过坐标轴和网格定义绘图区 |
| 容器内边距 | 左侧约 `30px`，顶部约 `30px`，底部约 `28px`，为坐标轴和图例预留空间 |
| 柱体 | 竖向矩形，最大宽度 `40px`，最小宽度不低于 `2px`，顶部圆角 `1px` |
| 分类间距 | 同系列分类间距不小于 `16px`；柱体两侧可保留 `2-8px` 的 category gap |
| 分组间距 | 同一分类内不同系列间距 `2px`，不同分类组之间保持更大间隔 |
| 颜色 | 单系列使用 `分类色板/item-1`，默认值为蓝色系；多系列递增使用 `item-*` |
| X 轴 | 底部 `1px` 轴线，分类标签位于轴线下方约 `10px` |
| Y 轴 | 默认不强化左侧实线；显示 `0 / 25 / 50 / 75 / 100` 等等距刻度和水平网格 |
| 轴标签 | 固定 `.textSmall 12/18 400`；数值轴可使用 `.numberSmall 12/18 400` |
| 轴标签色 | 绑定 `坐标轴/axislabelcolor`，默认近似 `#8c8c8c` |
| 图例 | 右上角，色块 `10 × 10px`，圆角 `1px`，图例文字 `.textSmall 12/18` |
| 图例文字色 | 绑定 `图例/legendtextcolor`，默认近似 `#595959` |
| Tooltip | 悬停柱体时展示分类、系列和值；多系列时使用 axis trigger 聚合展示 |
| 滚动条 | 分类过多时位于图表底部，基础滚动条高度可为 `4px` |

如果目标工具无法复用真实图表组件，应至少按上述规格绘制；如果只能用 SVG 合成，应在验证清单中说明“当前是视觉还原稿，不是组件实例”。

## 10. 交互规则

| 交互 | 启用条件 | 系统反馈 |
|---|---|---|
| 悬停 Tooltip | 需要读取柱体精确值 | 显示分类名、系列名、指标值和单位 |
| 柱体高亮 | 用户悬停或选中某个柱体 | 当前柱体增强，其他柱体保持弱化或不变 |
| 图例筛选 | 多系列柱状图 | 支持隐藏、显示或高亮某个系列 |
| 排序菜单 | 用户需要切换比较顺序 | 在指标轴标签区域右键触发下拉菜单，选择不同排序依据 |
| 滚动 | 分类数量超出容器宽度 | 通过滚动条查看超出部分 |
| 缩放 / 平移 | 分类密集或需要聚焦局部范围 | 支持拖拽、滚轮或按钮操作聚焦某一范围 |

交互应帮助用户比较分类差异，不应把柱状图变成复杂操作面板。

### 10.1 Hover 展示顺序

柱状图 hover 状态应按以下层级生成：

```txt
柱体 hover 命中
→ 当前柱体高亮
→ axisPointer 或分类辅助定位
→ Tooltip container
→ Tooltip item icon / label / value
```

Tooltip 内容顺序：

```txt
分类名
系列名
指标值 + 单位
必要时显示占比、排名或同比环比
```

### 10.2 滚动规则

当类别数量过多时，超出容器宽度的部分可以通过滚动条查看。屏幕空间有限时，滚动操作可以有效利用小屏幕展示更多数据。

执行规则：

- 滚动条应放在绘图区下方，不遮挡坐标轴标签。
- 默认视窗内仍应保留可比较的柱体密度。
- 滚动后坐标轴、Tooltip 和图例应保持联动。
- 若分类字段过长，滚动不是唯一解，应优先判断是否改用条形图。

## 12. G2 / ECharts 映射

### 12.1 G2 映射

基础柱状图：

```js
chart
  .interval()
  .data(data)
  .encode('x', 'brand')
  .encode('y', 'value')
  .encode('color', 'metric')
  .style('maxWidth', 40);
```

分组柱状图：

```js
chart
  .interval()
  .data(data)
  .encode('x', 'brand')
  .encode('y', 'value')
  .encode('color', 'series')
  .transform({ type: 'dodgeX' })
  .style('maxWidth', 40);
```

堆叠柱状图：

```js
chart
  .interval()
  .data(data)
  .encode('x', 'brand')
  .encode('y', 'value')
  .encode('color', 'series')
  .transform({ type: 'stackY' });
```

### 12.2 ECharts 映射

基础柱状图：

```js
option = {
  tooltip: { trigger: 'axis', axisPointer: { type: 'shadow' } },
  legend: { show: true },
  grid: { left: 30, right: 0, top: 30, bottom: 28, containLabel: true },
  xAxis: { type: 'category', data: categories },
  yAxis: { type: 'value' },
  series: [
    {
      name: '成交金额',
      type: 'bar',
      data: values,
      barMaxWidth: 40,
      itemStyle: { borderRadius: [1, 1, 0, 0] }
    }
  ]
};
```

分组柱状图：

```js
option = {
  tooltip: { trigger: 'axis', axisPointer: { type: 'shadow' } },
  legend: {},
  xAxis: { type: 'category', data: categories },
  yAxis: { type: 'value' },
  series: seriesList.map(item => ({
    name: item.name,
    type: 'bar',
    data: item.values,
    barMaxWidth: 40,
    barGap: '2px',
    barCategoryGap: '16px',
    itemStyle: { borderRadius: [1, 1, 0, 0] }
  }))
};
```

堆叠柱状图：

```js
option = {
  tooltip: { trigger: 'axis', axisPointer: { type: 'shadow' } },
  legend: {},
  xAxis: { type: 'category', data: categories },
  yAxis: { type: 'value' },
  series: seriesList.map(item => ({
    name: item.name,
    type: 'bar',
    stack: 'total',
    data: item.values,
    barMaxWidth: 40
  }))
};
```

分类数量较多时：

```js
option.dataZoom = [
  { type: 'slider', xAxisIndex: 0, height: 4, bottom: 0 },
  { type: 'inside', xAxisIndex: 0 }
];
```

## 13. AI 输出格式

当 AI 生成柱状图配置时，应输出以下结构：

```json
{
  "chartType": "bar",
  "variant": "basic | grouped | stacked | nested",
  "purpose": "category-comparison",
  "dataMapping": {
    "x": "categoryField",
    "y": "valueField",
    "series": "seriesField optional",
    "color": "seriesField optional"
  },
  "constraints": {
    "minCategoryCount": 3,
    "maxSeriesCount": 5,
    "barMaxWidth": 40,
    "categoryMinGap": 16,
    "seriesGap": 2,
    "barRadius": [1, 1, 0, 0]
  },
  "styleTokens": {
    "barColor": "分类色板/item-*",
    "axisLabelColor": "坐标轴/axislabelcolor",
    "legendTextColor": "图例/legendtextcolor",
    "font": ".textSmall 12/18 400"
  },
  "components": {
    "xAxis": true,
    "yAxis": true,
    "grid": true,
    "legend": "auto",
    "tooltip": true,
    "label": "key-values-only",
    "dataZoom": "when-overflow"
  }
}
```

## 14. 禁止项

AI 生成柱状图时禁止：

- 将无序分类强行解释为趋势。
- 用折线图逻辑描述柱状图。
- 在分类过多时强行旋转大量横轴标签。
- 默认给每个柱体添加数值标签。
- 使用 3D 柱体、投影、发光或大面积渐变。
- 将柱体设置为过大圆角，导致图形变成胶囊形。
- 多系列时不提供图例。
- 堆叠柱中叠加不能相加的指标。
- 分组柱中使用超过 5 个系列。
- 使用硬编码颜色替代主题变量。
- 只写 `type: "bar"`，不声明字段映射、系列关系和样式约束。

## 15. AI 自查清单

生成柱状图前，AI 必须检查：

- [ ] 是否存在分类字段和连续数值字段？
- [ ] 分类数量是否大于 2？
- [ ] 系列数量是否不超过 5？
- [ ] 是否明确选择了 `basic`、`grouped`、`stacked` 或 `nested`？
- [ ] 堆叠柱的子项是否可以相加？
- [ ] 嵌套柱的内外层指标是否存在明确对比关系？
- [ ] 柱体最大宽度是否控制在 `40px`？
- [ ] 分类间距和系列间距是否符合组件规格？
