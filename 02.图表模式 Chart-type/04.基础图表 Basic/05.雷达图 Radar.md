# 05.雷达图 Radar

## 1. 文档定位

雷达图用于在同一图表中比较一个或多个对象在多个维度上的综合表现，帮助用户观察优势、短板、整体轮廓和多系列差异。

本文档用于约束 AI 在生成雷达图时的图表选型、字段映射、维度数量、系列数量、视觉构成、交互行为和工程映射，避免把雷达图误用为精确数值比较、趋势分析或占比分析图表。

## 2. 图表定义

雷达图（Radar Chart），又称蜘蛛图（Spider Chart）或星状图（Star Chart），是一种用于显示多个定量变量的图表。它以从中心向外辐射的多个轴表示不同变量，每条轴代表一个维度，轴上的点表示该维度上的数值大小。

同一数据系列在不同维度上的数值点被依次连接后，会形成一个多边形。这个多边形的形状用于表达该系列在多个维度上的整体轮廓和分布特征，便于比较不同对象在多个指标上的优势与不足。

雷达图强调“多维画像”和“相对表现”，不强调单个数值的精确读取。

## 3. 属性摘要

| 属性      | 说明                                     |
| ------- | -------------------------------------- |
| 图表类型    | 基础图表 / 多维对比图表                          |
| 图形形态    | 线、多边形、闭合数据区域                           |
| 主要功能    | 对比、分布、多维画像                             |
| 适合的数据   | 一个维度集合 + 一个或多个数据系列 + 每个维度上的定量指标        |
| 数据与图形映射 | 维度映射到辐射轴；指标值映射到轴上的距离；同一系列的点连接成雷达链和数据区域 |
| 维度数量    | 建议控制在 5-10 个以内                         |
| 系列数量    | 2-3 个系列效果最佳；最多不超过 5 个                  |

## 4. 适用场景

雷达图适用于以下场景：

- 多维度对比：比较不同对象在多个指标上的相对表现。
- 能力评估：展示个人、团队、店铺或产品在各项能力上的优势与不足。
- 特性对比：对比不同产品在价格、功能、易用性、设计等关键特性上的差异。
- 风险评估：评估多个风险因素的影响程度。
- 用户画像：描绘用户在多个行为、偏好或属性上的特征。

典型业务问题包括：

- 某个店铺在客服咨询、物流履约、售后服务、商品体验等维度上的综合表现如何？
- 多个产品在功能、价格、体验、设计、服务上的优势分别是什么？
- 不同用户群体在内容偏好、购买力、活跃度和忠诚度上是否存在明显差异？

## 5. 不适用场景

以下场景不应优先使用雷达图：

| 不适用场景 | 原因 | 替代方向 |
| --- | --- | --- |
| 只比较单一指标 | 雷达图需要多个维度，单指标会造成图形冗余 | 柱状图、条形图、指标卡 |
| 需要精确比较数值大小 | 雷达图依赖形状判断，不利于精确读数 | 表格、柱状图 |
| 需要展示时间趋势 | 雷达图不表达连续时间变化 | 折线图、面积图 |
| 需要表达部分与整体 | 多边形面积不等于占比结构 | 饼图、环图、堆叠图 |
| 维度数量过少 | 低于 3 个维度无法形成有效多维轮廓 | 柱状图、散点图 |
| 维度数量过多 | 超过 10 个维度会导致标签拥挤、轴线密集、轮廓难以辨认 | 分组展示、表格、小多图 |
| 系列数量过多 | 超过 5 个系列会导致线条和区域重叠，难以区分 | 图例筛选、小多图、分面图 |

## 6. 数据结构

雷达图需要将多个维度上的指标值组织成同一结构。每个系列必须拥有同一组维度，才能进行有效对比。

### 6.1 基础字段

| 字段角色 | 是否必需 | 说明 |
| --- | --- | --- |
| 维度字段 | 必需 | 表示每条辐射轴，如客服咨询、物流履约、售后服务 |
| 指标字段 | 必需 | 表示该维度上的定量值 |
| 系列字段 | 可选 | 用于生成分组雷达图，如不同产品、国家、店铺或用户群体 |
| 标签字段 | 可选 | 用于 Tooltip 或关键点说明 |

### 6.2 基础雷达图

```json
[
  { "dimension": "客服咨询", "value": 3.8 },
  { "dimension": "物流履约", "value": 4.8 },
  { "dimension": "售后服务", "value": 4.3 },
  { "dimension": "商品体验", "value": 4.2 },
  { "dimension": "服务附加", "value": 5.3 },
  { "dimension": "星级表现", "value": 4.6 }
]
```

### 6.3 分组雷达图

```json
[
  { "series": "Metric A", "dimension": "Dimension A", "value": 20 },
  { "series": "Metric A", "dimension": "Dimension B", "value": 24 },
  { "series": "Metric A", "dimension": "Dimension C", "value": 10 },
  { "series": "Metric B", "dimension": "Dimension A", "value": 34 },
  { "series": "Metric B", "dimension": "Dimension B", "value": 54 },
  { "series": "Metric B", "dimension": "Dimension C", "value": 42 }
]
```

### 6.4 数据要求

- 维度数量建议为 5-10 个。
- 每个系列必须包含完整且一致的维度集合。
- 各维度应尽量单位一致，或经过标准化处理。
- 不同量纲的数据必须先归一化、评分化或转换为同一可比尺度。
- 同一张雷达图中系列数量建议为 2-3 个，最多不超过 5 个。

## 7. 视觉构成

| 构成元素 | 作用 | 是否必需 |
| --- | --- | --- |
| 维度轴 Axes / Spokes | 从中心向外辐射，每条轴表示一个维度 | 必需 |
| 刻度线 Scales / Grids | 表示数值大小，通常为等距同心网格 | 必需 |
| 数据点 Data Points | 表示系列在某个维度上的数值位置 | 必需 |
| 雷达链 Data Lines | 连接同一系列的各维度数据点，形成轮廓 | 必需 |
| 数据区域 Data Area | 由雷达链围合形成的闭合区域，用于观察整体轮廓 | 建议启用 |
| 维度标签 Labels | 显示每条轴对应的维度名称 | 必需 |
| 数值标签 Value Labels | 显示具体数值 | 可选 |
| 图例 Legend | 区分多个系列 | 多系列时必需 |
| Tooltip | 展示悬停点的精确值和上下文 | 建议启用 |

雷达图中的网格、轴线和维度标签属于图表读数骨架，不应被装饰性元素替代。

## 8. 设计原则

### 8.1 控制维度数量

维度数量应控制在 5-10 个以内。维度过多会导致雷达图拥挤复杂，标签和轴线难以辨认，也会削弱不同数据系列之间的差异判断。

执行规则：

- 3-4 个维度时，应优先判断是否用柱状图更清晰。
- 5-8 个维度适合常规雷达图。
- 9-10 个维度需要确认标签可读性。
- 超过 10 个维度时，应拆分为多个雷达图、改用表格或使用小多图。

### 8.2 保持单位一致

雷达图的各维度应尽量选择单位性质相似或经过标准化处理的指标。若不同单位直接混用，会让图形轮廓产生误导。

执行规则：

- 同一雷达图中的值应落在统一尺度下。
- 多指标原始单位不一致时，应转为百分位、评分、指数或归一化值。
- 归一化后的尺度需要在标题、说明或 Tooltip 中表达清楚。

### 8.3 控制系列数量

单张雷达图展示的数据系列不宜过多。2-3 个系列效果最佳，最多不超过 5 个。

执行规则：

- 1 个系列用于展示单对象画像。
- 2-3 个系列适合重点对比。
- 4-5 个系列必须启用图例，并保证颜色可区分。
- 超过 5 个系列时，应使用筛选、分面、小多图或默认只展示重点系列。

### 8.4 使用清晰数据线

数据线（dataLine）用于突出数据轮廓，应使用 `2px` 线宽。

执行规则：

- 不使用过粗线条遮盖数据点或网格。
- 不使用过细线条导致轮廓难以追踪。
- 多系列时线条颜色必须绑定分类色变量。

### 8.5 使用透明数据区域

当启用数据区域（dataArea）时，应使用透明叠加，避免遮挡其他系列和网格。

执行规则：

- 多系列重叠的数据区域透明度建议为 `10%`。
- 数据区域只用于辅助理解轮廓，不应成为主要读数方式。
- 不使用高饱和或不透明填充覆盖网格和标签。

### 8.6 克制显示数值标签

雷达图默认不显示所有数值标签。数值标签只在关键点、异常点、悬停或选中时展示。

原因：

- 维度标签已经占用外圈空间。
- 所有点同时显示数值会造成遮挡。
- 雷达图的主要任务是比较轮廓和相对表现，而不是逐项读数。

## 9. 交互规则

| 交互 | 启用条件 | 系统反馈 |
| --- | --- | --- |
| Tooltip | 需要读取某个维度上的精确值 | 悬停时显示系列名、维度名、数值和单位说明 |
| 高亮 | 用户悬停某个系列、数据点或图例项 | 当前系列线条和区域增强，其他系列弱化 |
| 图例过滤 | 多系列雷达图 | 支持隐藏、显示或聚焦某个系列 |
| 数据点提示 | 用户悬停在数据点附近 | 显示当前维度的点位和对应值 |

交互应优先服务“读清当前系列”和“比较系列差异”，不应加入与多维对比无关的复杂操作。

## 11. G2 / ECharts 映射

### 11.1 G2

| 语义 | G2 映射 |
| --- | --- |
| 图表类型 | `line()` / `area()` + polar coordinate |
| 维度字段 | `encode("x", dimensionField)` |
| 指标字段 | `encode("y", valueField)` |
| 系列字段 | `encode("color", seriesField)` |
| 闭合轮廓 | 使用极坐标并闭合每个系列路径 |
| 数据区域 | area mark 或 line + area 组合 |
| Tooltip | `interaction("tooltip")` |

### 11.2 ECharts

```js
option = {
  radar: {
    indicator: [
      { name: "客服咨询", max: 6 },
      { name: "物流履约", max: 6 },
      { name: "售后服务", max: 6 },
      { name: "商品体验", max: 6 },
      { name: "服务附加", max: 6 },
      { name: "星级表现", max: 6 }
    ]
  },
  tooltip: {},
  legend: {},
  series: [
    {
      type: "radar",
      data: [
        {
          name: "店铺综合表现",
          value: [3.8, 4.8, 4.3, 4.2, 5.3, 4.6],
          areaStyle: {
            opacity: 0.1
          },
          lineStyle: {
            width: 2
          }
        }
      ]
    }
  ]
};
```

ECharts 映射规则：

- `radar.indicator` 对应维度轴。
- `series.type = "radar"` 对应雷达图。
- `series.data[].value` 按维度顺序写入指标值。
- `series.data[].name` 对应系列名称。
- `lineStyle.width` 使用 `2`。
- `areaStyle.opacity` 使用 `0.1`。

## 12. AI 输出格式

AI 生成雷达图设计或代码时，应输出：

```json
{
  "chartType": "radar",
  "purpose": "multi-dimensional-comparison",
  "dataMapping": {
    "dimension": "维度字段",
    "value": "指标字段",
    "series": "可选系列字段"
  },
  "constraints": {
    "dimensionCount": "5-10",
    "seriesCount": "1-5，推荐 2-3",
    "normalized": true
  },
  "style": {
    "dataLineWidth": 2,
    "dataAreaOpacity": 0.1,
    "showValueLabel": false
  },
  "interaction": {
    "tooltip": true,
    "highlight": true,
    "legendFilter": "多系列时启用"
  }
}
```

## 13. 禁止项

- 禁止在维度少于 3 个时强行生成雷达图。
- 禁止在维度超过 10 个时直接生成单张雷达图。
- 禁止在系列超过 5 个时继续叠加展示。
- 禁止将不同单位、不同量纲的原始指标直接放在同一雷达图中比较。
- 禁止把雷达图用于时间趋势分析。
- 禁止把雷达图用于部分与整体占比表达。
- 禁止用数据区域面积解释精确数值大小。
- 禁止默认显示所有数值标签。
- 禁止使用不透明或高饱和填充遮挡网格、轴线和其他系列。
- 禁止在多系列雷达图中省略图例。

## 14. AI 自查清单

生成雷达图前，AI 必须检查：

- 是否存在多个可比较维度？
- 维度数量是否在 5-10 个以内？
- 系列数量是否不超过 5 个，且是否优先控制在 2-3 个？
- 各维度指标是否单位一致，或是否已经标准化？
- 是否明确了维度顺序和每个系列的完整值？
- 是否使用 `2px` 数据线表达轮廓？
- 是否将数据区域透明度控制为 `10%`？
- 多系列时是否启用了图例、高亮和过滤？
- 是否只在必要时显示数值标签？
- 是否绑定了主题变量、分类色、字体、图例和 Tooltip 规则？
