# 字体排版 Typography

## 1. 文档定位

本文定义图表中的文字样式、字号、行高、字重和不同图表元素的排版规则。

本文回答：

> 图表标题、坐标轴、图例、标签、Tooltip、指标值应该绑定哪一个文字样式？

本文不维护完整颜色 token 值。文字颜色、Light / Dark 解析值应查阅 `02.主题变量 Theme-token.md`。

参考来源：

- Figma 文件：`dongDesign for Chart AI`
- Figma 节点：`设计变量 Design Tokens / 字体样式 Typography`

## 2. 规范原则

可视化系统中的字体规范必须使用确定值，不使用区间值。

```txt
允许字号：12 / 14 / 16
允许行高：18 / 20 / 24
允许字重：400 / 600
```

禁止项：

- 禁止使用 `12-14px`、`14-16px`、`500-600` 这类模糊区间。
- 禁止 AI 根据空间自行推断字号。
- 禁止在图表内部使用 10、11、13、15、18、20、24、28、32 等非通用字号。
- 禁止通过额外加粗制造层级，字重只能使用 `400` 或 `600`。

说明：Figma 中存在 `Title / .D1` 等大屏标题样式，该样式用于大屏或展示页标题，不纳入本可视化 Wiki 的通用图表排版规范。只有在明确的大屏标题场景中，才允许单独引用大屏标题规范。

## 3. 字体样式 Token

当前图表系统只使用 Body 和 Number 两组通用样式。

### 3.1 Body 文本

Body 用于中文、英文、说明文字、标题、图例、坐标轴和 Tooltip 文本。

| Token | 字号 / 行高 | 字重 | 用途 |
|---|---:|---:|---|
| `.textLarge` | `16 / 24` | `400` | 大号正文、区块标题基础样式 |
| `.textMedium` | `14 / 20` | `400` | 图表标题、图例、Tooltip 标题基础样式 |
| `.textSmall` | `12 / 18` | `400` | 坐标轴、辅助说明、Tooltip 正文、数据标签 |

强调态规则：

- 标题、关键标签、强调说明可在对应字号上使用 `600`。
- 强调态只改变字重，不改变字号和行高。

### 3.2 Number 数字

Number 用于图表中的数值、指标、Tooltip 数值、数据标签数字和表格型图表数字。

| Token | 字号 / 行高 | 字重 | 用途 |
|---|---:|---:|---|
| `.numberLarge` | `16 / 24` | `400` | 核心指标值、小型指标卡主值 |
| `.numberMedium` | `14 / 20` | `400` | Tooltip 数值、表格数字、普通指标值 |
| `.numberSmall` | `12 / 18` | `400` | 坐标轴刻度、数据标签、辅助数值 |

强调态规则：

- 核心指标、异常数值、当前选中值可在对应字号上使用 `600`。
- 不允许为了突出指标而临时放大到 `16` 以上。

## 4. 字体族

图表文字优先使用系统字体，保证跨平台可读性。

推荐字体栈：

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", "Helvetica Neue", Arial, sans-serif;
```

建议映射：

| 类型 | 字体 | 用途 |
|---|---|---|
| Body | `PingFang SC` / 系统无衬线 | 中文、英文、图例、说明、Tooltip |
| Number | `Roboto` / 系统数字字体 | 数值、指标、刻度、数据标签 |

使用规则：

- 图表内不使用装饰性字体。
- 中文、英文、数字混排时，以数字可读性和对齐为优先。
- 数字和单位之间应保持清晰分隔。
- 如果代码环境无法加载 Roboto，使用系统无衬线字体回退，但仍保持 `.number*` 的字号、行高和字重。

## 5. 图表元素样式映射

AI 生成图表时，应按下表绑定文字样式，不自行选择区间值。

| 元素 | 默认样式 | 强调样式 | 颜色变量 |
|---|---|---|---|
| 看板区块标题 | `.textLarge` `16/24` `600` | 不再升级 | `text/400` |
| 单图标题 | `.textMedium` `14/20` `600` | 不再升级 | `text/400` |
| 副标题 / 口径说明 | `.textSmall` `12/18` `400` | 不建议强调 | `text/200` |
| 坐标轴标签 | `.textSmall` `12/18` `400` | 禁止强调 | `坐标轴/axisLabelColor` |
| 坐标轴标题 | `.textSmall` `12/18` `400` | 可用 `600` | `text/300` |
| 图例文字 | `.textSmall` `12/18` `400` | 禁止强调 | `图例/legendTextColor` |
| Tooltip 标题 | `.textMedium` `14/20` `600` | 不再升级 | `text/400` |
| Tooltip 正文 | `.textSmall` `12/18` `400` | 数值可用 `.numberSmall` | `text/300` |
| 数据标签 | `.numberSmall` `12/18` `400` | 关键值可用 `600` | `text/300` |
| 关键标注 | `.textSmall` `12/18` `600` | 不再升级 | 按语义或 `text/400` |
| 小型指标卡主值 | `.numberLarge` `16/24` `600` | 不再升级 | `text/400` |
| 普通指标值 | `.numberMedium` `14/20` `400` | 可用 `600` | `text/400` |
| 备注 / 来源 | `.textSmall` `12/18` `400` | 禁止强调 | `text/200` |

## 6. 层级规则

文字层级只通过三件事建立：

```txt
字号层级：16 > 14 > 12
字重层级：600 > 400
颜色层级：text/400 > text/300 > text/200 > text/100
```

使用规则：

- 页面或区块标题使用 `16/24 600`。
- 单图标题使用 `14/20 600`。
- 图表内部大多数辅助信息使用 `12/18 400`。
- 坐标轴、图例、备注不得通过加粗抢占视觉焦点。
- 当信息过密时，应减少文本、抽样标签或放入 Tooltip，不允许缩小到 12 以下。

## 7. 数字排版

数字是图表中的高频信息，应优先保证对齐、单位和格式一致。

规则：

- 数字使用 `.numberLarge`、`.numberMedium`、`.numberSmall` 三档。
- 大数字应使用千分位或中文单位，例如 `12,345` 或 `1.23 万`。
- 百分比保留位数应一致，例如统一 `12.3%`。
- 金额、数量、率值不得混用单位。
- 表格、排行榜、Tooltip 中数字应右对齐或视觉对齐。
- 趋势类增减值应带方向符号或说明，例如 `+12.3%`、`下降 5.2%`。

禁止项：

- 不要省略单位。
- 不要在同一图表中混用不同小数位。
- 不要只靠颜色表达正负变化。
- 不要用更大字号替代数字格式化。

## 8. 坐标轴文字

适用元素：

- xAxis label
- yAxis label
- axis title
- tick label

规则：

- 坐标轴标签固定使用 `.textSmall` `12/18` `400`。
- 坐标轴标签绑定 `坐标轴/axisLabelColor`。
- 坐标轴标签不得加粗。
- 类目名称过长时优先截断、换行、旋转或抽样。
- 时间轴标签应按粒度显示，例如日、周、月，不堆叠完整时间戳。
- y 轴标签应配合单位，避免每个刻度重复冗长单位。
- 双轴图必须明确左右轴单位和系列对应关系。

## 9. 图例文字

规则：

- 图例文字固定使用 `.textSmall` `12/18` `400`。
- 图例文字绑定 `图例/legendTextColor`。
- 图例文案应使用业务对象名称，不使用字段名或技术名。
- 图例项过多时应折叠、筛选或改用图表外控件。
- 图例不承载解释性长句。
- 图例文字不得加粗。

## 10. Tooltip 文字

Tooltip 用于承载细节，不应替代图表主体表达。

样式：

- Tooltip 标题：`.textMedium` `14/20` `600`
- Tooltip 正文：`.textSmall` `12/18` `400`
- Tooltip 数值：`.numberSmall` `12/18` `400`，关键值可用 `600`

规则：

- Tooltip 背景绑定 `提示/backgroundColor`。
- Tooltip 内容应包含系列名、数值、单位和必要上下文。
- 多系列 Tooltip 应保持顺序稳定。
- Tooltip 条目超过 10 个时应聚合或滚动。
- Tooltip 中的数值格式必须与图表主体一致。

推荐结构：

```txt
标题：时间 / 类目 / 对象
系列名：数值 单位
系列名：数值 单位
补充说明：可选
```

## 11. 数据标签

数据标签只用于强调关键值，不用于铺满所有数据。

适用场景：

- 饼图 / 环图主要占比
- 柱状图 Top / Bottom
- 折线图峰值、谷值、最新值
- 漏斗图阶段转化率
- 异常点标注

规则：

- 默认使用 `.numberSmall` `12/18` `400`。
- 关键标签可使用 `.numberSmall` `12/18` `600`。
- 标签密集时优先隐藏。
- 标签不得遮挡主图形。
- 标签与 Tooltip 不重复展示所有信息。
- 关键标签可使用语义色，但必须配合业务含义。

## 12. 标题与说明

标题规则：

- 看板区块标题固定使用 `.textLarge` `16/24` `600`。
- 单图标题固定使用 `.textMedium` `14/20` `600`。
- 标题应短，说明图表对象和指标。
- 标题不写完整分析结论。
- 同一看板中标题格式保持一致。

说明文字规则：

- 说明文字固定使用 `.textSmall` `12/18` `400`。
- 用于补充时间范围、统计口径、数据来源。
- 不在说明中堆叠过多业务解释。
- 重要口径应在图表外统一说明。

示例：

```txt
销售额趋势
2026-05-01 至 2026-05-31，按日统计
```

## 13. Light / Dark 排版注意事项

- 字号、行高、字重在 Light / Dark 下保持一致。
- 文字颜色通过 token 随 mode 切换。
- 不得在暗色模式下直接使用 light 模式的灰阶文字。
- 暗色模式下 `12/18` 小字号文字需要检查对比度。
- 如果坐标轴标签在暗色背景上不清晰，应调整 `坐标轴/axisLabelColor` 的 mode 解析值。

## 14. AI 输出格式

AI 生成或审核图表文字时，应输出明确样式，不输出范围。

推荐字段：

| 字段 | 示例 |
|---|---|
| 元素 | `xAxis.label` |
| 样式 token | `.textSmall` |
| 字号 / 行高 | `12 / 18` |
| 字重 | `400` |
| 颜色 token | `坐标轴/axisLabelColor` |
| 当前模式显示值 | 从 `Theme-token.md` 读取 resolved value |

## 15. AI 自查清单

生成或审核图表时检查：

- 是否只使用了 12、14、16 三档字号？
- 是否只使用了 18、20、24 三档行高？
- 是否只使用了 400、600 两档字重？
- 是否没有出现区间值或模糊建议？
- 标题、副标题、轴标签、图例、Tooltip 是否层级清晰？
- 是否绑定了正确的文字颜色变量？
- 坐标轴标签是否过密？
- 数字格式、单位、小数位是否统一？
- 数据标签是否只标关键值？
- Tooltip 是否包含单位和上下文？
- Light / Dark 下是否保持同一排版 token？
