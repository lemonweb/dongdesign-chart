# 坐标轴 Axis

## 1. 文档定位

本文定义图表坐标轴的类型、构成、显示规则和设计约束。

本文回答：

> 图表应该使用什么类型的坐标轴？轴标题、轴标签、轴线、刻度线、网格线应该如何显示？

参考来源：

- Figma 文件：`可视化设计实践`
- Figma 节点：`坐标轴 Axis`

本文依赖：

- `02.主题变量 Theme-token.md`

## 2. 定义

坐标轴是一维空间中统计图表中的轴，用来定义坐标系中数据在方向和值上的映射关系。

坐标轴是坐标系的构成部分，是定义域轴和值域轴的统称。坐标轴所表示的范围越大，轴所包含的概念量越多。

坐标轴用于帮助用户：

- 理解数据方向。
- 判断数据大小。
- 建立数据与图形位置之间的映射。
- 阅读图表中的趋势、分布、差异和比例关系。

## 3. 应用场景

### 3.1 直角坐标系

直角坐标系适合展示具有线性关系、趋势对比或类别对比的数据。

常见结构：

```txt
xAxis = 横向维度或时间
yAxis = 纵向数值或指标
```

适用图表：

- 折线图
- 柱状图
- 条形图
- 散点图
- 区域图
- 组合图

使用规则：

- X 轴通常承载维度、时间或有序分类。
- Y 轴通常承载连续数值。
- 条形图中，维度通常在 Y 轴，指标通常在 X 轴。

### 3.2 极坐标系

极坐标系适合展示环状、角度或比例关系的数据。

常见结构：

```txt
angleAxis = 角度方向
radiusAxis = 半径方向
```

适用图表：

- 饼图
- 玫瑰图
- 雷达图
- 玉玦图

使用规则：

- 角度轴用于表达分类或周期位置。
- 半径轴用于表达数值大小。
- 雷达图中的指示器名称相当于轴标签，必须保持可读。

## 4. 坐标轴构成

虽然不同坐标系下坐标轴的表现形式不同，但构成元素基本一致。

| 构成 | 英文 | 作用 | 变量 / 样式 |
|---|---|---|---|
| 轴标题 | title | 说明轴变量、单位或口径 | `.textSmall` `12/18`，`text/300` |
| 轴线 | axisLine / line | 标识坐标轴基线 | `坐标轴/axisLineColor` |
| 轴刻度 | tick | 标识刻度位置 | `坐标轴/tickColor` |
| 轴标签 | label | 展示刻度值或分类名称 | `.textSmall` `12/18`，`坐标轴/axisLabelColor` |
| 网格线 | grid / splitLine | 辅助跨区域读数 | `坐标轴/splitLineColor` |
| 分隔区域 | splitArea | 辅助分区阅读 | `坐标轴/splitAreaColor` |

基础样式：

| 项目 | 固定规则 |
|---|---|
| 轴标签字号 | `.textSmall` `12/18` |
| 轴标签字重 | `400` |
| 轴标题字号 | `.textSmall` `12/18` |
| 轴标题字重 | `400`，必要时可用 `600` |
| 轴线颜色 | `坐标轴/axisLineColor` |
| 刻度线颜色 | `坐标轴/tickColor` |
| 网格线颜色 | `坐标轴/splitLineColor` |
| 分隔区域颜色 | `坐标轴/splitAreaColor` |

坐标轴属于辅助阅读元素，视觉优先级必须低于数据图形。

## 5. 坐标轴类型

根据数据变量的类型，坐标轴分为以下类型。

| 类型 | 适用数据 | 常见用途 | 规则 |
|---|---|---|---|
| 连续轴 | 连续数值 | 金额、数量、百分比、评分 | 刻度应等距，单位明确 |
| 分类轴 | 无序分类 | 品类、渠道、会员等级 | 保留分类名称完整性 |
| 时间轴 | 时间序列 | 日期、小时、月份、季度 | 按时间粒度抽样显示 |
| 对数轴 | 跨数量级数值 | 金额、规模、流量 | 必须明确说明对数尺度 |
| 对称对数轴 | 含正负且跨度大的数值 | 增长率、偏离值、正负差异 | 必须说明中心值和尺度 |

AI 判断规则：

```txt
如果字段是连续数值 → 连续轴
如果字段是无序类别 → 分类轴
如果字段是日期或时间 → 时间轴
如果数值跨越多个数量级 → 对数轴
如果数值同时包含正负且跨度大 → 对称对数轴
```

## 6. 设计原则

坐标轴设计的核心原则：

```txt
弱化坐标系，突出数据元素。
```

应删除不必要的非数据元素，例如过多轮廓、背景、轴线、网格线，把重点放在数据元素上。

### 6.1 轴标题

轴标题用于说明坐标轴表达的变量、单位或统计口径。

文案规则：

- 简洁准确表达变量含义。
- 不使用过于专业或生僻的术语。
- 特殊格式数据应说明单位、范围或常规数据格式除外的口径，例如百分比、日期时间。
- 标题包含单位时，优先写成 `指标（单位）`，例如 `成交金额（万元）`。

布局规则：

- X 轴标题固定放在底部。
- Y 轴标题固定放在标签侧方。
- X 轴标题水平居中。
- Y 轴标题垂直居中。
- 轴标题应与轴标签保持清晰距离，避免贴合。
- 多个坐标轴标题应保持样式一致。

使用建议：

- 当图表标题、图例或上下文已经清楚说明轴含义时，可以省略轴标题。
- 当单位不明显、左右双轴、对数轴或复杂口径出现时，必须显示轴标题。

### 6.2 轴标签

轴标签用于展示刻度值、时间点或分类名称。

通用规则：

- 轴标签固定使用 `.textSmall` `12/18` `400`。
- 轴标签颜色绑定 `坐标轴/axisLabelColor`。
- 标签不得加粗。
- 标签不得与数据图形、图例、Tooltip 或其他标签重叠。
- 标签密集时，优先抽样、省略、旋转或使用缩略格式。

连续轴 / 时间轴：

- 可采用抽样显示。
- 当多个标签在容器内重叠时，优先保留首尾标签。
- 时间轴应按粒度显示，例如小时、日、周、月，不堆叠完整时间戳。

分类轴：

- 分类数据不具备连续性和相似性，应优先保证数据展示完整性。
- 分类名称过长时，优先使用旋转、换行或截断。
- 不应为了整齐而任意丢弃关键分类标签。

### 6.3 轴标签省略

对于连续轴和时间轴，可以采用抽样显示。

规则：

- 优先保留首尾标签。
- 中间标签按等步长显示。
- 抽样后仍需保持读数连续感。
- 不得让用户误以为被省略的区间没有数据。

适用：

- 小时级时间轴。
- 日期跨度较长的趋势图。
- 连续数值刻度过密的图表。

### 6.4 轴标签旋转

分类轴标签过长或数量较多时，可以旋转标签。

使用规则：

- 仅在横向空间不足时使用旋转。
- 旋转优先用于分类轴，不优先用于连续轴或时间轴。
- 旋转后仍需保证标签可读，不得过密堆叠。
- 如果旋转后仍不可读，应减少分类数量、换行或改用条形图。

不建议：

- 为了显示全部时间刻度而大面积旋转时间轴标签。
- 在窄容器中旋转过多长文本标签。

### 6.5 坐标轴隐藏

当图表已经提供有效数据标签时，可以弱化或隐藏坐标轴。

适用场景：

- 堆叠条形图已经显示百分比数据标签。
- 排名条形图每个条目都有直接数值。
- 简单指标对比图中，轴线和刻度不会增加理解。

规则：

- 隐藏坐标轴前，必须保证数据标签足够有效。
- 若隐藏轴标签，仍需通过数据标签或 Tooltip 提供数值。
- 不得同时隐藏坐标轴和所有数值说明。
- 隐藏后图表仍应能判断大小关系。

## 7. 轴线与网格线

轴线和网格线都属于辅助阅读元素，需要根据数据构成谨慎显示。

### 7.1 显示原则

根据数据构成：

- 维度数据对应轴线。
- 指标数据对应网格线。

常见映射：

| 图表结构 | 建议 |
|---|---|
| 维度在横轴 X | X 轴显示轴线，Y 轴显示水平网格线 |
| 维度在纵轴 Y | Y 轴显示轴线，X 轴显示垂直网格线 |
| 极坐标 / 雷达图 | 使用环形或径向分隔线辅助读数 |

### 7.2 网格线数量

默认情况下，直角坐标系的网格线建议只显示 `5` 个。

规则：

- 过多分割线会让图表过于杂乱。
- 网格线应弱于数据系列。
- 网格线颜色绑定 `坐标轴/splitLineColor`。
- 若图表空间较小，应减少网格线数量。
- 不应为了精确读数而堆叠大量网格线；精确值应由 Tooltip 或数据标签承载。

### 7.3 轴线与刻度线

规则：

- 轴线颜色绑定 `坐标轴/axisLineColor`。
- 刻度线颜色绑定 `坐标轴/tickColor`。
- 刻度线只用于辅助定位，不应成为视觉焦点。
- 坐标轴线可在极简趋势图中弱化或隐藏，但必须保留必要读数线索。

## 8. 异轴与双轴

异轴，也就是双轴图，指在同一个图表中使用两个不同的 Y 轴比较两个或多个数据系列。

适用场景：

- 两个数据系列单位不同。
- 两个数据系列值域差异很大。
- 需要展示两个数据系列之间的关系。

使用规则：

- 双轴必须明确左右轴分别对应哪个系列。
- 左右轴标题必须显示单位。
- 左右轴样式应保持一致。
- 不得通过双轴制造虚假的趋势相关性。
- 如果两个指标可以统一单位或归一化，应优先统一尺度，而不是使用双轴。

推荐：

- 左右 Y 轴都清楚显示单位和刻度。
- 系列颜色、图例和轴标题之间应建立明确对应关系。

不建议：

- 双轴标题位置不一致。
- 一个轴有标题，另一个轴无标题。
- 用双轴把无关指标强行放在同一图表中。

## 9. 与主题变量的关系

坐标轴组件必须绑定主题变量，不得写死颜色。

| 轴元素 | 绑定变量 | Light | Dark |
|---|---|---|---|
| 轴线 | `坐标轴/axisLineColor` | `#D9D9D9` | `#666666` |
| 刻度线 | `坐标轴/tickColor` | `#D9D9D9` | `#666666` |
| 网格线 | `坐标轴/splitLineColor` | `#EBEBEB` | `#4A4A4A` |
| 分隔区域 | `坐标轴/splitAreaColor` | `#F7F7F7` | `#333333` |
| 轴标签 | `坐标轴/axisLabelColor` | `#8C8C8C` | `#FFFFFF` |

说明：

- Light / Dark 只改变变量解析值，不改变坐标轴结构。
- 坐标轴颜色必须弱于数据系列。
- 如果暗色模式下轴标签过亮或过弱，应调整 mode token，而不是在单个图表中写死色值。

## 11. AI 输出格式

AI 生成或审核图表时，应输出坐标轴决策。

推荐字段：

| 字段 | 示例 |
|---|---|
| 坐标系 | `直角坐标系` |
| xAxis 类型 | `time` |
| yAxis 类型 | `value` |
| xAxis 标题 | `日期`，可省略 |
| yAxis 标题 | `成交金额（万元）` |
| xAxis 标签策略 | `保留首尾，按等步长抽样` |
| yAxis 网格线数量 | `5` |
| 轴标签样式 | `.textSmall 12/18 400` |
| 轴标签颜色 | `坐标轴/axisLabelColor` |
| 轴线颜色 | `坐标轴/axisLineColor` |
| 网格线颜色 | `坐标轴/splitLineColor` |
| 是否双轴 | `false` |

## 12. 禁止项

- 禁止让坐标轴视觉强于数据系列。
- 禁止在没有单位说明时使用复杂数值轴。
- 禁止分类轴随意省略关键分类标签。
- 禁止连续轴或时间轴标签密集重叠。
- 禁止为了展示全部时间标签而大面积旋转标签。
- 禁止隐藏坐标轴后不提供任何数值说明。
- 禁止网格线数量过多导致图表杂乱。
- 禁止不说明原因就使用双轴。
- 禁止写死坐标轴颜色，必须绑定主题变量。
- 禁止在图表内部使用非 `12/18 400` 的轴标签样式。

## 13. AI 自查清单

生成或审核图表时检查：

- 是否判断了坐标系类型？
- X 轴和 Y 轴的数据类型是否正确？
- 轴标题是否必要，单位是否清楚？
- 轴标签是否使用 `.textSmall 12/18 400`？
- 轴标签是否绑定 `坐标轴/axisLabelColor`？
- 轴线、刻度线、网格线是否绑定对应变量？
- 时间轴或连续轴标签是否做了合理抽样？
- 分类轴标签是否完整可读？
- 标签旋转后是否仍然可读？
- 隐藏坐标轴时是否有有效数据标签或 Tooltip 补充？
- 网格线数量是否控制在默认 `5` 个左右？
- 双轴是否有明确必要性和单位说明？
