# 主题变量 Theme Tokens

## 1. 文档定位

本文用于定义图表主题变量框架，是设计规范、图表组件和 G2 / ECharts 代码实现之间的变量中枢。

主题变量负责回答：

> 图表中的颜色、文字、轴线、图例、Tooltip、网格、状态和交互样式，应从哪些 token 中取值？

本文不直接定义具体图表选型规则，也不展开单个图表组件的结构细节。图表选型请查阅 `00.总览 Overview/03.图表选型决策树 Decision-tree.md`，具体组件请查阅 `02.图表模式 Chart Type/`。

与相邻视觉语言文档的关系：

| 文档 | 职责 | 与本文关系 |
|---|---|---|
| `02.主题变量 Theme-token.md` | 定义 token、alias、Light / Dark 解析值 | 变量事实表 |
| `03.色彩系统 Color.md` | 定义颜色使用策略、场景判断和禁止项 | 读取本文变量，不重复维护完整色值 |
| `04.字体排版 Typography.md` | 定义图表文字层级、字号使用和阅读规则 | 读取本文文字变量，不直接决定色彩 |

AI 使用时应先读取本文获得变量与值，再读取 `Color.md` 或 `Typography.md` 判断使用规则。

## 3. 变量分层

主题变量分为 4 层，并采用“Theme × Mode”的组合方式。

| 层级              | 名称     | 职责                                     | 是否直接用于图表 |
| --------------- | ------ | -------------------------------------- | -------- |
| Mode Token      | 模式基础变量 | 提供 light / dark 下的基础色板、文字、描边、背景等基础值    | 否        |
| Theme Token     | 主题语义变量 | 定义当前主题下的分类色、顺序色、发散色、语义色                | 是        |
| Component Token | 图表组件变量 | 绑定到轴线、图例、Tooltip、Label、Series 等图表组件    | 是        |
| Runtime Token   | 运行时变量  | 映射到 G2 / ECharts theme、option、style 配置 | 是        |

使用原则：

- 图表组件不得直接引用 Mode Token。
- 图表组件应优先引用 Theme Token 或 Component Token。
- Theme Token 可以 alias 到 Mode Token，从而自动获得 light / dark 切换能力。
- Runtime Token 只做代码映射，不反向定义设计规范。
- 当 Figma 与代码出现差异时，以本文和 Figma Variables 为单一事实来源。

主题与模式关系：

```txt
Mode Token
light.tokens.json / dark.tokens.json
  ↓
Theme Token
默认.tokens.json
  ↓
Component Token
axis / tooltip / legend / background / bar
  ↓
Runtime Token
G2 theme / ECharts option
```

多主题扩展时，应新增主题文件，而不是复制 mode 文件。

```txt
主题 A / 默认.tokens.json
主题 B / 默认.tokens.json
主题 C / 默认.tokens.json
        ↓
共用 mode：light / dark
```

## 4. 命名结构

建议采用以下命名结构：

```txt
chart/{domain}/{role}/{state?}/{level?}
```

示例：

```txt
chart/color/category/01
chart/color/sequential/blue/01
chart/color/diverging/blue-orange/negative/01
chart/color/semantic/success
chart/axis/line/default
chart/legend/text/default
chart/tooltip/background/default
chart/series/bar/fill/default
```

命名规则：

- 使用小写英文。
- 层级使用 `/` 分隔。
- 序列色使用两位数字，例如 `01`、`02`、`10`。
- 状态命名优先使用 `default`、`hover`、`active`、`disabled`、`selected`、`warning`、`danger`、`success`。
- 不在变量名中写具体色值，例如不要使用 `blue-2B6BFF`。


### 5.3 light / dark 切换规则

同一个 mode token 在 light 与 dark 中保持同名和同 variableId，但 `$value` 不同。

示例：

| Token | Light | Dark | 说明 |
|---|---|---|---|
| `基础色板/blue/1` | `#EFF3FF` | `#072C6E` | 暗色模式中蓝色色阶方向反转 |
| `基础色板/blue/8` | `#3365F7` | `#84B1FA` | 同名 token 在不同模式下取不同亮度 |
| `surface/default` | `{基础色板.light.12}` | `#1E1E1E` | 卡片默认背景随模式切换 |
| `surface/overlay` | `{基础色板.light.12}` | `#2E2E2E` | Tooltip / 浮层背景随模式切换 |
| `text/300` | `{基础色板.gray.10}` | `{基础色板.dark.9}` | 图例等正文文字随模式切换 |

因此组件变量应引用 `坐标轴/*`、`提示/*`、`图例/*`、`背景/*` 等主题层变量，而不是在组件中写死 light / dark 值。

### 5.4 变量绑定与值解析契约

AI 或代码生成工具在使用主题变量时，必须区分“绑定变量”和“显示值”。

```txt
绑定变量：图表元素应该绑定的 theme token
显示值：在当前 mode 下解析 alias 后得到的实际值
```

解析顺序：

```txt
组件元素
→ Theme Token
→ alias target
→ Mode Token(light / dark)
→ resolved value
```

示例：

```txt
图例文字
→ 图例/legendTextColor
→ alias text/300
→ light: text/300 = 基础色板.gray.10 = #595959
→ dark: text/300 = 基础色板.dark.9 = #FFFFFF
```

使用要求：

- 图表元素绑定时写 theme token，例如 `图例/legendTextColor`。
- 渲染预览、代码生成或自查时读取 resolved value，例如 light 下为 `#595959`，dark 下为 `#FFFFFF`。
- 不得为了显示正确值而绕过变量，直接把色值写到图表元素上。
- 当 theme token 有 alias 时，以 alias 在当前 mode 下的解析值为准。
- 当 theme token 没有 alias 时，light / dark 使用该 token 自身值。

## 8. 颜色变量

### 8.1 分类色板 Category Palette

分类色板用于描述分类数据。常见于柱状图、折线图、饼图、散点图、图例等。

适用场景：

- 不同商品品类
- 不同渠道
- 不同城市
- 不同人群
- 不同业务对象

变量框架：

| Token | 用途 | 说明 |
|---|---|---|
| `分类色板/item-1` | 第一分类色 | alias `colorPalette/1` |
| `分类色板/item-2` | 第二分类色 | alias `colorPalette/2` |
| `分类色板/item-3` | 第三分类色 | alias `colorPalette/3` |
| `分类色板/item-4` | 第四分类色 | alias `colorPalette/4` |
| `分类色板/item-5` | 第五分类色 | alias `colorPalette/5` |
| `分类色板/item-6` | 第六分类色 | alias `colorPalette/6` |
| `分类色板/item-7` | 第七分类色 | alias `colorPalette/7` |
| `分类色板/item-8` | 第八分类色 | alias `colorPalette/8` |
| `分类色板/item-9` | 第九分类色 | alias `colorPalette/9` |
| `分类色板/item-10` | 第十分类色 | alias `colorPalette/10` |
| `分类色板/item-11` | 第十一分类色 | alias `colorPalette/11` |
| `分类色板/item-12` | 第十二分类色 | alias `colorPalette/12` |

使用规则：

- 分类色用于表达“不同对象”，不表达数值大小。
- 一个图表中分类色不宜过多，超过 8 类时应考虑合并、排序或切换为表格。
- 饼图、环图最多优先使用前 5 个分类色。
- 同一业务对象在同一页面中应保持颜色一致。
- 分类色不得用于成功、失败、警告等状态表达。

### 8.2 顺序色板 Sequential Palette

顺序色板用于表达从低到高、从弱到强、从少到多的连续变化。

适用场景：

- 热力图
- 地图
- 矩阵强弱
- 数值密度
- 评分等级

变量框架：

| Token | 用途 | 说明 |
|---|---|---|
| `有序色板/蓝 blue/1-10` | 蓝色顺序色 | alias `基础色板/blue/2-11` |
| `有序色板/绿  green/1-10` | 绿色顺序色 | alias `基础色板/green/2-11` |
| `有序色板/橙 orange/1-10` | 橙色顺序色 | alias `基础色板/orange/2-11` |
| `有序色板/红 red/1-10` | 红色顺序色 | alias `基础色板/red/2-11` |

使用规则：

- 顺序色用于表达数值强弱，不用于表达分类。
- 同一图表中应只使用一个主色相的顺序色。
- 色阶数量应根据数据分布控制，常用 5、7、10 级。
- 热力图和地图必须提供图例说明最小值、最大值和单位。

### 8.3 发散色板 Diverging Palette

发散色板用于表达围绕中心值向两侧偏离的数值关系。

适用场景：

- 正负值
- 增长与下降
- 高于均值与低于均值
- 风险与机会
- 温度、海拔、偏差等连续变量

变量框架：

| Token | 用途 | 说明 |
|---|---|---|
| `发散色板/五色发散/序列01-05` | 五段发散色 | 适合低粒度正负或类别化偏离 |
| `发散色板/十色发散/序列01-10` | 十段发散色 | 适合连续偏离、热力图或地图 |
| `chart/color/diverging/neutral` | 中心值 | 后续可补充，表达 0、均值或基准线 |

使用规则：

- 发散色必须明确中心值，例如 `0`、均值、目标值或行业基准。
- 不得把发散色用于普通分类。
- 如果正负业务含义会因场景变化而反转，必须在图例或说明中标注。
- 发散色适合热力图、地图、面积场、差值图，不适合普通饼图。

### 8.4 语义色板 Semantic Palette

语义色板用于表达状态、风险和操作含义，而不是表达分类或数值大小。

Figma Guide 中可见的核心语义：

- `normal`
- `warning`
- `danger`

变量框架：

| Token | 用途 | 说明 |
|---|---|---|
| `语义色板/normal` | 正常 | 正常状态、通过、健康 |
| `语义色板/warning` | 警告 | 需要关注、临界值、风险预警 |
| `语义色板/danger` | 危险 | 异常、失败、严重风险 |
| `chart/color/semantic/success` | 成功 | 可选扩展，用于正向完成状态 |
| `chart/color/semantic/info` | 信息 | 可选扩展，用于提示和中性信息 |

使用规则：

- 语义色优先用于状态表达，不用于普通系列分类。
- 同一页面中相同状态必须使用相同语义色。
- 负数不一定等于 danger，必须结合业务语义判断。
- warning / danger 图表必须配合文本、Tooltip 或图例解释，不只依赖颜色。

## 14. AI 使用规则

AI 在生成图表设计或代码时必须遵守：

1. 先判断图表类型，再选择对应 token。
2. 分类数据使用分类色板。
3. 连续强弱数据使用顺序色板。
4. 正负偏离数据使用发散色板。
5. 状态、风险、异常使用语义色板。
6. 组件样式优先引用组件变量。
7. 不得随机生成颜色、字号、阴影和间距。
8. 不得把 Figma 中未确认的变量值写成确定规范。
9. 输出代码时必须通过 Theme Schema 映射到 G2 / ECharts。
10. 支持暗黑模式时，不得复制一套主题变量，应通过 mode 切换解析同名基础变量。
11. 新增主题时，应新增 theme token，并尽量复用 mode 中的基础变量。
12. 输出图表元素样式时，应同时说明绑定 token 与当前 mode 下的 resolved value。
