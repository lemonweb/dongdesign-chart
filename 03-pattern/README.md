## 1. 目录定位

本目录用于定义业务分析场景中的图表组合方式。

它不关注单个图表怎么画，而关注一个业务页面应该如何组织指标、图表和分析链路。

## 2. 使用时机

以下情况应读取本目录：

- 设计数据看板
- 设计业务分析页面
- 需要组合多个图表
- 需要围绕业务目标组织指标
- 需要从概览、趋势、构成、排名、明细形成完整分析链路

## 3. 读取方式

先根据业务域选择对应文档：

- `经营分析 Business-analysis.md`
- `流量分析 Traffic-analysis.md`
- `商品分析 Product-analysis.md`
- `交易分析 Transaction-analysis.md`
- `营销活动分析 Campaign-analysis.md`

再根据场景文档引用的图表，回到 `02-chart-type/` 查看具体图表规范。

## 4. 使用原则

业务场景文档应说明：

- 业务目标
- 核心指标
- 分析路径
- 推荐图表组合
- 页面布局建议
- 常见误用

## 5. 不在本目录处理的内容

- 单个图表设计细节：查阅 `02-chart-type/`
- 图表视觉样式：查阅 `01-design-language/`
- 图表代码实现：查阅 `04-adaptation/`
